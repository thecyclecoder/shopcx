# Amplifier® API Reference

**Base URL:** `https://api.amplifier.com/`  
**Last Updated:** 2026-03-19

---

## Authentication

All requests must be made over HTTPS using HTTP Basic Authentication. Your API key is the username; password can be left blank.

```
Authorization: Basic <YOUR_API_KEY_encoded_to_Base64>
```

Alternatively, pass `?auth_token=YOUR_API_KEY` as a query string parameter.

---

## General Notes

- All request/response bodies are JSON (`Content-Type: application/json`)
- Unicode characters are **not supported** in order data (Latin characters only)
- Pagination defaults to 50 items per page; use `?page` and `?per_page` to control
- Sorting: `?sort_by=<field>&?sort_direction=asc|desc`
- Shipment and Inventory report endpoints do **not** support pagination

### HTTP Status Codes
| Code | Meaning |
|------|---------|
| 200 | OK |
| 400 | Bad Request — invalid or missing parameter |
| 401 | Unauthorized — invalid API key |
| 404 | Not Found |
| 500 | Server error |

### Validation Error Response (400)
```json
{
  "message": "Invalid request",
  "errors": [
    { "name": "sku", "value": "", "message": "'sku' should not be empty." }
  ]
}
```

### Sandbox Mode
Create Sandbox API keys in your account settings. Orders created with a sandbox key are **not** sent for fulfillment. Shipment reports for sandbox orders return fake `shipment_id` and `tracking_number` values.

## Billing

### List Invoices
`GET /billing/invoices`

**Query Parameters:**
| Param | Type | Required | Description |
|-------|------|----------|-------------|
| start_date | string | Yes | ISO-8601 start date (inclusive) |
| end_date | string | Yes | ISO-8601 end date (inclusive) |
| page | number | No | Page number (default: 1) |
| per_page | number | No | Page size (default: 50) |

**Response 200:**
```json
{
  "data": [
    {
      "number": "103373",
      "date": "2016-07-31T05:00:00+00:00",
      "due_date": "2016-08-15T00:00:00",
      "status": "Completed",
      "amount": 1611.63,
      "amount_to_pay": 0.00,
      "links": {
        "csv": { "href": "https://api2.amplifier.com/billing/invoices/103373/csv" },
        "pdf": { "href": "https://api2.amplifier.com/billing/invoices/103373/pdf" }
      }
    }
  ],
  "page": 1,
  "per_page": 50,
  "total": 3,
  "total_pages": 1
}
```

### Get Invoice CSV
`GET /billing/invoices/{invoice_number}/csv`

Returns `text/csv`.

### Get Invoice PDF
`GET /billing/invoices/{invoice_number}/pdf`

Returns `application/pdf`.

| upc | string | No | Max 50 chars |

| color | string | No | Max 50 chars |
| size | string | No | Max 50 chars |
| weight | decimal | No | Decimal pounds |
| height | decimal | No | Decimal inches |
| length | decimal | No | Decimal inches |
| width | decimal | No | Decimal inches |
| backorderable | boolean | No | Default: false |
| discontinued | boolean | No | Default: false. Set true if no longer selling. Use DELETE to fully remove. |
| ship_separately | boolean | No | Default: false |
| group_ship | boolean | No | Whether to group ship |
| country_of_origin | string | Yes | ISO 3166-1 alpha-2 code |

`POST /items`

**Request Body:** Item attributes JSON (see table above)

**Response 200:**
```json
{ "id": "e515f421-21e5-40f9-a1be-4dc8c07f061c" }
```

### List / Search Orders
`GET /orders`

**Query Parameters:**
| Param | Type | Description |
|-------|------|-------------|
| page | number | Default: 1 |
| per_page | number | Default: 50 |
| sort_by | string | Default: date |
| sort_direction | string | asc or desc (default: desc) |
| start_date | string | ISO 8601 |
| end_date | string | ISO 8601 |
| order_source_code | array | Comma-separated list |
| status | array | Comma-separated. Values: `Awaiting Inventory`, `Processing Shipment`, `In Fulfillment`, `Shipped`, `Pending Cancellation`, `Cancelled` |

**Response 200** includes order objects with `shipments` array containing tracking info.

### Retrieve an Order
`GET /orders/{id}`

Returns full order object including `line_items` and `shipments` arrays.

## Order Sources

### List Order Sources
`GET /order-sources`

Query params: `page`, `per_page`, `sort_by` (default: name), `sort_direction` (default: asc)

**Response 200:**
```json
{
  "data": [
    { "id": "fdfd397a-...", "name": "Order Source 1", "code": "order-source-1" }
  ],
  "page": 1, "per_page": 50, "total": 3, "total_pages": 1
}
```

### Get Order Source
`GET /order-sources/{id}`

Returns `404` if not found.

### Create Order Source
`POST /order-sources`

**Request Body:**
```json
{ "name": "New Order Source" }
```

`name` is required, unique, max 50 chars. Returns `{ id, name, code }`.

### Delete Order Source
`DELETE /order-sources/{id}`

Returns `400` if system-created source. Returns `404` if not found.

## Webhooks

Events are sent as HTTP POST (JSON) to your configured URL. Timestamps are UTC.

Retries: up to 10 times with exponential backoff if non-2xx or timeout (60s).

### Webhook Event Types
- `order.received`
- `order.shipped`
- `campaign.created`
- `campaign.updated`
- `campaign.deleted`
- `campaign.offer.created`
- `campaign.offer.deleted`
- `campaign.offer.redeemed`
- `campaign.offer.revoked`
- `campaign.offer.notification_sent`
- `campaign.offer.notification_failed`

### Event Payload Examples

**order.received:**
```json
{
  "id": "163882cc-2fc8-4840-83d2-6e96136817f5",
  "type": "order.received",
  "timestamp": "2018-04-10T16:51:05.463",
  "data": {
    "id": "7cde7ed1-8f57-4e64-9864-00938bd1a50b",
    "reference_id": "TEST-123",
    "order_source": "test-orders"
  }
}
```

**order.shipped:**
```json
{
  "id": "896af072-a4cf-4c03-ad23-1ad4ee2aabb5",
  "type": "order.shipped",
  "timestamp": "2018-04-10T18:23:01.643",
  "data": {
    "id": "1cd6852b-...",
    "reference_id": "176",
    "method": "UPS Ground",
    "tracking_number": "1ZT283T6YW75570771",
    "date": "2018-04-10T18:21:05.784",
    "items": [
      { "sku": "ABC001", "description": "Sample Item", "quantity": 1 }
    ]
  }
}
```

### Create Webhook
`POST /webhooks`

```json
{ "event_type": "order.shipped", "url": "https://yoursite.com/webhooks/order-shipped" }
```

### Get Webhook
`GET /webhooks/{webhook_id}`

### List Webhooks
`GET /webhooks`

Query params: `page`, `per_page`, `sort_by` (default: event_type), `sort_direction`

### Delete Webhook
`DELETE /webhooks/{webhook_id}`
