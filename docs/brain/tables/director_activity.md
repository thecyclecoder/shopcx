# director_activity

The timestamped **action log** every director (and every worker a director supervises) writes a row to on each action it takes ([[../goals/devops-director]]). That single log is the substrate for **(1)** the autonomous-approval **audit history**, **(2)** the gamified [[../libraries/director-board|#directors board]] posts, and **(3)** the **EOD recap** ‚ÄĒ a read over *today's* rows, never hand-maintained.

Written via [[../libraries/director-activity]] `recordDirectorActivity` (best-effort, service-role only). The **first concrete writer** is the [[../specs/regression-agent|Regression Agent]] (a worker the Platform/DevOps Director supervises): every detect / dismiss / author / escalate action writes one row. The live directors (M4+) write the same shape (`approved_migration`, `fixed_bug`, `escorted_goal`, ‚Ä¶).

**Workspace-scoped** (mirrors [[director_messages]] / [[spec_card_state]]). RLS: any authenticated user reads (the board + recap surfaces are owner-gated above the DB); service role does all writes.

**Migration:** `supabase/migrations/20260702120000_director_activity.sql` ¬∑ apply via `npx tsx scripts/apply-director-activity-migration.ts`.

**Primary key:** `id`

## Columns

| Column | Type | Notes |
|---|---|---|
| `id` | `uuid` | PK ¬∑ `gen_random_uuid()` |
| `workspace_id` | `uuid` | FK ‚Üí `workspaces(id)` on delete cascade |
| `director_function` | `text` | the function slug whose objective owns the action (e.g. `platform`); a **worker** action carries its **supervising director's** function |
| `action_kind` | `text` | what was done ‚ÄĒ **open vocabulary, no CHECK** (new kinds land without a migration). regression-agent emits: `detected_regressionÔĹúdismissed_regressionÔĹúauthored_fixÔĹúescalated`; the [[../libraries/agent-coaching|coaching pass]] emits: `coached_workerÔĹúcoaching_routed_to_repairÔĹúescalated_coaching`; the [[../libraries/platform-director|Platform/DevOps Director]] emits: `approved_approvalÔĹúescorted_goalÔĹúescorted_fixÔĹúescorted_initÔĹúescalatedÔĹúreconciled_error` (the last = [[../specs/director-zero-backlog-error-autonomy]] Phase 1 ‚ÄĒ one row per backlog action: an enqueued repair diagnosis or a stuck-fix escalation, `metadata: { signature, source, action: 'enqueued_repair'ÔĹú'escalated_stuck' }`) + `reconciled_sequence` ([[../specs/goal-milestone-build-sequencing]] Phase 3 ‚ÄĒ one row per out-of-order milestone build re-sequenced: the premature build held at the `held` status + the authored `**Blocked-by:**` order applied to the spec file, `metadata: { goal_slug, held_job_id, held_from_status, held, declared_blockers, unmet_blockers, applied_blockers }`) + `triaged_needs_attention` ([[../specs/needs-attention-triage-and-verdict-robustness]] Phase 1 ‚ÄĒ one row per triaged NON-build parked item: a recoverable inconclusive QC verdict re-run once or a genuine blocker surfaced to the CEO, `metadata: { job_id, kind, action: 'rerun'ÔĹú'escalated', reason, dedupe_key }`) + (board-grooming) `groomed_continueÔĹúgroomed_split` + (board-grooming richer verdicts [[../specs/director-judgment-lanes-fold-author-dismiss]]) `groomed_fold_nowÔĹúgroomed_authored_specÔĹúgroomed_dismissedÔĹúinit_authored_specÔĹúinit_dismissedÔĹúrepair_authored_spec` ([[../specs/director-supervised-repair-dismissal]] ‚ÄĒ supervising Rafa's no-fix items) `dismissed_repairÔĹúkept_repair` (`metadata: { dismiss_key, repair_job_id, signature, title, verdict }`; Phase 2 adds `reopened_repair` ‚ÄĒ written by `POST /api/developer/control-tower/repair {action:'reopen'}` when the owner overrides a dismissal, `metadata: { repair_job_id, signature, requeued, reopened_by }`, which clears the item from the Control Tower's Director-dismissed surface); a director proposing a goal emits `proposed_goal` ([[../specs/director-proposed-goals]] ‚ÄĒ written by `scripts/builder-worker.ts` `runProposedGoalJob` when the inert `proposed` artifact is committed, `metadata: { goal_slug, owner_function, proposer_function }`); [[../specs/director-dismiss-park-and-short-circuit-spec]] Phase 1 ‚ÄĒ a director's `dismiss-park` action emits `dismissed_park` (`metadata: { job_id, spec_slug, prior_class, target_kind, autonomous: true }`); the owner-gated `POST /api/developer/director-activity/reopen-park` re-open emits `reopened_park` (`metadata: { job_id, target_kind, spec_slug, reopened_by: 'owner' }`); a malformed dismiss-park payload (missing jobId / reason, out-of-leash owner, row not parked) emits `invalid_dismiss_park_action` instead. Phase 2 ‚ÄĒ a director's `spec-status` action carrying `shortCircuit:true` emits the standard `spec_status_flipped` row with `metadata.short_circuit=true` so the activity feed surfaces the short-circuit transition. CEO bouncing director escalations back ([[../specs/bounce-escalation-back-to-director]] / [[../libraries/director-bounce-back]]) emits `bounced_back_by_ceo` (`metadata: { lane, original_notification_id, original_escalation_reason, ceo_note?, new_job_id, depth:1 }`); a director re-escalating after a bounce emits `re_escalated_after_bounce` (`metadata: { lane, depth:2, original_notification_id, bounce_job_id, diagnoses:[original, post_bounce] }`). **re-author-re-opens-dismissed** ‚ÄĒ `[[../libraries/director-activity]] clearDirectorSpecDismissals` (called by `[[../libraries/author-spec]] reopenIfReauthoredAndChanged`) emits `spec_reopened_after_reauthor` (one row when a RE-AUTHORED spec's standing `init_dismissed`/`groomed_dismissed` dedup rows are DELETED so the corrected content re-enters review‚Üíbuild instead of sitting dead under the stale verdict, `metadata: { cleared_dismissals, cleared_keys, autonomous:true }`). The [[../libraries/deploy-guardian|Deploy Guardian]] emits `deploy_healthyÔĹúdeploy_regressedÔĹúdeploy_unsureÔĹúdeploy_rolled_back` (one per evaluated [[deploy_watches]] canary window ‚ÄĒ `deploy_rolled_back` = a regressed deploy auto-reverted to the prior good build (Phase 2), `metadata: { deploy_watch_id, branch, merge_sha, verdict, new_error_signatures, new_red_loops, revert_sha?, prior_rollbacks?, loop_guard?, revert_failed?, ... }`) |
| `spec_slug` | `text` | the spec the action touched (null for a non-spec action) |
| `reason` | `text` | the plain-text "why" ‚ÄĒ the reasoning the recap/audit reads back ¬∑ default `''` |
| `metadata` | `jsonb` | structured per-action context: `{ job_id?, signature?, failing?, attempt?, verdict?, approver?, owner_function?, ... }` ¬∑ default `{}` ¬∑ **`owner_function`** ([[../specs/director-drives-all-specs-and-deferred-status-board-reflects-cross-dept-drive]] Phase 1) ‚ÄĒ the OWNING function of the spec the action touched, stamped by every Platform-director drive lane that may cover for a not-yet-live department (`escorted_fixÔĹúescorted_initÔĹúgroomed_continueÔĹúgroomed_splitÔĹúgroomed_fold_nowÔĹúgroomed_authored_specÔĹúgroomed_dismissedÔĹúinit_authored_specÔĹúinit_dismissed`); the daily Platform watch counts rows where `owner_function ‚Č† 'platform'` (= the keystone covering cross-dept) and names them per-owner ("Drove K specs for other departments (cs: 2, growth: 1)"). Repair-lane and approval-lane rows don't carry it (no spec owner / it's an approval, not a drive). |
| `created_at` | `timestamptz` | default `now()` (the EOD recap filters `created_at >= today`) |

## Indexes

- `director_activity_ws_created_idx` on `(workspace_id, created_at desc)` ‚ÄĒ the recap/audit read.
- `director_activity_function_idx` on `(director_function, created_at desc)` ‚ÄĒ per-director slice.
- `director_activity_spec_idx` on `(spec_slug)` ‚ÄĒ per-spec audit slice.
