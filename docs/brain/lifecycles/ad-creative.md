# Lifecycle — Ad Creative Agent (Dahlia)

End-to-end trace of how a fresh Meta static ad goes from raw product intelligence to a ready-to-test creative in [[../functions/growth|Bianca]]'s bin, with **no human gate** — because every claim is verifiable by construction and the render is vision-QA'd. Dahlia is a worker under Max, peer to Bianca ([[../functions/growth]]).

## The chain

1. **Cadence** — [[../inngest/ad-creative-cadence]] `adCreativeCadenceCron` (`0 11 * * *`) fans out per workspace; `dispatchAdCreativeCadence` measures each intelligence-backed product's bin depth ([[../libraries/ready-to-test]] `listReadyToTest`) and enqueues an [[../tables/agent_jobs]] `kind='ad-creative'` job (with the deficit) for any product below `DEFAULT_BIN_FLOOR` (4). Idempotent per UTC day.

2. **Claim + dispatch** — [[../libraries/builder-worker]] fills the `ad-creative` lane (`MAX_AD_CREATIVE`, deterministic Node) and `runAdCreativeJob` calls [[../libraries/creative-agent]] `runAdCreativeLoop`.

3. **Intelligence** — [[../libraries/product-intelligence]] `getProductIntelligence` fans out every `product_*` table for the product (benefits, ingredient research + citations, ad angles, review clusters, featured/transformation reviews, media, variants, store proof points, the computed offer).

4. **Angle** — [[../libraries/creative-brief]] `selectAngles` scores candidates on **two axes**: `acquisitionPower` (cold-scroll stopping power — a real transformation, a skeptic objection, curiosity) vs `retentionTruth` (energy-no-crash, taste). The commodity "no-crash" angle — the #1 review cluster but a converts-nobody cold hook — is **demoted** to supporting body copy. The top acquisition angle leads.

5. **Brief** — `buildCreativeBrief` assembles the hook + lead proof (a real review or ingredient citation) + a **single-reviewer transformation** (quote + name + photo, all the same person) + supporting retention truths + proof stack + the **offer as an allowed price treatment** (strikethrough→discount or per-serving-vs-latte — never bare MSRP) + image refs.

6. **Generate — 3-placement pack ([[../libraries/creative-pack]] Phase 2)** — [[../libraries/creative-generate]] `generateCreative` renders THREE placement-sized statics from the same brief concept: feed 4:5 (canonical), stories/reels 9:16, right-column 1:1. Same psychology, only aspect ratio varies. Meanwhile, [[../libraries/creative-pack]] `buildMetaCopyPack` produces four headline variations + four primary-text variations (one LF8 driver, varied hooks — Meta rotates them). A before/after MAY be an AI-generated **photorealistic full-body** transformation (CEO grey-area call, 2026-07-10) — but the quote+name are a real review and nothing captions the image as authentic.

7. **QC gate** — [[../libraries/creative-qa]] runs the vision pass on the canonical feed_4x5: headline exact, all text legible, no bare price, no fabricated authenticity caption, transformation photorealistic (not cartoon). Two paths keyed on the `DAHLIA_QC_MODE` env kill-switch: **`box` (default)** = `qaCreativeViaBoxSession` — a top-level `claude -p` on Max via the [[../libraries/creative-qc|creative-qc]] skill (dahlia-creative-qc-via-box-session Phases 1–3; the tmp jpeg lives at `/tmp/creative-qc-<uuid>.jpg` and is deleted after), so the lane never needs `ANTHROPIC_API_KEY`. Phase 3 / Fix 1 wraps the prompt via [[../libraries/creative-qc-sandbox]] `buildQcPrompt` (injection-safe prompt layer) and gates tool access via [[../libraries/ad-creative-qc-permission-gate]] (PreToolUse hook). **`direct`** = `qaCreative`, a direct Opus vision API call unchanged — the one-flag revert if the box path misbehaves (any other value degrades to `box`). Both paths **fail closed**: any session error, cap, timeout, undecodable image, or unparseable/non-JSON verdict yields `pass:false` so nothing unchecked reaches the bin. On `pass:false`, regenerate (up to `MAX_QA_ATTEMPTS`). The canonical passes; siblings inherit the same status.

7b′. **Imitate-then-innovate path (dahlia-preserve-competitor-copy-dna-debranded Phase 2 — WIRED)** — when the driving `angle.source='competitor'`, step 7b runs the **imitate path**: the same composition-transferred image the M1 keystone rendered (Nano Banana Pro path unchanged — the competitor's proven VISUAL layout with our packshot swapped in) is paired with a debranded six-slot `COMPETITOR_DNA` payload (`hook / framework / mechanism_claim / proof / offer + competitor_advertiser`) in Dahlia's session input. The worker resolves the workspace brand from `workspaces.name`, then calls the pure `debrandForOurBrand(slot, competitorAdvertiser, ourBrand)` helper from [[../libraries/debrand]] (`src/lib/ads/debrand.ts`) per slot so the winner's market-tested WORDS reach Dahlia with brand tokens stripped. The [[../../.claude/skills/dahlia-copy-author/SKILL.md|dahlia-copy-author]] skill's IMITATE-DEBRANDED rule instructs Dahlia to prefer the debranded slot values as the seed for her headline / primary text — reusing the language the competitor's 45+ paid days already proved instead of collapsing back to a generic benefit — and to cite every preserved claim with `source='competitorDna'` and a `source_ref` naming which slot the claim came from (the M2 never-fabricate firewall layer 3 already recognises this source in `verifyClaimTrace` and reads the exact slot values). Own-brand angles carry `null` competitorDna and fall through to the fresh-authoring path in 7b; deterministic-mode / no-dispatcher runs skip 7b entirely (step 8 uses `buildMetaCopyPack` unchanged).

7b″. **RIFF — lead benefit woven into the competitor angle ([[../specs/dahlia-hooks-riff-competitor-angle-and-weave-in-lead-benefit]] Phase 2, 2026-07-18)** — every competitor-source brief also carries `leadBenefitWeave: { benefitName, softPhrasings }` from `pi.benefits` where `role='lead'` ([[../libraries/creative-brief]]), so Dahlia's SKILL RIFF rule requires the authored hook to BLEND the competitor's proven framework (from `competitorDna`) with our differentiated lead benefit (soft phrasing OK, e.g. `"feel lighter"`) — CEO's north-star shape is `"Tasty coffee, feel lighter, no jitters"` for Amazing Coffee, not the pure-borrow `"Tired of the coffee jitters?"` that leaves the differentiator absent. The `softPhrasings` array is the benefit row's `customer_phrases` verbatim (grounded — no fabrication). [[../libraries/creative-agent]] `stockProduct` runs the pure `markPureCompetitorMinoritySlot(plan)` allocator BEFORE `buildCreativeBrief`: it flags AT MOST ONE competitor slot per batch (the last one, only when the batch already carries ≥2 competitor slots) as `pureCompetitor: true`, threaded into `buildCreativeBrief(..., { pureCompetitor })` so the minority explore slot omits `leadBenefitWeave` and ships a pure-borrow imitation for learning while the anchor riffs still carry the differentiator. A batch with 0 or 1 competitor slots never emits a pure slot — the riff is the strong default. Own-brand angles are untouched (their hook already carries the benefit). Pinned by `src/lib/ads/creative-brief.test.ts` (`buildCreativeBrief — RIFF …`) + `src/lib/ads/creative-agent.test.ts` (`markPureCompetitorMinoritySlot`).

7b. **Author-mode caption ([[../libraries/creative-agent]] Phase 3 — behind `DAHLIA_COPY_MODE=author`)** — when the workspace-level env kill-switch is `author` AND [[../libraries/builder-worker]] `runAdCreativeJob` injected a `copyAuthorDispatcher`, the QC-passed canonical image is handed to Dahlia's per-creative Max box session (kind `ad-creative-copy-author`, skill [[../../.claude/skills/dahlia-copy-author/SKILL.md|dahlia-copy-author]]) via the same `runBoxLane` pattern as `qaCreativeViaBoxSession`. Same least-privilege `sandbox: "qc"` env + PreToolUse gate (Read-only on the exact `/tmp/creative-author-<uuid>.jpg`). Dahlia writes the finished caption — one headline + one primary text + one description — against the shared [[../libraries/copy-rubric]] `renderRubricForPrompt()` text + the resolved `audience_temperature` (`resolveAudienceTemperature` — cold when `angle.source='competitor'` OR `angle.acquisitionPower ≥ 8`; warm otherwise; `hot` is reserved for future retention audiences) + the debranded competitor DNA (only on `angle.source='competitor'`) + self-scores against the same rubric. **Revise loop (self-heal, copy-author-self-heal 2026-07-17):** on parse failure / session error / `self_score.total < AUTHOR_SELF_SCORE_FLOOR = 6` / `hasColdOfferLeak` trip on a cold-audience emit / **never-fabricate firewall miss** (see 7c — the firewall now runs INSIDE this loop), the worker re-invokes with a revise prompt (retry cap `MAX_COPY_AUTHOR_REVISE_ATTEMPTS`, raised **1 → 4**). Each retry RESUMES Dahlia's SAME box session (`claude --resume`) with a SHORT revise turn — the image+brief+rubric context is cached within the 1h prompt-cache TTL, so a retry costs a few hundred tokens, not a fresh context load; on a lost session (box restart → `isMissingSessionError`) or an account-cap hop the loop FAILS SAFE by re-dispatching FRESH with the full prompt. One ad per session (the qc allowlist is per-image); cold-start volume fans out as N single-ad jobs. **Exhaustion:** `stockProduct` records a `director_activity` row with `directorFunction='growth'`, `actionKind='dahlia_copy_author_exhausted'`, `specSlug='dahlia-copy-author-box-session'`, and metadata `{ product_id, product_title, angle_source, angle_hook, audience_temperature, attempts, last_reason }` — then HOLDS the campaign out of the bin (never falls back to `buildMetaCopyPack` — a silent fallback would erase the audit trail the M1 graduation gate depends on). When the flag is unset / `deterministic` OR the dispatcher is missing (a test / a manual invocation with no runBoxLane), this step is SKIPPED and step 8 uses `buildMetaCopyPack` unchanged.

7c. **Three-layer never-fabricate firewall ([[../libraries/creative-agent]] + `src/lib/ads/never-fabricate.ts` — dahlia-never-fabricate-copy-firewall Phases 1–3)** — before step 8's bin insertion, every author-mode verdict must clear three checks that together make "never fabricate a claim" structural rather than aspirational. **Layer 1** (instructional) is the CLAIM-ONLY-WHAT'S-IN-THE-BRIEF section of the [[../../.claude/skills/dahlia-copy-author/SKILL.md|dahlia-copy-author]] skill — the five claim classes (numbers · first-person testimony · ingredient names · timeframes · comparative claims) each named against the ONLY brief / ProductIntelligence source field a claim in that class may cite (`pi.ingredients` / `pi.ingredientResearch` / `pi.reviews.byClaim` / `brief.transformation` / `brief.leadProof` / `brief.supportingBenefits` / `brief.competitorDna`), so a fabricated claim never enters the copy because Dahlia can't cite a source for it. **Layer 2** (session artifact) is the REQUIRED `claim_trace: Array<{claim, source, source_ref}>` field on the AuthorModeCopy verdict — `parseAuthorVerdict` rejects a missing / empty / mis-shaped `claim_trace` with the distinct `firewall_missing_claim_trace` reason, the revise loop consumes it, and the field itself raises the cost of fabrication for a probabilistic model. **Layer 3** (deterministic gate) is `verifyClaimTrace(claim_trace, brief, pi, reviewsByClaim)` in `src/lib/ads/never-fabricate.ts`, run as the LAST gate INSIDE `runCopyAuthorSession`'s revise loop (copy-author-self-heal, 2026-07-17 — was a post-session check between `runCopyAuthorSession` and `insertReadyCreative`). `stockProduct` injects a per-attempt closure (`CopyAuthorSessionInputs.verifyClaimTrace`) that resolves `pi.reviews.byClaim(source_ref)` for every unique reviews-byClaim entry then runs the pure verifier — **fact-grounded** (CEO 2026-07-17): a faithful paraphrase passes, but any NUMBER not traceable to the cited source is `fabricated_number` and blocks (a competitor's "500M cups" grounds against OUR corpus, never their slot). A miss becomes a revise reason and Dahlia's session RESUMES to re-ground the claim; only on exhaustion is the campaign HELD out of the bin, emitting a **DISTINCT** `director_activity` row: `actionKind='dahlia_copy_firewall_exhausted'` (keyed off the exhausted outcome's `firewallMisses`), `specSlug='dahlia-never-fabricate-copy-firewall'`, metadata `{ product_id, product_title, angle_source, angle_hook, audience_temperature, misses[] }`. The distinct kind (never conflated with `dahlia_copy_author_exhausted`) lets operators slice fabrication failures from self-score failures. The StockedCreative failure row's reason string carries `firewall_claim_miss: <src>:<reason>[, ...]` for the batch summary. When step 7b is SKIPPED (deterministic mode / no dispatcher / flag off), step 7c is also SKIPPED — the deterministic `buildMetaCopyPack` produces byte-identical strings from the fully-backed brief and needs no runtime witness.

7d. **Max's FINAL copy-QC ([[../libraries/creative-agent]] + [[../libraries/creative-qa]] — max-final-qa-7of10-eligibility-gate-with-bounce-to-dahlia Phase 1–2)** — after Dahlia's author verdict clears the never-fabricate firewall (step 7c) and BEFORE step 8's `insertReadyCreative`, if [[../libraries/builder-worker]] `runAdCreativeJob` injected a `copyQcDispatcher`, `stockProduct` runs Max's INDEPENDENT copy-QC via [[../libraries/creative-qa]] `runQaCreativeCopyViaBoxSession`. The dispatcher spawns a top-level `claude -p` on Max via `runBoxLane` (kind `ad-creative-copy-qc`, `sandbox: "qc"` — no `ANTHROPIC_API_KEY`, minimal env, same `AD_CREATIVE_QC_ALLOWED_IMAGE` PreToolUse gate the image-QC + copy-author lanes use, tmp jpeg at `/tmp/creative-copy-qc-<uuid>.jpg` deleted on the way out). Max grades the entire ad on **parity access with Dahlia**: same shared consumer-psychology rubric ([[../libraries/copy-rubric]] `renderRubricForPrompt`), same `BRIEF` (the fully-backed product-intelligence surface Dahlia authored from — ingredients / ingredient research / reviews / offer / competitor DNA when the driving angle was a competitor imitation), the same `AUDIENCE_TEMPERATURE` / `TARGET_SCHWARTZ_LEVEL` / `MARKET_SOPHISTICATION_EVIDENCE`, plus `DAHLIA_SELF_SCORE` for context only (the [[../../.claude/skills/max-copy-qc/SKILL.md|max-copy-qc]] SKILL's north-star rule: judge as a scrolling buyer would refuse to click — never as a rubric mirror of Dahlia's self-score). The parsed `CopyQaVerdict` is persisted to [[../tables/ad_creative_copy_qc_verdicts]] via `insertCopyQaVerdict` after step 8's `insertReadyCreative` returns the campaign id (`retry_index: 0`; a write miss logs `max_copy_qc_verdict_insert_failed` and the pipeline continues — durable audit is best-effort). **Phase 2 (2026-07-18) — 7/10 eligibility gate WIRED:** after Max's verdict parses, `stockProduct` gates the `insertReadyCreative` call on the pure predicate `isCopyQcEligible(verdict)` — eligible IFF the verdict exists AND `hard_gate_pass` AND `persuasion_score >= MAX_QC_ELIGIBILITY_FLOOR` (`= 7`, the named exported constant). Scroll-stop sub-scores are DELIBERATELY ignored (advisory-only Goodhart guard — only the top-line score gates). A non-eligible verdict (null / hard-gate fail / sub-7 / parse error / dispatch error) does NOT reach `insertReadyCreative`: instead `stockProduct` records a growth `director_activity` row `action_kind='max_qc_below_floor_hold'` with the score / hard-gates / bounce reason on the metadata, and the StockedCreative row surfaces the distinct `max_qc_below_floor: <reason>` string for the batch summary. Phase 3 will replace the pure hold with a bounce-back-to-Dahlia self-heal through `runCopyAuthorSession`'s in-session revise loop (cache-warm resume) so Max's critiques make Dahlia BETTER. Kill-switch `DAHLIA_QC_COPY_MODE=off` skips the dispatcher (workspace-level rollback); default `box` (ON) — the CEO's design is that Max grades EVERY author-mode creative. The bench lane `runAdCreativeCopyAuthorJob` injects the dispatcher unconditionally, so the real Max QC path exercises even while the workspace-level flag stays off. The DATA fence Max reads is composed by the pure `buildCopyQcPromptPreamble` helper (`===BEGIN_COPY_QC_DATA_v1===` … `===END_COPY_QC_DATA_v1===`) — a divergence between this composer and the SKILL's documented DATA-block schema surfaces at the unit-test seam.

8. **Bin insertion ([[../libraries/creative-pack]] Phase 2–3)** — `insertReadyCreative`: [[../tables/product_ad_angles]] (stamped with the 4×4 copy pack via `metadata.copy_pack`) → [[../tables/ad_campaigns]] `status='ready'` → THREE static [[../tables/ad_videos]] siblings (feed 4:5 canonical + stories/reels 9:16 + right-column 1:1), linked via `format_variant_of_id`. All three renders uploaded to the private `ad-tool` bucket ([[../libraries/ad-storage]]). Single `landing_url` = the **battle-tested Shopify PDP** `{shopify_primary_domain}/products/{handle}` (policy CEO 2026-07-10; storefront / advertorial-variant landers are a later winner-only experiment). **Author-mode fields (Phase 3):** when step 7b produced a verdict, the row also carries `ad_campaigns.audience_temperature` = Dahlia's echoed target + `ad_campaigns.author_self_score` (jsonb) = the full rubric score envelope; the pack is a single-variant broadcast (`authorCopyPack` fills 4 headlines + 4 primary texts with Dahlia's single authored copy so `planCreativePackInserts` + `isCreativePackComplete` are unchanged). Deterministic-mode rows leave both NULL (null-means-deterministic-mode). **Completeness gate (Phase 3):** [[../libraries/creative-pack]] `isCreativePackComplete` re-checks at publish time (when Bianca calls [[../libraries/media-buyer-agent]] `publishToMeta`) — only packs with all 3 placements + ≥4 headlines + ≥4 primary texts are allowed to Meta. **LF8 gate satisfied by construction (2026-07-14):** the caption (headline + primaryText) is guaranteed to carry a [[../libraries/lf8|Life-Force-8]] term whenever the brief carries an LF8-adjacent supporting benefit, so Bianca's test loop never surfaces zero-LF8 copy to the ads-supervisor (the [[../libraries/ads-supervisor]] `live_ad_lf8_thin` gate gates on the same [[../libraries/lf8]] keywords).

9. **Handoff** — `listReadyToTest` now surfaces the campaign to Bianca's [[../libraries/media-buyer-agent]] test loop.

## North star
Dahlia optimizes a bounded proxy — *bin depth × brief quality* — and feeds Bianca; **Max owns the objective** (does the creative actually win in-market) and holds her leash. She never launches an ad herself; she stocks the bin, Bianca tests, the media-buyer grader scores realized ROAS. She is audited by the [[../libraries/ads-supervisor]] standing every-3h pass ([[../inngest/ads-supervisor-cadence]]), which checks for drift (bin depth below floor, zero seeded angles) and authors fix-specs. See [[../operational-rules.md]] § North star.
