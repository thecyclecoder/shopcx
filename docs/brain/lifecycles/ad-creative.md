# Lifecycle ‚Äî Ad Creative Agent (Dahlia)

End-to-end trace of how a fresh Meta static ad goes from raw product intelligence to a ready-to-test creative in [[../functions/growth|Bianca]]'s bin, with **no human gate** ‚Äî because every claim is verifiable by construction and the render is vision-QA'd. Dahlia is a worker under Max, peer to Bianca ([[../functions/growth]]).

## The chain

1. **Cadence** ‚Äî [[../inngest/ad-creative-cadence]] `adCreativeCadenceCron` (`0 11 * * *`) fans out per workspace; `dispatchAdCreativeCadence` measures each intelligence-backed product's bin depth ([[../libraries/ready-to-test]] `listReadyToTest`) and enqueues an [[../tables/agent_jobs]] `kind='ad-creative'` job (with the deficit) for any product below `DEFAULT_BIN_FLOOR` (4). Idempotent per UTC day.

2. **Claim + dispatch** ‚Äî [[../libraries/builder-worker]] fills the `ad-creative` lane (`MAX_AD_CREATIVE`, deterministic Node) and `runAdCreativeJob` calls [[../libraries/creative-agent]] `runAdCreativeLoop`.

3. **Intelligence** ‚Äî [[../libraries/product-intelligence]] `getProductIntelligence` fans out every `product_*` table for the product (benefits, ingredient research + citations, ad angles, review clusters, featured/transformation reviews, media, variants, store proof points, the computed offer).

4. **Angle** ‚Äî [[../libraries/creative-brief]] `selectAngles` scores candidates on **two axes**: `acquisitionPower` (cold-scroll stopping power ‚Äî a real transformation, a skeptic objection, curiosity) vs `retentionTruth` (energy-no-crash, taste). The commodity "no-crash" angle ‚Äî the #1 review cluster but a converts-nobody cold hook ‚Äî is **demoted** to supporting body copy. The top acquisition angle leads.

5. **Brief** ‚Äî `buildCreativeBrief` assembles the hook + lead proof (a real review or ingredient citation) + a **single-reviewer transformation** (quote + name + photo, all the same person) + supporting retention truths + proof stack + the **offer as an allowed price treatment** (strikethrough‚Üídiscount or per-serving-vs-latte ‚Äî never bare MSRP) + image refs.

6. **Generate ‚Äî 3-placement pack ([[../libraries/creative-pack]] Phase 2)** ‚Äî [[../libraries/creative-generate]] `generateCreative` renders THREE placement-sized statics from the same brief concept: feed 4:5 (canonical), stories/reels 9:16, right-column 1:1. Same psychology, only aspect ratio varies. Meanwhile, [[../libraries/creative-pack]] `buildMetaCopyPack` produces four headline variations + four primary-text variations (one LF8 driver, varied hooks ‚Äî Meta rotates them). A before/after MAY be an AI-generated **photorealistic full-body** transformation (CEO grey-area call, 2026-07-10) ‚Äî but the quote+name are a real review and nothing captions the image as authentic.

7. **QC gate** ‚Äî [[../libraries/creative-qa]] runs the vision pass on the canonical feed_4x5: headline exact, all text legible, no bare price, no fabricated authenticity caption, transformation photorealistic (not cartoon). Two paths keyed on the `DAHLIA_QC_MODE` env kill-switch: **`box` (default)** = `qaCreativeViaBoxSession` ‚Äî a top-level `claude -p` on Max via the [[../libraries/creative-qc|creative-qc]] skill (dahlia-creative-qc-via-box-session Phases 1‚Äì3; the tmp jpeg lives at `/tmp/creative-qc-<uuid>.jpg` and is deleted after), so the lane never needs `ANTHROPIC_API_KEY`. Phase 3 / Fix 1 wraps the prompt via [[../libraries/creative-qc-sandbox]] `buildQcPrompt` (injection-safe prompt layer) and gates tool access via [[../libraries/ad-creative-qc-permission-gate]] (PreToolUse hook). **`direct`** = `qaCreative`, a direct Opus vision API call unchanged ‚Äî the one-flag revert if the box path misbehaves (any other value degrades to `box`). Both paths **fail closed**: any session error, cap, timeout, undecodable image, or unparseable/non-JSON verdict yields `pass:false` so nothing unchecked reaches the bin. On `pass:false`, regenerate (up to `MAX_QA_ATTEMPTS`). The canonical passes; siblings inherit the same status.

7b‚Ä≤. **Imitate-then-innovate path (dahlia-preserve-competitor-copy-dna-debranded Phase 2 ‚Äî WIRED)** ‚Äî when the driving `angle.source='competitor'`, step 7b runs the **imitate path**: the same composition-transferred image the M1 keystone rendered (Nano Banana Pro path unchanged ‚Äî the competitor's proven VISUAL layout with our packshot swapped in) is paired with a debranded six-slot `COMPETITOR_DNA` payload (`hook / framework / mechanism_claim / proof / offer + competitor_advertiser`) in Dahlia's session input. The worker resolves the workspace brand from `workspaces.name`, then calls the pure `debrandForOurBrand(slot, competitorAdvertiser, ourBrand)` helper from [[../libraries/debrand]] (`src/lib/ads/debrand.ts`) per slot so the winner's market-tested WORDS reach Dahlia with brand tokens stripped. The [[../../.claude/skills/dahlia-copy-author/SKILL.md|dahlia-copy-author]] skill's IMITATE-DEBRANDED rule instructs Dahlia to prefer the debranded slot values as the seed for her headline / primary text ‚Äî reusing the language the competitor's 45+ paid days already proved instead of collapsing back to a generic benefit ‚Äî and to cite every preserved claim with `source='competitorDna'` and a `source_ref` naming which slot the claim came from (the M2 never-fabricate firewall layer 3 already recognises this source in `verifyClaimTrace` and reads the exact slot values). Own-brand angles carry `null` competitorDna and fall through to the fresh-authoring path in 7b; deterministic-mode / no-dispatcher runs skip 7b entirely (step 8 uses `buildMetaCopyPack` unchanged).

7b. **Author-mode caption ([[../libraries/creative-agent]] Phase 3 ‚Äî behind `DAHLIA_COPY_MODE=author`)** ‚Äî when the workspace-level env kill-switch is `author` AND [[../libraries/builder-worker]] `runAdCreativeJob` injected a `copyAuthorDispatcher`, the QC-passed canonical image is handed to Dahlia's per-creative Max box session (kind `ad-creative-copy-author`, skill [[../../.claude/skills/dahlia-copy-author/SKILL.md|dahlia-copy-author]]) via the same `runBoxLane` pattern as `qaCreativeViaBoxSession`. Same least-privilege `sandbox: "qc"` env + PreToolUse gate (Read-only on the exact `/tmp/creative-author-<uuid>.jpg`). Dahlia writes the finished caption ‚Äî one headline + one primary text + one description ‚Äî against the shared [[../libraries/copy-rubric]] `renderRubricForPrompt()` text + the resolved `audience_temperature` (`resolveAudienceTemperature` ‚Äî cold when `angle.source='competitor'` OR `angle.acquisitionPower ‚â• 8`; warm otherwise; `hot` is reserved for future retention audiences) + the debranded competitor DNA (only on `angle.source='competitor'`) + self-scores against the same rubric. **Revise loop (self-heal, copy-author-self-heal 2026-07-17):** on parse failure / session error / `self_score.total < AUTHOR_SELF_SCORE_FLOOR = 6` / `hasColdOfferLeak` trip on a cold-audience emit / **never-fabricate firewall miss** (see 7c ‚Äî the firewall now runs INSIDE this loop), the worker re-invokes with a revise prompt (retry cap `MAX_COPY_AUTHOR_REVISE_ATTEMPTS`, raised **1 ‚Üí 4**). Each retry RESUMES Dahlia's SAME box session (`claude --resume`) with a SHORT revise turn ‚Äî the image+brief+rubric context is cached within the 1h prompt-cache TTL, so a retry costs a few hundred tokens, not a fresh context load; on a lost session (box restart ‚Üí `isMissingSessionError`) or an account-cap hop the loop FAILS SAFE by re-dispatching FRESH with the full prompt. One ad per session (the qc allowlist is per-image); cold-start volume fans out as N single-ad jobs. **Exhaustion:** `stockProduct` records a `director_activity` row with `directorFunction='growth'`, `actionKind='dahlia_copy_author_exhausted'`, `specSlug='dahlia-copy-author-box-session'`, and metadata `{ product_id, product_title, angle_source, angle_hook, audience_temperature, attempts, last_reason }` ‚Äî then HOLDS the campaign out of the bin (never falls back to `buildMetaCopyPack` ‚Äî a silent fallback would erase the audit trail the M1 graduation gate depends on). When the flag is unset / `deterministic` OR the dispatcher is missing (a test / a manual invocation with no runBoxLane), this step is SKIPPED and step 8 uses `buildMetaCopyPack` unchanged.

7c. **Three-layer never-fabricate firewall ([[../libraries/creative-agent]] + `src/lib/ads/never-fabricate.ts` ‚Äî dahlia-never-fabricate-copy-firewall Phases 1‚Äì3)** ‚Äî before step 8's bin insertion, every author-mode verdict must clear three checks that together make "never fabricate a claim" structural rather than aspirational. **Layer 1** (instructional) is the CLAIM-ONLY-WHAT'S-IN-THE-BRIEF section of the [[../../.claude/skills/dahlia-copy-author/SKILL.md|dahlia-copy-author]] skill ‚Äî the five claim classes (numbers ¬∑ first-person testimony ¬∑ ingredient names ¬∑ timeframes ¬∑ comparative claims) each named against the ONLY brief / ProductIntelligence source field a claim in that class may cite (`pi.ingredients` / `pi.ingredientResearch` / `pi.reviews.byClaim` / `brief.transformation` / `brief.leadProof` / `brief.supportingBenefits` / `brief.competitorDna`), so a fabricated claim never enters the copy because Dahlia can't cite a source for it. **Layer 2** (session artifact) is the REQUIRED `claim_trace: Array<{claim, source, source_ref}>` field on the AuthorModeCopy verdict ‚Äî `parseAuthorVerdict` rejects a missing / empty / mis-shaped `claim_trace` with the distinct `firewall_missing_claim_trace` reason, the revise loop consumes it, and the field itself raises the cost of fabrication for a probabilistic model. **Layer 3** (deterministic gate) is `verifyClaimTrace(claim_trace, brief, pi, reviewsByClaim)` in `src/lib/ads/never-fabricate.ts`, run as the LAST gate INSIDE `runCopyAuthorSession`'s revise loop (copy-author-self-heal, 2026-07-17 ‚Äî was a post-session check between `runCopyAuthorSession` and `insertReadyCreative`). `stockProduct` injects a per-attempt closure (`CopyAuthorSessionInputs.verifyClaimTrace`) that resolves `pi.reviews.byClaim(source_ref)` for every unique reviews-byClaim entry then runs the pure verifier ‚Äî **fact-grounded** (CEO 2026-07-17): a faithful paraphrase passes, but any NUMBER not traceable to the cited source is `fabricated_number` and blocks (a competitor's "500M cups" grounds against OUR corpus, never their slot). A miss becomes a revise reason and Dahlia's session RESUMES to re-ground the claim; only on exhaustion is the campaign HELD out of the bin, emitting a **DISTINCT** `director_activity` row: `actionKind='dahlia_copy_firewall_exhausted'` (keyed off the exhausted outcome's `firewallMisses`), `specSlug='dahlia-never-fabricate-copy-firewall'`, metadata `{ product_id, product_title, angle_source, angle_hook, audience_temperature, misses[] }`. The distinct kind (never conflated with `dahlia_copy_author_exhausted`) lets operators slice fabrication failures from self-score failures. The StockedCreative failure row's reason string carries `firewall_claim_miss: <src>:<reason>[, ...]` for the batch summary. When step 7b is SKIPPED (deterministic mode / no dispatcher / flag off), step 7c is also SKIPPED ‚Äî the deterministic `buildMetaCopyPack` produces byte-identical strings from the fully-backed brief and needs no runtime witness.

8. **Bin insertion ([[../libraries/creative-pack]] Phase 2‚Äì3)** ‚Äî `insertReadyCreative`: [[../tables/product_ad_angles]] (stamped with the 4√ó4 copy pack via `metadata.copy_pack`) ‚Üí [[../tables/ad_campaigns]] `status='ready'` ‚Üí THREE static [[../tables/ad_videos]] siblings (feed 4:5 canonical + stories/reels 9:16 + right-column 1:1), linked via `format_variant_of_id`. All three renders uploaded to the private `ad-tool` bucket ([[../libraries/ad-storage]]). Single `landing_url` = the **battle-tested Shopify PDP** `{shopify_primary_domain}/products/{handle}` (policy CEO 2026-07-10; storefront / advertorial-variant landers are a later winner-only experiment). **Author-mode fields (Phase 3):** when step 7b produced a verdict, the row also carries `ad_campaigns.audience_temperature` = Dahlia's echoed target + `ad_campaigns.author_self_score` (jsonb) = the full rubric score envelope; the pack is a single-variant broadcast (`authorCopyPack` fills 4 headlines + 4 primary texts with Dahlia's single authored copy so `planCreativePackInserts` + `isCreativePackComplete` are unchanged). Deterministic-mode rows leave both NULL (null-means-deterministic-mode). **Completeness gate (Phase 3):** [[../libraries/creative-pack]] `isCreativePackComplete` re-checks at publish time (when Bianca calls [[../libraries/media-buyer-agent]] `publishToMeta`) ‚Äî only packs with all 3 placements + ‚â•4 headlines + ‚â•4 primary texts are allowed to Meta. **LF8 gate satisfied by construction (2026-07-14):** the caption (headline + primaryText) is guaranteed to carry a [[../libraries/lf8|Life-Force-8]] term whenever the brief carries an LF8-adjacent supporting benefit, so Bianca's test loop never surfaces zero-LF8 copy to the ads-supervisor (the [[../libraries/ads-supervisor]] `live_ad_lf8_thin` gate gates on the same [[../libraries/lf8]] keywords).

9. **Handoff** ‚Äî `listReadyToTest` now surfaces the campaign to Bianca's [[../libraries/media-buyer-agent]] test loop.

## North star
Dahlia optimizes a bounded proxy ‚Äî *bin depth √ó brief quality* ‚Äî and feeds Bianca; **Max owns the objective** (does the creative actually win in-market) and holds her leash. She never launches an ad herself; she stocks the bin, Bianca tests, the media-buyer grader scores realized ROAS. She is audited by the [[../libraries/ads-supervisor]] standing every-3h pass ([[../inngest/ads-supervisor-cadence]]), which checks for drift (bin depth below floor, zero seeded angles) and authors fix-specs. See [[../operational-rules.md]] ¬ß North star.
