# Lifecycle: Roadmap Build Console (self-driving roadmap)

Describe a feature тЖТ spec тЖТ autonomous build on the **Max subscription** тЖТ answer questions / approve prod actions тЖТ merge тАФ all from the dashboard (phone-friendly), no laptop or terminal. This is the end-to-end home for the shipped [[../specs/roadmap-build-console]] + [[../specs/build-approval-gates]] specs.

## Two layers (don't conflate)

- **Brain markdown (`main`)** тАФ canonical, *static* spec definitions + final status (the `тП│ planned ┬╖ ЁЯЪз in progress ┬╖ тЬЕ shipped ┬╖ тЭМ cut` phase emojis). Changes only when a PR merges. The board parses it.
- **`agent_jobs` (DB companion)** тАФ *live, actionable* state: build status, `needs_input` questions, `needs_approval` pending actions. The box worker polls it; the dashboard renders it as buttons. Pending actions live here (not on the unmerged branch) тАФ that's why a DB companion is required.

## Two runtimes

- **Vercel app** тАФ the dashboard surfaces + APIs. Never touches the box. A "Build" tap just **inserts an `agent_jobs` row** in Supabase.
- **The box** ([[../recipes/build-box-setup]]) тАФ a Hetzner CCX33, **Tailscale-only inbound**, runs `systemd: shopcx-builder` as the **non-root `builder`** user. It **polls Supabase outbound** (no inbound; the firewall stays shut тАФ the box dials out, the app never dials in), claims a job, and runs the build as a headless `claude -p` on **Max**.

## End-to-end trace

1. **Author.** `/dashboard/roadmap` тЖТ **тЬи New feature** opens the authoring chat (`POST /api/roadmap/chat`), now **box-hosted on Max** (box-spec-chat тАФ no longer the Anthropic API). Each turn enqueues a `kind='spec-chat'` [[../tables/agent_jobs]] row that resumes a long-running `claude -p` session on the build box, so the model has the **whole working tree** (`Read`/`Grep`/`Glob` over `docs/brain/` + `src/`) and **`WebSearch`** every turn тАФ far more grounded than the old brain-index + two-GitHub-tools call, and **$0 marginal**. The route appends the user turn + sets `turn_status='thinking'`; the box appends the reply; `AuthoringChat.tsx` polls `chat-session?id=` and shows "thinking on the boxтАж (a minute)". Talk it through тЖТ **Save spec** (the box resumes the session, emits the full spec markdown, the worker commits `docs/brain/specs/{slug}.md` to `main` via the GitHub Contents API) or **Save & build** (also queues a `build` job). **Refine with Opus** on a spec's detail page passes `slug` тЖТ the box Reads the current spec from the working tree as grounding. (Hand-written specs work too.)
2. **Dispatch.** **Build** on a card тЖТ `POST /api/roadmap/build` тЖТ inserts `agent_jobs` (`queued`). One active build per spec **for a plain Build tap** (no instructions): a second tap coalesces into the live job (`alreadyActive`). Variants: per-phase **build** ([[../dashboard/roadmap]] PhaseList) and **Report issue** both queue a build scoped via `instructions` (no spec edit, spec stays тЬЕ) тАФ and because they carry **new, distinct instructions**, when a build is already live they do **not** coalesce-and-drop; they enqueue a **distinct follow-up `queued` row** (`queuedBehindActive`) that runs after the active build (per-spec serialization). The UI/Slack say "queued to run next" тАФ the issue text is never lost. See [[../specs/fix-report-issue-dropped]].
3. **Claim.** The worker calls `claim_agent_job()` (atomic, `FOR UPDATE SKIP LOCKED`) тЖТ `building`. It runs `claude -p --dangerously-skip-permissions` (bypass, no prompts) via the **`build-spec` skill**, as `builder`, with prod-write secrets **stripped from the build env** and no `ANTHROPIC_API_KEY` (stays on Max).
4. **Outcomes** (the build emits one final-status JSON):
   - `completed` тЖТ worker runs `npx tsc --noEmit` gate, commits, opens a `claude/*` PR (`ensurePr` retries `gh pr create` with backoff first тАФ most failures are transient).
   - **PR-create failed but branch pushed** ([[../specs/build-recover-pr-create]]) тЖТ after the retries are exhausted the worker flags `needs_attention` `error="branch pushed but PR creation failed"`. This is **recoverable** (the build is done + on origin): the card shows **Create PR** (primary) тЖТ `POST /api/roadmap/build { jobId, recoverPr:true }` тЖТ `createPrForJob` opens/adopts the PR for the pushed branch and flips the job тЖТ `completed` тАФ *not* a Rebuild that throws the build away. (Re-validates the #185 case.)
   - `needs_input` (product question) тЖТ worker records `questions`, draft PR, pauses; the card shows an **answer form** тЖТ `POST /api/roadmap/answer` тЖТ `queued_resume` тЖТ worker `claude --resume`s.
   - `needs_approval` (gated prod action: `apply_migration` / `run_prod_script` / `merge_pr`) тЖТ worker records `pending_actions` + pauses; the card shows **Approve & apply** with the command preview тЖТ `POST /api/roadmap/approve` тЖТ worker (which holds prod creds) **executes the approved action**, then `--resume`s. The build itself never touches prod (credential-enforced).
   - `failed` / `needs_attention` тЖТ surfaced with `error` + `log_tail`. A build that finishes with **no file changes and no PR** is flipped to `needs_attention` carrying the agent's `no_changes_reason` (the build-spec contract emits it when it makes no edits) тАФ never a bare `completed` with no PR that masquerades as done (the card shows Rebuild as a retry). See [[../specs/fix-report-issue-dropped]] Phase 3.
5. **Review + merge.** The card's **Squash & merge** (or [[../dashboard/branches]]) merges the `claude/*` PR тАФ owner-only, server-revalidated.
6. **Status reflects reality.** Spec emojis drive the board columns; `agent_jobs` drives the live per-card build chip + buttons.
7. **Verify тЖТ fold тЖТ archive.** **Shipped (тЬЕ)** = built + deployed (automated). The owner-only, human gate **Verified** comes after: once Dylan confirms a shipped feature works in prod, **Mark verified & archive** (`BuildButton`) queues a **fold-build** (`POST /api/roadmap/build` `{ verify: true }` тЖТ canonical fold instructions in the build route, not a spec rebuild). The build folds the spec into its permanent brain homes, appends a one-line entry to [[../archive]], `git rm`s `specs/{slug}.md`, and opens a PR. Merge тЖТ the spec leaves the board into the collapsed **Archived** section (reads `archive.md`). The Shipped column is relabeled **"Shipped тАФ awaiting verification"** so it stays a short, real to-do list. Convention: `shipped тЖТ verified тЖТ fold + delete + archive-index` ([[../project-management]]). Nothing is lost тАФ knowledge in the brain, pointer in the archive, spec `git show`-recoverable.
8. **Re-hydrate (New spec from brain).** Revisiting an archived (or any reference) feature doesn't reactivate the stale spec: **New spec from brain** seeds the authoring chat with the *current* brain page (`POST /api/roadmap/chat` `{ seedSlug }`) and Opus drafts a *fresh* spec to extend/fix it тЖТ normal Build flow. The authoring chat in reverse.

## Safety model

Bypass is safe because of four things, not prompts: (1) **PR-gate** тАФ code never hits prod until you merge; (2) **powerless build env** тАФ secrets stripped, so a runaway command can't reach prod (credential-enforced); (3) **gated prod actions need a tap** тАФ `apply_migration`/`run_prod_script`/`merge_pr` only run after owner approval, executed by the worker; (4) **non-root** builder user. The worker is the *only* component with prod creds.

## Billing

- **Authoring chat** тЖТ Anthropic API (Opus `claude-opus-4-8`), cheap conversation tokens.
- **Builds** тЖТ **Max subscription** (box `claude -p`, no API key). Verified: a real build ran bypass-as-builder on Max and opened a PR (2026-06-18).

- Board + detail: `src/app/dashboard/roadmap/{page,[slug]/page}.tsx`; parser `src/lib/brain-roadmap.ts` (`getRoadmap` + `getArchive` for the Archived section).
- Components: `BuildButton.tsx` (build ┬╖ status ┬╖ answer ┬╖ approve ┬╖ squash-merge ┬╖ report-issue ┬╖ **mark verified & archive**), `StatusControl.tsx`, `PhaseList.tsx` (per-phase status + cut + build), `AuthoringChat.tsx` (new + refine + **seed / re-hydrate** + **persist / resume** + **box polling** тАФ each turn enqueues a `spec-chat` job and polls [[../tables/roadmap_chats]] `turn_status` while the box thinks; the thread survives closing the modal + resumes cross-device).
- APIs: `src/app/api/roadmap/{build,status,answer,approve,chat,chat-session}/route.ts` (build accepts `{ verify: true }` тЖТ `enqueue_fold` coalesces into one batch fold-build; **chat now ENQUEUES a `spec-chat` box job** тАФ turn / finalize / `{ action:'generate_verification' }` (verify) / `{ seedSlug }` re-hydrate тАФ it no longer calls Anthropic; chat-session is the poll + resume read); merge reuses `/api/branches/[number]/merge`. Archive index: `docs/brain/archive.md` is **generated** from `docs/brain/archive.d/*.md` by `scripts/brain-index.mjs` (`npm run brain:index`).
- Queue: [[../tables/agent_jobs]] + [[../tables/pending_folds]] + `src/lib/agent-jobs.ts` (`getLatestJobsBySlug` ┬╖ `getPendingFolds`) + `claim_agent_job(p_kinds)` / `enqueue_fold()`.
- Worker: `scripts/builder-worker.ts` (box) тАФ `runJob` / `runPlanJob` / `runFoldJob` / `runProductSeedJob` / `runSpecChatJob`, per-kind lanes (5 build + 1 fold + 2 seed + 1 spec-chat). Box runbook: [[../recipes/build-box-setup]].
