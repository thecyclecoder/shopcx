# Lifecycle: CEO Control Tower — universal kill-switches + org-mirror monitoring

The end-to-end home for the **CEO Control Tower goal** (M1–M6, all shipped) — one CEO-only on/off system spanning the whole AI org (**department → director → agent → tool**) fused with the Control Tower into a single monitoring-**plus**-control surface. Turning any node off verifiably STOPS its enqueue/claim/execute within one monitor tick; every node has an **owner + switch + heartbeat**; zero orphan (unowned) tools. This is the concrete realization of the [[../operational-rules#north-star|north star — supervisable autonomy]]: a tool optimizes a bounded proxy, a role agent owns the objective and supervises the tool, and the CEO owns company objectives — so every autonomous node must be **placeable** (owned), **switchable** (stoppable), and **visible** (beating). Anything that can't be placed, owned, or switched is itself a first-class alert.

Owned by **Ada** ([[../functions/platform]]); the switch is **CEO-only**. Replaces ~7 scattered ad-hoc kill-switches with one cascade-aware primitive enforced at the real execution chokepoints (`claim_agent_job`, cron deciders, tool executors), and turns the read-only kind-named Control Tower into a persona-first, egress-conscious three-tier org mirror.

## The two halves, unified

- **Control** — a CEO flip on any node writes ONE [[../tables/kill_switches]] row; the [[../libraries/kill-switch-resolver|cascade resolver]] walks the [[../libraries/control-tower-node-registry|canonical node registry]] parent→parent so a department-off row stops every descendant. Enforced at the chokepoints (below).
- **Monitoring** — every node beats a [[../tables/loop_heartbeats]] row; the [[../inngest/control-tower-monitor|monitor cron]] rolls the beats up per org function into the persona-first three-tier [[../dashboard/control-tower]] (department health → per-department agents/tools + switches → infra errors), and the [[../libraries/control-tower-self-audit|self-audit]] catches any node missing an owner, a switch, or a heartbeat.

Both halves read the SAME sources of truth — the [[../libraries/control-tower-node-registry|node registry]], [[../tables/kill_switches]], and the [[../libraries/control-tower|heartbeat helpers]] — so a node's owner, on/off state, and liveness cannot diverge across surfaces (approval routing, grader owner-scoping, and the org-chart roster all read the one registry).

## End-to-end trace

1. **Place (M1 — canonical node registry).** Every worker, tool, agent-kind, cron, reactive fn, and inline agent resolves to exactly one `OwnerFunction` through [[../libraries/control-tower-node-registry|node-registry.ts]] — the fusion of `MONITORED_LOOPS` + `personas.KIND_PERSONA_ALIAS` + the `builder-worker.ts` job-kind union, superseding three sources that used to disagree (and the `KIND_TO_FUNCTION` shim that hand-patched them). `resolveNodeOwner(kind)` is the primary lookup; `resolveNodeOwnerOrOrphanDefault` preserves the historical `ORPHAN_OWNER='platform'` fallthrough as an audit hook (bumps `getOrphanSightings()` + `console.warn`s, never silently defaults). `scripts/_check-node-registry-drift.ts` (wired into `predeploy`) fails CI on any missed piece.
2. **Switch (M2 — kill-switch primitive + cascade resolver).** [[../tables/kill_switches]] is the universal on/off primitive — one row per node the CEO has explicitly turned OFF; **missing row ⇒ ON** (fail-open, mirrors [[../tables/function_autonomy]]'s "missing row ⇒ off" fail-safe on the opposite polarity). [[../libraries/kill-switch-resolver|resolveEffectiveSwitch(nodeId)]] loads the rows once per snapshot and walks the registry parent→parent up to the department seat: **cascades down, never up**; **nearest-off wins**; sibling isolation. Ad-hoc flags (media-buyer arming, `is_active`, `ENABLED_ADAPTERS`) were migrated to consult the resolver.
3. **Enforce (M3 — the execution chokepoints).** Turning a node off must verifiably halt it within one tick, so the switch is checked where work actually starts:
   - **`public.claim_agent_job` RPC** joins the kill-switch cascade in SQL — the DB-side mirror of the registry, [[../tables/node_ancestry]], kept fresh by [[../libraries/node-ancestry-sync|syncNodeAncestry]] (box-worker startup + the nightly [[../inngest/node-ancestry-sync-cron]] backstop). A department-off row makes every box lane under it un-claimable centrally, with **zero app-code changes**. Fail-open: a missing/empty mirror degrades the `not exists` guard to true (claim proceeds).
   - **Cron deciders + tool executors** call `resolveEffectiveSwitch` before enqueue/execute; an OFF result short-circuits and records the attribution.
4. **See + switch (M4 — persona-first three-tier Control Tower).** The [[../dashboard/control-tower]] leads with one **department health rollup** per org function (worst-of color across its loops), each a collapsible drill-in listing its agents/tools/crons with an inline **CEO switch control** (the `POST /api/developer/control-tower/switch` writer — CEO-gated, calls `invalidateKillSwitchCache()`, logs a [[../tables/director_activity]] `kill_switch_toggle`). **L0 loads a handful of department rows, not the whole org.** The **monitoring-cost guardrail** (CEO 2026-07-11): the monitor tick is a **5-minute floor**, and each node's liveness window scales with its cadence — codified as the build-time invariant `MONITOR_TICK_FLOOR_MS` + `REGISTRY_LIVENESS_JITTER_GRACE` (1.2×) in [[../libraries/control-tower|registry.ts]] `assertRegistryInvariants` (throws a line-numbered error at import on any sub-5-min cron or too-tight window; daily ⇒ 30h, weekly ⇒ 9d, monthly ⇒ 37d).
5. **Trace down (M5 — infra sub-page + orphan self-audit).** Each department drill-in carries a **Loops | Infra** tab; [[../libraries/control-tower-infra-tab|buildInfraTabPayload]] ancestry-filters the global error-feed + DB-Health panel to that one `OwnerFunction` (via `resolveNodeOwner` / `routeOwnerFromPath`), so a red rollup on `retention` traces to the exact `error_events` row without leaving the CT. The **orphan-node self-audit** (`buildCoverageAudit` in [[../libraries/control-tower-self-audit]] + the `orphan-node-self-audit` sweep) reds any node missing an owner OR a switch OR a heartbeat — "a tool with no owner" is impossible to ship silently.
6. **Enforce forever (M6 — the north-star node-completeness rule).** Mirroring "code without a brain page is incomplete": **a node without a switch + heartbeat + owner is incomplete.** Every new agent-kind, cron, reactive fn, inline agent, or tool executor ships in the same PR with the trio — an OWNER in the [[../libraries/control-tower-node-registry|registry]], a [[../tables/kill_switches]] ancestry (its own row or an ancestor's), and an end-of-run heartbeat via one of the `emit*Heartbeat` helpers. The [[../libraries/control-tower-self-audit|self-audit]] is the runtime enforcer; codified in [[../operational-rules#node-completeness--a-node-without-a-switch--heartbeat--owner-is-incomplete-hard-rule|operational-rules § Node completeness]] and [[../../../CLAUDE.md]] § Local conventions.
