# PDP edge-served variant 'use cache' — unblock the build (round 2) by stripping the cacheComponents-incompatible `revalidate` + `runtime = "edge"` knobs the prior fix preserved

**Owner:** [[../functions/platform]] · **Parent:** extends [[../specs/regression-agent]] · **Regression-of:** [[pdp-edge-served-variant-use-cache-unblock-build-strip-route-]]
**Regression-signature:** `regression:pdp-edge-served-variant-use-cache-unblock-build-strip-route-:5058e8d93f37`

PR #613 stripped every `export const dynamic = "force-dynamic"` but, by the prior spec's explicit instruction, kept `export const revalidate = 3600` on the storefront pages and `runtime = "edge"` + `revalidate = 31536000` on the storefront-img route handler. Next 16's cacheComponents migration guide is explicit those are ALSO incompatible (replace `revalidate` with `'use cache'` + `cacheLife`; `runtime = 'edge'` is unsupported, cacheComponents requires Node.js). Every post-fix Vercel production deploy is in Error at the same files for `revalidate`, so the `renderEdgeAssignedPdp` 'use cache' helper never reaches prod and prod still serves `MISS` + `private,no-cache,no-store`. Strip those knobs — the PDP's existing 'use cache' + cacheLife helper preserves its caching, the storefront-img route already drives the edge via its CDN-Cache-Control header, and blog/links pages move their cache window into a 'use cache' + cacheLife block — and the post-fix deploy lands READY, restoring per-arm + bare PDP HITs.

## What regressed
Per-arm `x-vercel-cache: HIT` after warm-up on served-arm PDP URLs AND the bare PDP's SSG/`'use cache'`-cached render — both still serve `MISS` + `cache-control: private, no-cache, no-store, max-age=0, must-revalidate` in prod (confirmed live on https://shop.superfoodscompany.com/ashwavana-zen-relax on two consecutive curls) because every post-fix Vercel production deploy errors with 'Route segment config "revalidate" is not compatible with `nextConfig.cacheComponents`. Please remove it.' and the renderEdgeAssignedPdp 'use cache' helper never ships.

Per node_modules/next/dist/docs/01-app/02-guides/migrating-to-cache-components.md `revalidate` and `runtime = 'edge'` are ALSO incompatible with cacheComponents.
