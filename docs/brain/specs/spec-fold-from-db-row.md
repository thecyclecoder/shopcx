# Fold writes the brain from a DB row + flips the row to `folded`

**Owner:** [[../functions/platform]] ¬∑ **Parent:** [[../goals/db-driven-specs]] M4 ‚Äî Fold writes the brain + preserves the row
**Blocked-by:** [[spec-readers-from-db-retire-parser]]

The endgame for the spec arc. Once nothing READS spec markdown ([[spec-readers-from-db-retire-parser]] retired the parser) and nothing WRITES it from the author path ([[spec-authoring-writes-db-and-worker-materialize]] moved authoring to row writes), only ONE thing should ever touch `docs/brain/specs/*.md` again: **fold**, and only on a SHIPPED spec, and only to extract its permanent knowledge into the brain pages (lifecycle / library / table / inngest / integration / dashboard / recipe ‚Äî per [[../skills/fold-to-brain]]). After fold, the `public.specs` row is FLIPPED to `status='folded'` and PRESERVED ‚Äî not deleted (audit + the board's archive view). The `docs/brain/specs/{slug}.md` file gets `git rm`'d (the brain's lifecycle/library/etc. pages now carry the knowledge; the archive index records what was folded ‚Äî same as today's fold). Retire the last of the spec-status mirror: [[../tables/spec_card_state]] becomes redundant once `specs.status` carries all the live state, and the `phase_states[i]` slot is gone post-[[spec-readers-from-db-retire-parser]]. Closes the gap: today [[../skills/fold-to-brain]] reads the `.md` files + appends to `archive.md` and `git rm`s the spec file ‚Äî but the spec body lived on disk, not in the DB; in the post-[[spec-readers-from-db-retire-parser]] world the body lives in the row, so fold must READ the row. [[director-judgment-lanes-fold-author-dismiss]] currently operates on disk markdown; its fold lane reads the row instead.

## Phase 1 ‚Äî fold reads the shipped `specs` row, writes the brain pages, flips to `folded`
- The [[../skills/fold-to-brain]] skill's entrypoint (today: read `docs/brain/specs/{slug}.md`) changes to read `public.specs` + `public.spec_phases` for the slug via [[../libraries/specs-table]] `getSpec(workspaceId, slug)` (from [[spec-body-table-and-backfill]]). Guard: refuse to fold a spec whose `specs.status` is anything other than `shipped` (the DB-trigger rollup from [[spec-body-table-and-backfill]] ensures `shipped` ‚â° all `spec_phases` rows are `shipped` ‚Äî the structural rail this whole goal protects).
- Render the spec row as the in-memory text the fold-agent reads (the same H1/Owner/Parent/phases/verification shape, derived from the row + its `spec_phases` children). NO `.md` file is touched in this step ‚Äî the fold-agent operates on the in-memory text.
- The fold-agent's existing job (extract knowledge ‚Üí write/update [[../tables/]], [[../libraries/]], [[../inngest/]], [[../integrations/]], [[../recipes/]], [[../dashboard/]], [[../lifecycles/]] pages + the lifecycle Status / open work block, cross-link with `[[wikilinks]]`, append the archive index, update brain README counts) stays the SAME ‚Äî it operates on the in-memory text, then commits the brain edits to `main`.
- After the brain commits succeed, the worker flips the spec row: `UPDATE public.specs SET status='folded', updated_at=now() WHERE id=$1`. The row is PRESERVED ‚Äî every column (title, summary, owner, parent, blocked_by, milestone_id) stays so the board's archive view + audit history can render the folded spec card unchanged.
- The `docs/brain/specs/{slug}.md` file is removed: `git rm docs/brain/specs/{slug}.md` + commit. This is the ONLY path that ever touches a spec markdown file post-[[spec-authoring-writes-db-and-worker-materialize]]. With the mirror commit retired in [[spec-readers-from-db-retire-parser]], pre-fold no `.md` exists for newly-authored specs ‚Äî the `git rm` is a no-op for them and only operates on legacy markdown carried over from [[spec-body-table-and-backfill]]'s backfill.
- Verification ‚Äî Phase 1: fold a shipped spec end-to-end; expect the brain pages updated as today (the [[../skills/fold-to-brain]] output is unchanged), `specs.status` flipped to `folded`, the row otherwise intact, and the legacy `.md` removed where it existed.

The `archive.d/` directory becomes a backup surface only (still committed when fold writes it, for the git-history-of-shipped-specs use case the [[../specs/spec-test-agent]] uses to list shipped-unverified specs).
- [[../tables/spec_card_state]] retires. With `specs.status` carrying the live status (set by the rollup trigger), `spec_phases.pr + merge_sha` carrying provenance ([[spec-status-phase-pr-provenance]] ‚Äî set post-[[spec-authoring-writes-db-and-worker-materialize]]), `specs.priority/deferred/intended_status` carrying the flags ([[spec-body-table-and-backfill]] columns), and `specs.last_merge_sha` (added here as a small additive migration from the existing `spec_card_state.last_merge_sha`) carrying the deploy-aware UI slot, [[../tables/spec_card_state]] holds nothing else worth keeping. Once the spec-test agent itself reads the DB for shipped-unverified, those traces can be dropped too ‚Äî out of scope for this spec.
- Verification ‚Äî Phase 2: the board's archive surface lists every folded spec from `specs` directly; no read of `archive.d/` for the archive listing. `grep -rn "spec_card_state" src/ docs/brain/` returns zero matches in code (a stub redirect in [[../libraries/spec-card-state]] is acceptable for a release cycle to catch external consumers, then is removed).

## Safety / invariants
- **Fold ONLY a shipped spec.** The guard is in the fold entrypoint: refuse if `specs.status !== 'shipped'`. The DB-trigger rollup ([[spec-body-table-and-backfill]]) guarantees `shipped` ‚â° every phase shipped ‚Äî so this is the same guarantee written twice (app + DB rail).
- **Preserve the row.** Never `DELETE FROM public.specs` in fold. The row is the audit + the board's archive card + the milestone's "what shipped" surface. Use `status='folded'` exclusively.
- **The brain pages are the source of truth for FOLDED knowledge.** The spec row records WHAT shipped + the per-phase PR provenance; the brain pages explain WHY/HOW. After fold, the lifecycle/library/etc. pages are the canonical home for the knowledge ‚Äî the row is the receipt.
- **`git rm` only the spec `.md`.** Fold MUST NOT remove the brain pages it just wrote. Verify the commit fold makes contains exactly one `D` line (the spec) and N `A`/`M` lines (the brain pages); flag a commit with multiple `D` lines as a regression.
- **Workspace-scoped + service-role writes.** `specs.status` flip and the `git rm` both go through `createAdminClient()` (mirrors the rest of the goal). RLS keeps direct-from-browser writes off.
- **Idempotent re-fold is a no-op.** Re-running fold on an already-`folded` spec is rejected (`specs.status='folded' ‚Üí already folded`). The brain pages are NOT re-written.

## Completion criteria
- The [[../skills/fold-to-brain]] skill reads from `public.specs` + `public.spec_phases`. A real shipped spec, folded end-to-end, produces the same brain edits the markdown-driven fold produced (compared on a snapshot pair), flips `specs.status` to `folded`, preserves the row, and removes the legacy `.md` where it exists. `npx tsc --noEmit` clean.
- The fold guard refuses to fold a non-`shipped` spec (manually set `specs.status='in_progress'` ‚Üí expect a clear "spec is not shipped" error; no brain edits made).
- The board's archive view reads `specs.status='folded'` rows directly; the [[../tables/spec_card_state]] table is dropped (or, if any external caller is still attached, parked behind a deprecation note for one release cycle and then dropped). `grep -rn "spec_card_state" src/` returns zero matches (or only a stub-redirect in [[../libraries/spec-card-state]]).
- [[../tables/spec_status_history]] continues recording transitions, including the `shipped‚Üífolded` transition (the fold writer logs `actor='fold-agent'`).
- The success metric from [[../goals/db-driven-specs]] is met: a `specs.status='shipped'` flip is structurally impossible while any `spec_phases` row is non-`shipped` (the [[spec-body-table-and-backfill]] trigger holds across this whole arc).
- Brain pages: UPDATE [[../skills/fold-to-brain]] SKILL.md (read the row, not the disk), UPDATE [[director-judgment-lanes-fold-author-dismiss]] (the director's fold lane reads the row), RETIRE [[../tables/spec_card_state]] + [[../libraries/spec-card-state]] (or mark deprecated with a wikilink to [[../libraries/specs-table]]), UPDATE `docs/brain/README.md` (the brain count changes when the `spec_card_state` page retires).
