# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.8/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently reports only a tally (✅0 ⚠1) without naming the spec reviewed, the specific defect category (mangled phases, missing owner/parent/blockers, missing Verification), or the markdown evidence that justified the flag.:** When you see your sweep produce any flag or pass verdict, do emit a per-spec diagnosis block that names (1) the spec ID/title, (2) the exact defect category from the rubric, (3) the specific field or section that failed, and (4) a one-sentence rationale quoting or citing the offending markdown — before closing the job. — The rubric requires that diagnoses match the markdown and stay scoped to Phase 3 QUALITY concerns; without field-level evidence in the log the director cannot distinguish a well-reasoned call from a lucky guess, which is why every sweep is capped at 6/10 regardless of correctness. Emitting structured, evidence-linked diagnosis notes makes the reasoning auditable and directly satisfies the 'diagnoses match the markdown' criterion that unlocks scores above 6.
- **When Vale consistently reports only a tally (e.g., '✅0 ⚠1') with no per-spec detail on which spec failed, which field/section was defective, or how the finding maps to a rubric category.:** When you see a completed sweep, output a labeled diagnosis entry for every spec reviewed — naming the spec, the exact defect category (mangled phases / missing owner/parent/blockers / missing Verification), the specific offending field or section, and a one-sentence rationale tying it to the markdown evidence — before logging the summary tally. — The grader cannot confirm a call is substantively correct without field-level evidence; a bare count is unverifiable and scores no higher than 6 regardless of whether the outcome was right. Providing explicit, spec-referenced diagnostic reasoning transforms an unauditable tally into a traceable verdict that the director can confirm matches the rubric without chasing secondary logs.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
