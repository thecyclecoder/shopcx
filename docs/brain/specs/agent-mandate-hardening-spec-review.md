# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.7/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently closes sweeps with only aggregate counters (✅0 ⚠1) and no per-spec, per-field diagnosis, making it impossible to verify that the rubric checklist was actually applied.:** When you issue any pass or needs_fix verdict, immediately follow it with an explicit per-spec diagnosis that names: (1) the exact spec file reviewed, (2) each rubric field checked (phases, owner, parent, blockers, Verification), and (3) for any needs_fix, the precise defect found (e.g., 'missing Verification section', 'owner field blank') tied to the actual markdown — never emit a bare counter as the sole output. — Graders cannot distinguish a correct rubric-driven judgment from a vacuous flag without field-level evidence; surfacing the exact defect name and location proves the checklist was applied and allows the diagnosis to be audited against the spec markdown, which is the core of what a 10 requires.
- **When Vale consistently emits only aggregate counters (✅0 ⚠1) with no per-spec, per-field diagnosis, making it impossible to verify the rubric was applied.:** When you produce a pass or needs_fix verdict on any spec, immediately follow it with an explicit per-field callout — naming the exact spec, the exact defect (e.g., 'missing Verification section', 'owner field blank', 'parent not set'), and the markdown evidence — rather than logging a bare counter; if a spec is skipped, state the reason inline. — Graders cannot award marks for correct judgment they cannot see: a bare '⚠1' is indistinguishable from a random flag, so every run without field-level evidence is capped well below 10. Surfacing the exact field and evidence also forces Vale to actually walk the checklist, which is the mechanism that catches real defects and avoids false-fix calls.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
