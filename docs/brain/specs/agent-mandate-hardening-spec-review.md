# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.7/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale reports a bare status tally (e.g., ✅0 ⚠1) with no named spec, no identified defect category, and no field-level evidence tying the verdict to the markdown.:** When you see yourself about to emit only a count or status symbol as your finding, instead write a structured per-spec entry that names: (1) the spec reviewed, (2) the exact rubric category violated (mangled phases / missing owner·parent·blockers / missing Verification), (3) the specific field or section in the markdown that triggered the call, and (4) a one-sentence rationale confirming you stayed in Phase 3 quality scope and did not rule on planned/deferred status. — Every low grade cites the same root cause: the log is too sparse to verify whether Vale's judgment was sound or whether it missed defects or made false-fix calls. Providing field-level, traceable evidence for every verdict makes the diagnosis auditable, proves rubric adherence, and eliminates the ambiguity that caps scores at 6.
- **When Vale emits only a bare status tally (e.g., '1/1 reviewed, ⚠1') with no named spec, no identified defect category, and no reference to the specific markdown field that failed.:** When you produce a needs_fix or pass verdict, log: (1) the exact spec name, (2) the rubric category violated or confirmed clean (mangled phases / missing owner|parent|blockers / missing Verification), and (3) the specific field or section in the markdown that triggered or cleared the check — before finalising the sweep output. — Every low grade traces back to the same gap: reviewers cannot distinguish a genuine defect catch from a false-fix or a missed defect without field-level evidence. Anchoring each verdict to a named spec and a cited markdown field makes the diagnostic auditable, proves Phase 3 lane discipline, and is the only way to score above 6.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
