# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.5/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently reports only a pass/fail tally (e.g., '✅0 ⚠2') without naming the specific spec, the exact field that failed, or the defect category, making it impossible for the grader to verify the diagnosis was correct.:** When you see a spec reviewed and a pass or needs_fix verdict reached, log a one-line diagnosis per spec naming: (1) the spec identifier, (2) the exact defect class (mangled phase, missing owner/parent/blockers, missing Verification, or none), and (3) the specific field or section in the markdown that triggered the verdict — before moving to the next spec. — The rubric rewards confirmed defect-catching, not just a correct tally; without traceable per-spec reasoning, graders cannot distinguish a sound diagnosis from a false-fix call or a missed defect. Explicit field-level evidence lets the grader validate correctness directly from the log, which is the only path from a 6 to a 10.
- **When Vale logs only a summary counter (e.g., '✅0 ⚠1') with no per-spec defect detail, making it impossible for the grader to verify whether the diagnosis was accurate, well-scoped, or matched the markdown.:** When you emit a needs_fix or pass verdict, always log a named per-spec diagnosis that includes: (1) the spec identifier, (2) the exact defect category caught (e.g., 'missing Verification section', 'owner field absent', 'mangled phase label'), and (3) explicit confirmation that planned/deferred decisions were left to Ada — never just a tally symbol. — The rubric awards full marks only when the grader can independently confirm the diagnosis matches the markdown without reopening the spec; a bare counter provides zero traceability. Embedding a precise, named defect description in the log tail makes every needs_fix call self-evidencing and rules out both false-fix calls and missed defects in a single step.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.

Owner: [[../functions/platform]] (Ada supervises Vale). The director authored this from Vale's coaching ledger; building it hardens the agent so the coaching never has to repeat.
