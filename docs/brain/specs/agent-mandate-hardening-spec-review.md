# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.0/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently reports only a summary count (e.g., '1/1 reviewed, ⚠1') with no per-spec breakdown naming the exact defect category or offending field, making every verdict unauditable.:** When you record a pass or needs_fix verdict for any spec, immediately follow it with a one-line field-level diagnosis — e.g., 'needs_fix: missing Verification section' or 'needs_fix: owner field absent, parent unset' — so the ruling is traceable to the markdown without re-reading the spec; never emit a bare count as your final output. — Graders cannot distinguish a genuine catch from a false-fix call, or confirm Phase 3 lane discipline, when the log contains only a tally. Attaching the specific defect category and field name to every verdict turns an opaque status count into an auditable, rubric-grounded diagnosis, which is exactly what the scoring criteria require.
- **When Vale consistently reports only a summary count (e.g., '1/1 reviewed, ⚠1') without naming the specific defect category, the offending field/section, or the spec it applies to, making every review verdict unauditable.:** When you complete a spec review, do not emit a bare status count — instead, write a per-spec line that names the spec, the exact defect category (e.g., 'missing Verification section', 'no owner field', 'Phase listed as Planned — out of scope for Phase 3 quality check'), and the offending field or section verbatim from the markdown; for passing specs, state which required fields were present and why no defect was found. — Graders cannot distinguish a sound diagnosis from a lucky guess when the log contains only counts; a field-level, spec-named finding is the only evidence that Vale's call matches the actual markdown and stays within the Phase 3 QUALITY scope. Providing that detail closes the auditability gap that has capped every recent score at 6/10.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
