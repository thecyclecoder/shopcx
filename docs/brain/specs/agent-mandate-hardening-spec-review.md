# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.3/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only a bare status count (e.g., '1/1 reviewed, ⚠1') with no named spec, no identified defect category, and no field-level evidence tying the verdict to the markdown.:** When you issue any pass or needs_fix verdict, immediately log: (1) the spec name, (2) the exact defect category from the rubric (mangled phases / missing owner-parent-blockers / missing Verification), (3) the specific field or section in the markdown that triggered it, and (4) one quoted or paraphrased line from the spec confirming the diagnosis — before closing the sweep. — Graders cannot distinguish a correct diagnosis from a lucky guess or a false-fix call without field-level traceability; every low grade cited the same gap — a bare tally proves mechanics ran but not that judgment was sound. Logging the named spec, rubric category, offending field, and markdown evidence makes every verdict independently auditable and directly satisfies the rubric's requirement to confirm diagnoses match the markdown.
- **When Vale consistently logs only a bare status tally (e.g., '1/1 reviewed, ⚠1') with no named spec, no identified defect category, and no field-level evidence tying the verdict to the markdown.:** When you issue any pass or needs_fix verdict, immediately follow it with: (1) the spec name, (2) the exact defect category from the rubric (mangled phases / missing owner|parent|blockers / missing Verification), (3) the specific field or section in the markdown that triggered the call — and if a spec is skipped, log the explicit reason (e.g., 'deferred by Ada, out of Phase 3 scope') before moving on. — Graders cannot distinguish a genuine diagnostic hit from a lucky guess or a false-fix call without field-level traceability; attaching named evidence to every verdict makes the judgment auditable and proves Phase 3 lane discipline. Unexplained skips create the same opacity problem — they look like silent bypasses of potentially defective specs, which directly undermines the rubric's thoroughness requirement.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
