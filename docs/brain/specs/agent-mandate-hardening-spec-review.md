# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.6/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only a summary tally (e.g., '✅0 ⚠1') with no named spec, no defect category, and no field-level evidence, making every finding unverifiable.:** When you flag a spec as needs_fix (or pass), always append a structured diagnosis line to the log: '<SpecName> → <defect category: mangled-phase | missing-owner | missing-parent | missing-blockers | missing-Verification> → <the exact field or section that is absent/malformed and the markdown evidence>' — never let a tally stand alone. — Graders cannot distinguish a substantive, rubric-aligned call from a lucky guess without field-level evidence; adding the named spec, defect category, and markdown reference gives the Director the minimum context needed to confirm the call was correctly scoped to Phase 3 quality criteria and not drifting into planned/deferred territory.
- **When Vale logs only a tally (e.g., '✅0 ⚠1') with no named spec, no defect category, and no field-level evidence, making every finding unverifiable.:** When you complete a sweep and assign any disposition (pass or needs_fix), always append to your log: (1) the exact spec name, (2) the defect category from the rubric (mangled phase · missing owner/parent/blockers · missing Verification), (3) the specific field or section that is defective, and (4) a one-sentence rationale quoting or referencing the markdown evidence — never let a tally stand alone. — The grader cannot distinguish a substantive diagnosis from a lucky guess when only a count is emitted; a 10 requires enough trace for the Director to confirm rubric alignment without re-reading the source spec. Attaching named, field-level evidence to every finding closes that verification gap and proves the call was intentional and correctly scoped to Phase 3 quality criteria.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
