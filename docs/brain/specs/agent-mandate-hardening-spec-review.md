# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.4/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only summary counts (e.g., '✅0 ⚠1') with no per-spec defect descriptions, making it impossible to verify that diagnoses match the markdown or that the rubric was correctly applied.:** When you see yourself about to conclude a spec-review job, do instead log a named finding for every spec reviewed — format: spec name → verdict → exact defect (e.g., 'missing Verification section', 'no owner field', 'Phase 2 and Phase 3 steps merged') → which rubric dimension it violates — and explicitly note that no planned/deferred scope decisions were made; never emit only a count summary. — Graders cannot distinguish a sharp diagnostic call from a lucky guess when output is opaque; detailed per-spec findings tied to concrete markdown evidence are the only way to confirm diagnosis quality and rule out false-fix calls, which is exactly what the rubric rewards at 10/10.
- **When Vale consistently logs only summary counts (e.g., '1 needs_fix') without naming the exact defect, affected field, or spec section, leaving every finding unverifiable.:** When you emit any needs_fix or pass verdict, always include in your log output: the spec identifier, the exact defect found (e.g., 'missing Verification section', 'no owner field', 'mangled Phase 3 label'), and a one-line quote or reference to the markdown evidence — never log a verdict without these three elements. — The rubric requires that diagnoses match the markdown; without naming the specific field or section that is defective, neither a true catch nor a false-fix call can be confirmed by the Director. Explicit per-finding detail makes every verdict auditable and directly satisfies the 'caught real spec defects' and 'diagnoses match the markdown' criteria.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
