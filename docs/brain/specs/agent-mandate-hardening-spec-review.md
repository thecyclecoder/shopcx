# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.6/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only a tally (e.g., '✅0 ⚠1') without naming the spec, the defect category, or the markdown evidence that justified each pass/needs_fix call.:** When you see yourself about to emit a summary count as your final log output, instead write one explicit line per spec reviewed: '<spec-name>: <pass|needs_fix> — <defect category: mangled phase / missing owner|parent|blockers / missing Verification / none> — <the specific field or section in the markdown that failed or confirmed sound>.' — The rubric awards points for verifiable diagnostic accuracy, not just correct outcomes — a 10 requires the Director to be able to audit that the right fields were checked and the call was correctly scoped to Phase 3 quality concerns without chasing secondary logs. Adding per-spec, field-level evidence in the primary log tail transforms an unverifiable tally into an auditable diagnosis.
- **When Vale consistently outputs only a tally (e.g., '✅0 ⚠1') with no per-spec reasoning, making it impossible to verify which field was defective, why it failed, or that the diagnosis maps to the actual markdown.:** When you log a spec-review result, always include for each spec reviewed: (1) the spec name, (2) the specific defect category caught (mangled phase, missing owner/parent/blockers, missing Verification, or none), (3) a one-line rationale quoting or citing the exact field/section in the markdown that is absent or malformed, and (4) explicit confirmation it is a Phase 3 quality issue, not a planned/deferred judgment — even for passing specs, state why they are sound. — The rubric scores on verifiable diagnostic accuracy, not just the correct count of flags. Sparse logs make it structurally impossible for the Director to confirm a real defect was caught versus a false positive or false negative, so every sweep lands at 6/10 regardless of correctness. Field-level evidence tied to the markdown is the only way to demonstrate rubric adherence and earn full credit.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
