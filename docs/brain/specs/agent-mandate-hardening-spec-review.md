# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.2/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently outputs only a summary count (e.g., '✅0 ⚠1') without naming the spec, the defect category, or the markdown evidence that drove each pass/needs_fix call.:** When you write your review output, for every spec reviewed include: (1) the spec name/ID, (2) the verdict (pass or needs_fix), and (3) a one-line diagnosis citing the exact defect category (mangled phase, missing owner/parent/blockers, missing Verification) and the specific field or section in the markdown that failed — e.g., 'SPEC-42: needs_fix — Verification section absent; no acceptance criteria found after the Deliverables block.' If a spec is skipped, log the reason explicitly. — Every low grade shares the same root cause: the sparse log makes it impossible for the Director to confirm the diagnosis matches the markdown or that no false-fix calls were made without re-reading the spec themselves. Attaching a named, field-level rationale to each verdict turns the review log into an auditable trail, satisfying the rubric's 'diagnoses match the markdown' and 'caught real defects' criteria rather than forcing the grader to assume correctness from a bare count.
- **When Vale consistently reports only a summary count (e.g., '✅0 ⚠1') without documenting which spec failed, which rubric field was defective, or the markdown evidence behind the call.:** When you write the sweep output, for every spec reviewed include: (1) the spec name/ID, (2) the exact field or section that failed (e.g., 'missing Verification', 'no owner field', 'mangled phase label'), and (3) one sentence of markdown evidence tying the flag to what is literally absent or broken in the spec — only then append the tally line. — Graders cannot verify diagnostic accuracy or rule out false-fix calls from a bare count alone; per-spec findings with cited evidence make the judgment auditable without re-reading every spec. This directly satisfies the rubric criteria that diagnoses must match the markdown and that no false-fix calls were made on sound specs.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
