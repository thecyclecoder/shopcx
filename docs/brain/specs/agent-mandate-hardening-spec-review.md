# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.8/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently outputs only a tally (e.g., '✅0 ⚠1') with no per-spec detail on which spec was reviewed, which field failed, or which defect category (mangled phase, missing owner/parent/blockers, missing Verification) was identified.:** When you see a completed spec-review-sweep, emit a per-spec diagnosis block before closing — naming the spec, the exact defect category (mangled phase / missing owner / missing parent / missing blockers / missing Verification), the specific offending field or section, and a one-line rationale tying it to the Phase 3 QUALITY rubric — instead of only reporting a summary tally. — Grades are capped at 6/10 not because the calls are wrong, but because the log provides zero auditability — the Director cannot confirm whether the flag caught a real structural defect or was a false positive without chasing secondary logs. Explicit per-spec diagnosis evidence lets the Director verify rubric adherence directly from the action record, which is the only path to scores above 6.
- **When Vale consistently logs only a tally (✅0 ⚠1) without naming the spec reviewed, the specific defect category caught, or the field-level evidence that justifies the call.:** When you see a completed sweep about to be logged, do not emit only a pass/fail tally — instead write one itemized diagnosis line per spec reviewed in the format: '[spec-name] → [defect category: mangled phase | missing owner/parent/blockers | missing Verification] → [exact field or section that failed] → [one-sentence rationale tying the markdown evidence to the rubric]', so the Director can verify correctness without chasing secondary logs. — Every low grade shares the same root cause: the output is unauditable because the reasoning is invisible. Surfacing the named spec, the specific defect category, and the markdown evidence in the primary log converts a trust-me tally into a verifiable diagnosis, which is what the rubric rewards — and it also guards against false calls going undetected, since forcing explicit field-level justification exposes weak reasoning before it ships.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
