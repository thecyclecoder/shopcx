# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.8/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only a pass/fail count (e.g., '✅0 ⚠1') with no named spec, no identified defect category, and no field-level evidence, making every flag unverifiable.:** When you emit a needs_fix or pass verdict, always log: (1) the exact spec name/ID reviewed, (2) the specific defect category triggered (mangled phases / missing owner|parent|blockers / missing Verification), and (3) the precise field or section that is absent or malformed — e.g., 'spec SHOP-42: missing Verification section' or 'spec SHOP-17: owner field absent in header' — before recording the summary counter. — Every low grade traces to the same root: the grader cannot confirm the diagnosis matches the markdown without re-reading the spec themselves, so the flag is unauditable and trust in the rubric application collapses. Emitting named, field-level evidence per spec lets the grader verify correctness in seconds, satisfying the 'diagnoses match the markdown' rubric criterion and converting a 6 into a 10.
- **When Vale consistently emits only a summary count (e.g., '✅0 ⚠1') with no per-spec defect detail, making every diagnosis unverifiable regardless of whether the pass/fail call was correct.:** When you complete a spec review and assign any verdict (pass or needs_fix), always log: (1) the spec name, (2) the exact rubric dimension violated (mangled phases / missing owner|parent|blockers / missing Verification), (3) the specific field or section in the markdown that is absent or malformed, and (4) an explicit note that planned/deferred scope decisions were left to Ada — before closing the job. — The grader cannot confirm diagnostic quality from a bare counter alone; without a named spec and a precise defect description tied to the markdown, a correct outcome is indistinguishable from a lucky guess or a mis-scoped call. Adding structured per-spec reasoning lets the Director audit the judgment in seconds and directly satisfies the rubric criterion that 'diagnoses match the markdown.'

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.

Owner: [[../functions/platform]] (Ada supervises Vale). The director authored this from Vale's coaching ledger; building it hardens the agent so the coaching never has to repeat.
