# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.5/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently produces aggregate counters (✅0 ⚠1, ✅0 ⚠2) with no per-spec defect detail, making it impossible to verify whether the rubric (phases, owner/parent/blockers, Verification) was actually applied or just tallied.:** When you issue any pass or needs_fix verdict, immediately follow it with a per-spec breakdown naming (a) the spec reviewed, (b) each rubric field checked, and (c) the exact defect found or confirmation it was clean — never emit only a counter; if a spec is skipped, state the explicit reason (e.g., already deferred, wrong status) in the same log line. — Graders cannot distinguish a correct diagnosis from a lucky flag or a no-op sweep when only tallies are logged; explicit per-field callouts tied to the actual spec content are the only evidence that Vale reasoned through the checklist rather than cycling through files, which is precisely what the rubric rewards and what raises scores from 5-6 to 10.
- **When Vale consistently closes sweeps with only aggregate counters (✅0 ⚠1) and no per-spec defect detail, making it impossible for reviewers to verify whether the rubric was actually applied or the diagnosis is correct.:** When you issue any pass or needs_fix verdict, immediately follow it with an explicit per-spec defect callout — name the spec file, state which specific field failed (e.g., 'missing Verification section', 'no owner field', 'mangled phase label'), and quote or reference the exact markdown evidence; never emit only a tally counter as the sole output. — Graders cannot distinguish a well-reasoned verdict from a blind flag when the log shows only '⚠1' with no supporting detail — every low grade explicitly cites the inability to audit the diagnosis against the rubric. Surfacing the field-level evidence directly in the log is the only way to prove the checklist (phases, owner/parent/blockers, Verification) was genuinely traversed and that needs_fix calls are not false positives.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
