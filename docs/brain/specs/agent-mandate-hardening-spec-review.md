# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.5/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only a summary count (e.g., '✅0 ⚠1') with no per-spec detail on which spec failed, which field was defective, or why — making every sweep unauditable.:** When you complete a spec review sweep, for every spec reviewed output a named diagnosis entry citing the exact defect category (mangled phases, missing owner/parent/blockers, missing Verification) and the specific markdown evidence that drove the pass or needs_fix call; never emit only a tally. — The rubric requires that diagnoses match the markdown and that false-fix calls can be ruled out — both are impossible to verify from a bare count. Per-spec reasoning with field-level evidence is the only artifact that lets the supervisory chain confirm catch quality and Phase 3 scope compliance without re-reading every spec themselves.
- **When Vale consistently logs only a summary tally (e.g., '✅0 ⚠1') with no per-spec breakdown of which fields failed, what the defect category was, or how the diagnosis maps to the markdown.:** When you see yourself about to log a sweep result, do not emit just a count — instead write one labeled entry per spec reviewed that names the spec, states the verdict (pass/needs_fix), identifies the exact defect category (missing owner/parent/blockers, missing Verification, mangled phases) or confirms its absence, and quotes or references the specific markdown evidence, because every finding must be auditable without re-reading the source spec. — The rubric explicitly requires that 'diagnoses match the markdown' and that 'no false-fix calls on sound specs' can be confirmed — neither can be verified from a bare tally. Per-spec diagnostic detail is the only output that lets the supervisory chain validate rubric adherence without chasing secondary logs, which is what separates a 6 from a 10.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
