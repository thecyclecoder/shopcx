# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.4/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently emits only a summary counter (e.g., '✅0 ⚠1') with no per-spec defect detail, leaving the Director unable to verify whether the flag was a true catch or a false-fix call.:** When you conclude a spec-review sweep, do not emit only a count — for every spec reviewed, output a named diagnosis: spec identifier, the exact rubric field that failed (e.g., 'missing Verification section', 'no owner field', 'mangled phase label'), and an explicit confirmation that no planned/deferred scope decisions were made. — The rubric grades on verifiable diagnostic quality, not just correct counts; a grader who cannot trace the flag to a specific spec field has no basis to award more than a passing score. Naming the exact defect per spec makes the reasoning auditable without reopening the source markdown, which is the transparency a 10 requires.
- **When Vale consistently logs only summary counts (e.g., '✅0 ⚠1') with no per-spec defect detail, making it impossible to verify whether flags are true catches or false-fix calls.:** When you write your sweep output, for every spec reviewed emit at least one structured line naming: (1) the spec identifier, (2) the exact defect found (e.g., 'missing Verification section', 'no owner field', 'mangled phase label'), and (3) explicit confirmation that no planned/deferred scope decisions were made — or, if the spec passes, a one-line confirmation that all rubric fields were present and sound. — The rubric requires the director to confirm that diagnoses match the markdown without re-reading the spec; a bare count provides zero evidence of reasoning quality. Named, field-level defect descriptions make the diagnostic chain auditable and are the only way to distinguish a genuine catch from a careless false-fix call.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
