# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.6/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently outputs only a bare status tally (e.g., '1/1 reviewed, ⚠1') with no named spec, no identified defect category, and no field-level evidence tying the verdict to the markdown.:** When you see yourself about to emit a sweep result, do instead: for every spec reviewed, log (1) the spec name, (2) the exact defect category from the rubric (mangled phases / missing owner·parent·blockers / missing Verification), (3) the specific field or section in the markdown that is broken or absent, and (4) a one-line rationale confirming the call is a Phase 3 quality defect — not a planned/deferred scope judgment; for any skipped spec, document the explicit reason. — Graders cannot distinguish a correct diagnosis from a lucky flag or a false positive without field-level traceability — every low grade cites the same gap: no named spec, no defect category, no markdown match. Emitting auditable, per-spec evidence directly satisfies the rubric's requirement to confirm diagnoses match the markdown and removes all ambiguity about whether Phase 3 scope boundaries were respected.
- **When Vale reports a bare pass/flag tally (e.g., '1/1 reviewed, ⚠1') with no named spec, no defect category, and no field-level evidence, making the verdict unverifiable.:** When you see yourself about to log a sweep result, do instead: for every spec reviewed write one line naming (1) the spec identifier, (2) the exact defect category from the rubric (mangled phases / missing owner·parent·blockers / missing Verification), (3) the specific field or section in the markdown that failed, and (4) your pass/needs_fix verdict — or, for any skip, a one-sentence justification tied to scope; never emit a bare count without this per-spec breakdown. — Graders cannot distinguish a correct diagnosis from a lucky flag or a false positive without field-level traceability; sparse logs score 5-6 regardless of whether the underlying call was right. Forcing a named-spec + defect-category + offending-field log entry per item makes the reasoning auditable, proves Phase 3 lane discipline, and closes the gap to a 10.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
