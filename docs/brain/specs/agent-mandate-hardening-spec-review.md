# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.6/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently reports only a tally (e.g., '✅0 ⚠1') without naming the reviewed spec, the defect category (mangled phases, missing owner/parent/blockers, missing Verification), or the markdown evidence that justified the call.:** When you see any spec-review sweep complete, do NOT emit only a count summary — instead, output a per-spec diagnosis block that names: (1) the spec ID/title, (2) the exact defect category (mangled phases · missing owner/parent/blockers · missing Verification), (3) the specific field or section that failed, and (4) a one-sentence rationale tying the flag to the markdown evidence. — The rubric awards points for verifiable diagnostic quality, not just correct tallies; without field-level evidence the Director cannot confirm the call was substantive, correctly scoped to Phase 3 QUALITY, and not a lucky guess — so every sparse log caps the grade at ~6 regardless of outcome correctness.
- **When Vale reports a tally (e.g., '✅0 ⚠1') without naming the reviewed spec, the specific defect category, or the offending field/section in the markdown.:** When you log a needs_fix or pass disposition, immediately follow it with: (1) the spec name, (2) the exact defect category (mangled phase, missing owner/parent/blockers, or missing Verification), and (3) the specific field or section in the markdown that is defective — e.g., 'Spec: checkout-flow · Defect: missing Verification section · Evidence: no Verification heading found after Acceptance Criteria block.' — Every low grade shares the same root cause: the Director cannot verify whether Vale's call was substantively correct or accidental without field-level evidence tied to the markdown. Logging a named spec + defect category + offending field transforms the output from an unverifiable tally into an auditable diagnosis, directly satisfying the rubric's requirement to confirm real defects were caught and correctly scoped.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
