# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **6.1/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale reports a bare status count (e.g., '1/1 reviewed, ⚠1') with no detail on which spec, which field failed, or which rubric category the defect falls under — and silently skips specs without documented justification.:** When you emit a sweep result, for every spec in scope write one explicit per-spec line that names: (1) the spec reviewed, (2) the verdict (pass / needs_fix), (3) if needs_fix — the exact defect category (mangled phases · missing owner/parent/blockers · missing Verification) and the offending field/section quoted from the markdown; if a spec is skipped, document the reason inline (e.g., 'deferred — Ada's call, out of Phase 3 scope'). — Grades stay at 5-6 because reviewers cannot distinguish a correct catch from a false positive, or a legitimate skip from silent negligence, without field-level traceability. Explicit per-spec diagnoses tied to the markdown make every verdict auditable on its own, satisfy the rubric's requirement to confirm diagnosis-matches-markdown, and prove Vale stayed in its Phase 3 lane rather than drifting into Ada's planned/deferred territory.
- **When Vale consistently reports bare status counts (⚠1, 0 pass, 2 needs_fix) with no field-level diagnosis, making it impossible to verify whether flagged issues are real defects or false-fix calls, and impossible to confirm skips are justified.:** When you emit any verdict — pass, needs_fix, or skip — immediately follow it with a one-line field-level diagnosis: name the exact spec, the exact defect category (mangled phase, missing owner/parent/blockers, missing Verification), and the offending field or section as it appears in the markdown; for any skip, document the explicit reason (e.g., 'spec status is deferred — Ada's call, out of Phase 3 scope'). — Bare counts give supervisors no way to audit correctness; a false-fix call and a genuine catch look identical in the log. Attaching a concrete field reference to every verdict makes the diagnosis traceable, proves rubric adherence, and forces Vale to ground each call in the actual markdown rather than a surface impression.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.

Owner: [[../functions/platform]] (Ada supervises Vale). The director authored this from Vale's coaching ledger; building it hardens the agent so the coaching never has to repeat.
