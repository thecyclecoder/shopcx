# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.9/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only a summary counter (e.g., '✅0 ⚠1') with no per-spec detail, making it impossible to verify whether the needs_fix call is accurate, well-scoped, or based on the correct rubric dimension.:** When you emit a sweep result, do Y: for every spec reviewed, log a named diagnosis in the format '<spec-name>: <pass|needs_fix> — <exact defect category: missing Verification | missing owner/parent/blockers | mangled phases | none> — <one-sentence rationale tied to the markdown field>'; never emit only a count. — The grader cannot distinguish a sound catch from a lucky guess when output is a bare counter — explicit per-spec field-level diagnosis is the only evidence that the rubric was applied correctly and that planned/deferred scope was left to Ada. Structured, traceable output directly satisfies the rubric's requirement that diagnoses match the markdown and enables the director to audit without reopening every spec.
- **When After flagging a spec as needs_fix, Vale logs only a summary counter (e.g., '✅0 ⚠1') with no named spec, no identified defect field, and no mapping to the rubric dimension that triggered the call.:** When you complete a spec review and record any needs_fix or pass decision, always emit a per-spec diagnosis line naming: (1) the exact spec reviewed, (2) the precise field or section that failed or passed (e.g., 'missing Verification section', 'no owner field', 'Phase label mangled'), and (3) which rubric dimension it maps to — before writing the summary counter. — The grader cannot distinguish a correct catch from a false-fix call, or confirm full coverage, when the log contains only counts; traceable per-spec evidence is the only way to verify diagnostic quality without reopening every spec. Emitting explicit field-level findings before the summary transforms unverifiable outputs into auditable ones, which is precisely what the rubric rewards at the 10 level.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.

Owner: [[../functions/platform]] (Ada supervises Vale). The director authored this from Vale's coaching ledger; building it hardens the agent so the coaching never has to repeat.
