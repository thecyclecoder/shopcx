# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.4/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently records only a binary pass/flag tally (e.g., '✅0 ⚠1') with no per-spec, per-field diagnosis, leaving the director unable to verify that rubric criteria were actually applied.:** When you see yourself about to log a sweep result, do instead write at least one explicit diagnosis line per reviewed spec naming the spec identifier, the exact defect category (mangled phases / missing owner / missing parent / missing blockers / missing Verification), and the specific offending field or section — e.g., 'SPEC-42: needs_fix — missing Verification section; no owner field present' — before emitting the summary counters. — The grader cannot distinguish sound judgment from a lucky flag without field-level evidence tied to the actual markdown; a named defect category plus offending field is the minimum proof that Vale applied the rubric checklist rather than guessing. Supplying that detail converts an unverifiable tally into an auditable diagnosis, which is exactly what a 10 requires.
- **When Vale consistently reports only aggregate counters (✅0 ⚠1) with no per-spec, per-field diagnosis, making it impossible to verify that the rubric was applied rather than a lucky flag.:** When you issue any pass or needs_fix verdict, immediately log the specific spec name, the exact defect category from the rubric (mangled phases / missing owner / missing parent / missing blockers / missing Verification), and the offending field or section — e.g., 'SPEC-42: needs_fix — missing Verification section' — before moving to the next spec. — Graders cannot distinguish sound judgment from a lucky call without field-level evidence tied to the markdown; explicit per-spec diagnosis confirms the rubric checklist was actually walked and gives the director enough context to act without re-examining the source file.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
