# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.5/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently logs only summary tallies (✅0 ⚠1) with no per-spec, field-level diagnostic detail, making it impossible for the director to verify whether the finding is correct or a false call.:** When you issue any pass, needs_fix, or warning verdict, immediately log: (1) the exact spec name/ID reviewed, (2) the specific rubric field that failed or passed (e.g., 'missing Verification section', 'owner field blank', 'Phase label mangled'), and (3) a one-line quote or reference from the markdown that justifies the call — before moving to the next spec. — Graders cannot distinguish a well-reasoned correct flag from a lucky guess when only a count is surfaced; field-level evidence tied to the actual markdown is the only way to confirm rubric adherence (mangled phases, missing owner/parent/blockers, missing Verification) without re-examining the source. Providing this trace turns an unverifiable 6 into a verifiable 10.
- **When Vale consistently reports only a binary tally (✅0 ⚠1) with no named spec, no identified field, and no explanation of why a flag was raised, making it impossible for reviewers to verify the diagnosis.:** When you issue any pass, needs_fix, or warning verdict, log: (1) the exact spec name/ID reviewed, (2) the specific rubric field that failed or passed (e.g., 'missing Verification section', 'owner field empty', 'phase label mangled'), and (3) one sentence tying the field state in the markdown to the verdict — before closing the sweep. — Every low grade cites the same cause: the log is too sparse to confirm whether Vale applied the rubric or got lucky, so scores cannot exceed 6 regardless of the binary outcome. Naming the spec, the defective field, and the markdown evidence makes the diagnosis independently verifiable and demonstrates rubric adherence rather than just a non-zero flag count.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
