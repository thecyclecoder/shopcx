# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.7/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently emits only a bare tally (✅0 ⚠1) in the log without naming the spec, the defect category, or the offending field, making every finding unverifiable.:** When you see yourself about to log a spec-review result, do Y instead: write a per-spec diagnosis line that names (1) the spec identifier, (2) the exact defect category (mangled phase / missing owner / missing parent / missing blockers / missing Verification), and (3) the specific field or section in the markdown that is defective — e.g., 'SPEC-42: needs_fix — Verification section absent; owner field blank' — before emitting any summary tally. — Graders cannot distinguish a sound rubric-aligned judgment from a lucky guess when the log contains only counts; a named, field-level diagnosis is the minimum evidence that Vale actually checked each rubric criterion against the markdown. Providing this detail also forces Vale to confirm it stayed in Phase 3 quality scope (pass/needs_fix) and did not drift into planned/deferred territory, directly satisfying the two axes the rubric measures.
- **When Vale consistently outputs only summary tallies (✅0 ⚠1) with no named spec, no identified defect category, and no field-level reasoning, making every finding unverifiable.:** When you complete a spec review and assign any verdict (pass or needs_fix), log a structured diagnosis entry that names: (1) the exact spec reviewed, (2) the specific rubric category violated (mangled phase / missing owner or parent or blockers / missing Verification), (3) the offending field or section quoted from the markdown, and (4) one sentence confirming you are staying in Phase 3 quality scope and not making a planned/deferred call — before recording the tally. — Graders cannot distinguish a correct judgment from a lucky guess when the log contains only a count; rubric criteria (mangled phases, missing owner/parent/blockers, missing Verification) are field-level checks that must be shown, not assumed. Explicit per-field diagnosis in the log is the only evidence that Vale applied the rubric rather than pattern-matched on superficial signals.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
