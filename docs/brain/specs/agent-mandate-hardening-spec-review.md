# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **5.7/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently outputs only aggregate counters (e.g., '✅0 ⚠1') with no per-spec defect detail, making every verdict unauditable regardless of whether the underlying judgment was correct.:** When you issue any pass or needs_fix verdict, immediately follow it with a per-spec breakdown listing: the spec name, the specific field(s) checked (phases, owner, parent, blockers, Verification), the exact defect found or the explicit confirmation it was present and valid — never emit just a tally counter. — Graders cannot distinguish a correct diagnosis from a lucky guess or a false call without seeing the reasoning tied to the actual spec content; sparse logs systematically cap scores at 5–6 regardless of accuracy. Explicit per-field callouts make the rubric criteria directly verifiable and demonstrate that Vale applied its full checklist rather than short-circuiting to a flag.
- **When Vale consistently reports only tally counters (e.g., '✅0 ⚠1') without naming the specific spec, the defect category (missing owner/parent/blockers, mangled phases, missing Verification), or the reasoning behind each pass/needs_fix call.:** When you emit a sweep result for any spec — pass or needs_fix — always include a per-spec line that names the spec, the verdict, and the exact defect or confirmation (e.g., 'spec-X: needs_fix — Verification section absent' or 'spec-Y: pass — all phases well-formed, owner/parent/blockers/Verification present'); if a spec is skipped, state the explicit reason (e.g., 'spec-Z: skipped — status is deferred, not in_review'). — Graders cannot verify that the rubric was applied correctly from a bare counter alone — they need a traceable diagnosis to confirm each flag is a real defect and not a false-fix call. Per-spec defect callouts make the reasoning auditable, which is the primary evidence of a quality review and the direct path from a 6 to a 10.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
