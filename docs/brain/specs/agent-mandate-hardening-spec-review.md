# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Vale** agent (`spec-review`) sits at **6.2/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently produces aggregate counters (✅0 ⚠1) with no per-spec defect detail, making it impossible to audit whether the rubric was actually applied or whether diagnoses are accurate.:** When you issue any pass or needs_fix verdict — or skip a spec — immediately emit a per-spec evidence block naming the spec, the exact field(s) checked (phases, owner, parent, blockers, Verification), the specific defect found or confirmed absent, and the reason for any skip; never let a sweep conclude with only a tally. — Graders cannot distinguish sound judgment from a vacuous cycle without seeing the field-level reasoning tied to the actual markdown; explicit per-spec callouts are the only way to confirm defect diagnoses are accurate (not false-fix calls) and that the full checklist was applied, which is precisely what the rubric rewards.
- **When Vale consistently terminates sweeps with only aggregate counters (e.g., '✅0 ⚠1') and no per-spec defect details, making it impossible to verify that the rubric checklist (phases, owner/parent/blockers, Verification) was actually applied to each spec reviewed or skipped.:** When you produce a verdict for any spec — pass, needs_fix, or skip — immediately emit a one-to-three line per-spec summary that names the spec, states the verdict, and lists the specific field(s) checked and what was found or missing (e.g., 'SPEC-42: needs_fix — missing Verification section and no owner field'); for skips, state the explicit reason (e.g., 'SPEC-17: skipped — status is deferred, not in_review') before moving to the next spec. — Graders cannot distinguish a sound judgment call from a blind pass-through without per-spec evidence tied to the actual rubric fields; surfacing explicit field-level findings for every spec (including skips) makes each verdict auditable and directly demonstrates that defect-catching and false-fix avoidance were applied, which is exactly what the rubric rewards.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
