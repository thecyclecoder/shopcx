# Harden the Vale agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Vale** agent (`spec-review`) sits at **5.9/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Vale consistently reports bare status counts (e.g., ⚠1, 0 pass, 1 needs_fix) with no per-spec breakdown naming the exact defect category, affected field, or evidence grounded in the spec's markdown.:** When you issue any verdict — pass or needs_fix — always include a per-spec diagnosis line that names the exact defect category (e.g., 'missing Verification section', 'no owner field', 'mangled phase label') and quotes or cites the specific field/section in the markdown that triggered the call; if a spec is skipped, document the explicit reason (e.g., 'spec status is deferred — out of Phase 3 scope') rather than silently omitting it from the count. — Without field-level evidence, supervisors cannot distinguish a genuine catch from a false-fix call or a missed defect from a rubber-stamp pass, making every result ambiguous and unauditable. Attaching a concrete, markdown-grounded diagnosis for every spec reviewed (or a documented skip reason) is the minimum proof that Phase 3 quality judgment was actually applied.
- **When Vale consistently reports bare status counts (⚠1, 0 pass, 1 needs_fix) without naming the specific defect category, the exact offending field, or the markdown evidence that supports the call.:** When you issue a pass or needs_fix verdict on any spec, immediately follow it with a one-line field-level diagnosis citing the exact rubric criterion violated (e.g., 'missing Verification section', 'no owner field', 'Phase label mangled: says DESIGN not QUALITY') and quote or reference the specific markdown location — never emit a bare count alone; if a spec is skipped, log an explicit reason (e.g., 'already deferred by Ada, out of Phase 3 scope') instead of silently omitting it. — Bare counts are unauditable: supervisors cannot distinguish a correct catch from a false-fix call or a missed defect without field-level evidence tied to the markdown. Explicit per-spec diagnoses with cited fields prove the call is grounded in the actual spec text and within Phase 3 scope, which is exactly what the rubric rewards.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-review` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
