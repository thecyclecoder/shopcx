# Harden the Mira agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Mira** agent (`migration-fix`) sits at **6.0/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When On subscriptions whose only failing check is a non-catalog 'phantom' line (a $0 free/one-time Appstle promo line or a 'Shipping Protection' add-on), Mira reaches for price_reconcile/price_override_cents or punts to completed/human-needed instead of emitting the purpose-built fix_kind that already exists for exactly that case.:** When the failing line is a $0 free/one-time promo line, propose fix_kind='remove_line' — NEVER price_reconcile with price_override_cents=0 (0 is invalid, it cannot encode $0). When the failing line is a 'Shipping Protection' add-on, propose fix_kind='shipping_protection_convert' (it sets the flag+amount, drops the line, AND corrects the inflated baseline) — do NOT declare human-needed. Only conclude 'completed' after the worker's re-verify actually passes; a still-failing audit means propose again or emit human_needed, never completed. — price_override_cents=0 is rejected by the `c <= 0` guard at src/lib/migration-fix.ts:142 (Job 33e38b94's 'out-of-range cents'), while remove_line (migration-fix.ts:296, whose header comment describes this exact free-$0-line case) and shipping_protection_convert (migration-fix.ts:256, which fixes the over-counted pre_migration_charge_cents at :287-291) are the dedicated handlers Jobs dd128f73/cb155f5d/a7e14d7e diagnosed but never emitted.
- **When On subscriptions carrying a free/$0 one-time promo line (_appstle-free-product), Mira tries to preserve the $0 via price_reconcile→price_override_cents=0 (job 33e38b94) or diagnoses removal but emits no fix action (job dd128f73), instead of proposing the purpose-built remove_line fix.:** When a failing items_on_uuids/pricing_preserved line is a free/$0 one-time promo (currentPrice $0.00, _appstle-free-product / _appstle-one-time-product, a lingering Shopify-id with no catalog UUID), NEVER reconcile it with price_reconcile and price_override_cents=0 — the fixer hard-rejects c<=0. Propose fix_kind:'remove_line' identifying that single line (by shopify_variant_id/title), because a $0 gift is not a recurring billable; removing it clears items_on_uuids and leaves the real line's pricing untouched. Only conclude 'completed' after verifyMigration re-passes. — price_reconcile validates every override at src/lib/migration-fix.ts:142 (`c <= 0` → 'out-of-range cents'), which is exactly the error job 33e38b94 hit setting variant a9c8e1d7 to 0; the correct handler remove_line (src/lib/migration-fix.ts:296-329) was built for precisely this '$0 FREE/promo line with no catalog identity' case that job dd128f73 correctly diagnosed but never emitted.
