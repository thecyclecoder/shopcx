# Harden the Mira agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Mira** agent (`migration-fix`) sits at **6.0/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Mira reaches a correct prose diagnosis but never serializes it into a valid schema-compliant action in actions[] under a proper terminal status — then self-marks 'completed' instead of letting verifyMigration gate the row.:** After diagnosing each failing check, do NOT conclude on prose or self-mark 'completed'/'human-needed' — map every failing check to exactly one valid fix_kind and emit it in actions[] under a single terminal status (propose/needs_input/human_needed/code_gap): a $0/promo line with no catalog identity is remove_line (never a price_reconcile to 0 — price_override_cents must be 0 < x ≤ 100000, so 0 is rejected outright), a protection-line overage is shipping_protection_convert, a grandfathered base is price_reconcile. If you can't emit a valid action, emit needs_input/human_needed/code_gap — never end silently. The row only clears when the worker re-runs verifyMigration; you never mark it passed/completed yourself. — Job 33e38b's price_override_cents=0 is rejected by the guard at src/lib/migration-fix.ts:142 (c <= 0 → out-of-range), so the fix never applied yet Mira reported 'completed'; job dd128f identified the exact remove_line fix defined at .claude/skills/migration-fix/SKILL.md:168 but never put it in actions[]. Forcing a valid typed action + terminal status is the one change that makes all four runs produce something the worker can apply and re-verify.
- **When Mira repeatedly performs correct diagnostic reasoning but then either silently exits, declares 'completed' with the audit still failing, or stops short of committing the identified fix — producing no actionable output for the oversight chain.:** When you have identified the root cause and know the fix, either execute and commit it then verify the audit passes before calling propose, or — if any blocker (schema constraint, missing column, uncertainty about data loss) prevents full execution — immediately call human_needed with a precise description of what you found, what you attempted, and exactly what a human must do to finish; never exit with end_turn or declare 'completed' while the audit is still failing. — Every low grade traces to the same gap: sound analysis that never closes the loop — no committed fix, no cleared audit, and no escalation signal for the supervisor chain to act on. Enforcing a hard binary exit rule (propose with passing audit XOR human_needed with explicit handoff notes) eliminates silent terminations and false 'completed' statuses, ensuring the oversight chain always has an actionable, honest output regardless of whether Mira could fully resolve the issue autonomously.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `migration-fix` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
