## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When When a subscription carries a free/$0 one-time promo line (a lingering Shopify-id artifact), Mira fails to emit the purpose-built remove_line fix — job 33e38b94 tried price_reconcile with price_override_cents=0, and job dd128f73 correctly diagnosed 'remove the line' but proposed no action — so items_on_uuids/pricing_preserved stay red and the audit never clears.:** When items_on_uuids or pricing_preserved fails on a line that is $0 / _appstle-free-product / one-time promo / has no catalog variant row, propose a remove_line fix (payload {shopify_variant_id|title|line_id}), NOT a price_reconcile with price_override_cents=0 and NOT analysis-only. price_override can never represent $0, so remove the artifact line instead — then let the worker re-run verifyMigration; if you can't emit a fix, exit via propose or human_needed, never end_turn. — src/lib/migration-fix.ts:142 rejects any override with c<=0 as 'out-of-range cents' (exactly job 33e38b94's failure), while migration-fix.ts:296 documents remove_line as built for 'a FREE/promo line with no catalog identity ($0)' — the fix job dd128f73 correctly identified but never emitted; using remove_line clears items_on_uuids and pricing_preserved that price_override_cents=0 structurally cannot.
- **When When a sub fails items_on_uuids because the old migration dragged in a $0 free/one-time Appstle promo line (lingering Shopify variant id, _appstle-free-product/_appstle-one-time-product), Mira either fixes it with the wrong kind (price_reconcile → price_override_cents=0) or correctly names remove_line but never emits the propose, so the audit never clears.:** When the only failing check is items_on_uuids and the offending line is a $0 free/one-time promo line (no product_variants row, _appstle-free-product/_appstle-one-time-product on the live Appstle contract), emit a remove_line propose action keyed on the line — NOT price_reconcile with price_override_cents=0, and NOT variant_backfill — then actually emit the propose JSON before concluding; a $0 override can never pass, so that path leaves the audit failing. — Job 33e38b94 proposed price_reconcile price_override_cents=0, which is rejected by the guard at src/lib/migration-fix.ts:142 (c <= 0 ⇒ 'out-of-range cents') — structurally unable to clear the audit; the documented fix for this exact case is remove_line (src/lib/migration-fix.ts:296-301: 'A FREE/promo line with no catalog identity ($0, no product_variants row)… we want it GONE').
