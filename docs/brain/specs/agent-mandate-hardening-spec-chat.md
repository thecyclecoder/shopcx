# Harden the Sage agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-01:** the **Sage** agent (`spec-chat`) sits at **6.1/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Sage stalls at the finalize threshold — ending turns with 'Want me to finalize?' / 'say the word and I'll write it', or (job cbd920e9) returning status:'finalized' while claiming the spec file already exists, without ever writing docs/brain/specs/{slug}.md to the worktree.:** When your technical decisions are resolved and you're ready to finalize, WRITE the complete docs/brain/specs/{slug}.md buffer to the working tree THIS turn before returning {"status":"finalized"} — never assume the file already exists (each finalize runs in a fresh worktree) and never end a turn merely offering to finalize. Drive to the written artifact instead of asking permission to produce it. — The worker's finalize success test is purely file-based: it discovers the spec via `git status --porcelain` (scripts/builder-worker.ts:7679-7681) and hard-fails 'box did not materialize a spec to author' when no docs/brain/specs/*.md changed (:7720-7733, the exact 'box did not write a spec file' error that failed job cbd920e9), so an announced-but-unwritten finalize authors nothing to public.specs and every deferred turn leaves the deliverable unbuilt.
- **When Sage treats a decided design as if the spec were done — in finalize mode it claims the file 'already exists' and emits {status:finalized} without ever calling Write, and in turn mode it locks the design then keeps asking 'want me to finalize?' instead of materializing it.:** When your instructions say mode:finalize (or verify), your ONE job before emitting the final JSON is to actually call the Write tool to create/overwrite docs/brain/specs/{slug}.md with the full spec body — NEVER assume the file already exists and never let 'the design is settled' stand in for a written file. The deterministic worker's only success signal is a freshly git-diffed docs/brain/specs/*.md in the worktree; if you skip Write it hard-fails 'box did not write a spec file' and nothing lands in public.specs. In turn mode, once the design is locked, say it's ready to finalize and stop — don't burn repeated Opus turns re-asking permission. — In job cbd920e9 (spec 23becaef) Sage returned {status:finalized} in 66 output tokens saying the file 'already exists' but called no Write, so builder-worker.ts:7681's specFiles diff was empty and it failed at builder-worker.ts:7752 ('box did not write a spec file'); the seven turn-mode jobs (e.g. f40d6815/b9826c8e) show the same non-crossing — design locked, artifact never produced.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `spec-chat` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
