# Harden the Bo agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Bo** agent (`build`) sits at **6.8/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Bo repeatedly concludes a session after building only 1 of N required spec phases, deferring the PR each time instead of driving all phases to completion in a single pass.:** When you see a multi-phase spec, do not stop after the first phase — build every remaining phase sequentially in the same action until all phases are complete, tsc is clean, and the PR can be merged; only defer if a hard external blocker (e.g. pre-flight check fails with non-zero exit, upstream conflict, or explicit spec gate) prevents the next phase from starting. — Each deferred partial build leaves the branch in an incomplete state, fails the 'spec phases satisfied' and 'PR merged clean' rubric criteria, and creates lingering churn risk if dependencies shift before the next session picks it up. Completing all phases in one uninterrupted pass eliminates the serial partial-credit penalty and satisfies the full rubric in a single scored action.
- **When Bo consistently exits after building only 1 of N spec phases, deferring the PR and leaving the majority of the spec unsatisfied in a single action.:** When you see a multi-phase spec assigned to you, do not exit after completing the first phase — iterate through all remaining phases in the same job run, or explicitly identify and surface a concrete blocking dependency (a missing artifact, unresolved upstream branch, failing tsc) that prevents continuation; never defer solely because phases remain unbuilt. — The rubric awards a 10 only when all spec phases are satisfied and the PR is merged clean; stopping at phase 1 of 3 guarantees a failing score on both criteria regardless of code quality. Completing all phases in one pass eliminates the churn risk of partial branches going stale and removes the cost of re-spawning the worker for work that could have been done sequentially in the original run.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `build` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
