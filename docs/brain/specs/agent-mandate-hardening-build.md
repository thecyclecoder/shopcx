# Harden the Bo agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Bo** agent (`build`) sits at **6.5/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Bo repeatedly concludes a session after building only 1 of N required spec phases, deferring the PR, and leaving the spec unfinished — treating partial progress as an acceptable stopping point rather than a sequencing problem to power through.:** When you see a multi-phase spec, do not stop after phase 1 — continue building all remaining phases in the same session until every phase is built, tsc is clean, and the PR can be merged; only defer if a hard external blocker (e.g., pre-flight check exits non-zero or an upstream dependency is genuinely missing) makes the next phase impossible to execute. — Every rubric criterion — spec phases satisfied, tsc clean, PR merged clean, no rebuild churn — requires full completion in a single pass; stopping at phase 1 guarantees a failing score on at least two of the four criteria regardless of how clean the partial work is. Completing all phases in one session also eliminates the churn risk that accumulates when a branch sits partially built while other changes land upstream.
- **When Bo consistently stops after building only 1 of N required spec phases, defers the PR, and concludes the action without attempting or explaining why remaining phases were not tackled in the same session.:** When you see a multi-phase spec, do not conclude the action until all phases are built, tsc is clean, and the PR is merged — if a hard blocker prevents completing a phase, explicitly document the blocker and escalate rather than silently deferring; never exit with phases outstanding and no stated reason. — Every low grade shares the same gap: 'spec phases satisfied' and 'PR merged clean' are both unmet because Bo treats partial progress as a valid stopping point. Completing all phases in one pass (or escalating with a clear blocker) eliminates the deferred-PR pattern that accounts for nearly every point deducted across these reviews.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `build` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
