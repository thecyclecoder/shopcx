# Harden the Pax agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-02:** the **Pax** agent (`pr-resolve`) sits at **6.9/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Pax repeatedly hard-fails at the very first step because it attempts git worktree operations without first detecting and cleaning up stale or conflicting worktrees already present at the target path or checking out the target branch.:** When you see a worktree operation is needed, first run `git worktree list` to check for existing entries on the target branch or path — if a stale or conflicting worktree exists, remove it with `git worktree remove --force <path>` followed by `git worktree prune`, then proceed; if removal fails, use a unique alternate path (e.g., append the PR number and a timestamp) before attempting `git worktree add`. — Every low-scored job aborted at environment setup before a single conflict was touched, meaning zero rubric value was delivered and the queue was left blocked. Adding this mandatory pre-flight check eliminates the entire class of self-inflicted worktree failures, allowing Pax to reach actual conflict resolution work on the first attempt rather than exhausting retries on a preventable setup error.
- **When Pax repeatedly attempts git worktree operations without first inspecting the environment, causing hard failures on stale locks, already-checked-out branches, and broken ancestry states that abort all meaningful work.:** When you see any git worktree or rebase operation about to run, first execute `git worktree list` and verify no existing worktree holds the target branch or path — if one exists, run `git worktree prune` and/or `rm -rf <stale-path>` before proceeding; additionally, after any rebase, assert `git merge-base --is-ancestor origin/main HEAD` before marking a PR merged, and if the check fails, abort and escalate with the specific failure detail rather than pushing through. — Every low-scored job failed at setup or cleanup — never at actual conflict resolution — because Pax skips environment validation and dives straight into operations that require a clean state. Adding mandatory pre-flight checks (worktree inventory + post-rebase ancestry assertion) eliminates the entire class of self-inflicted fatal errors and ensures the queue is only marked merged when the rebase integrity is confirmed, directly satisfying 'clean rebase, queue left mergeable'.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `pr-resolve` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
