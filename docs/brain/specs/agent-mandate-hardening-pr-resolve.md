# Harden the Pax agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Pax** agent (`pr-resolve`) sits at **6.4/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When Pax repeatedly attempts git worktree operations without first inspecting the environment, causing immediate hard-failures on stale locks or conflicting checkouts that are entirely detectable and recoverable before any work begins.:** When you see any worktree-dependent operation (checkout, add, rebase setup), first run `git worktree list` and inspect `/tmp` for existing paths matching the target branch; if a conflict exists, run `git worktree prune` and remove the stale path, then proceed — never attempt `git worktree add` without confirming the target path and branch are free. — Every low-scored run aborted at step zero because a stale or conflicting worktree blocked setup, producing zero conflict resolution and leaving the queue blocked — a problem that a two-line pre-flight check would have eliminated. Embedding this check as a mandatory first action turns a recurring fatal blocker into a handled, recoverable condition and restores the 'clean rebase, queue left mergeable' baseline the rubric demands.
- **When Pax repeatedly attempts git worktree operations without first inspecting and cleaning up stale or conflicting worktree state, causing immediate hard failures before any conflict resolution work begins.:** When you see a worktree add or checkout operation is needed, first run `git worktree list` to detect any existing or stale worktrees for the target branch, then either `git worktree remove --force <path>` / `git worktree prune` to clean up stale entries or reuse the existing path — only then proceed with the operation. — The majority of low-graded jobs failed at step zero with a fully preventable worktree conflict, meaning zero rubric criteria were met despite no inherent difficulty in the actual PR work. Adding a mandatory pre-flight worktree inspection-and-cleanup step eliminates this self-inflicted failure class entirely, giving Pax a clean environment before attempting any rebase or conflict resolution.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `pr-resolve` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
