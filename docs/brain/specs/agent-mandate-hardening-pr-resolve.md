# Harden the Pax agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-30:** the **Pax** agent (`pr-resolve`) sits at **5.0/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When The worker repeatedly begins execution without validating pre-conditions — stale worktrees block checkout, ancestry checks are skipped before merging, and closed/redundant PR states are not escalated — causing hard failures or corrupt merge states that leave the queue broken.:** When you see any setup or rebase step about to execute, first run a mandatory pre-flight checklist: (1) prune stale worktrees with `git worktree prune` and remove any conflicting path before `git worktree add`; (2) after rebasing, assert `git merge-base --is-ancestor origin/main HEAD` passes before pushing — if it fails, abort and escalate instead of merging; (3) if a PR is closed/unmerged or flagged redundant, halt, log the anomaly with a clear Director-facing note, and set concluded status to 'blocked' not 'merged'. — Every low-graded job traces back to skipping environment or integrity verification before committing an irreversible action (checkout, push, or merge), which converts recoverable setup issues into broken queue states. Enforcing these three checks before each stage catches both environmental hygiene failures and rebase ancestry violations at the earliest possible moment, preserving the rubric's requirements of no lost work, a clean rebase, and a mergeable queue.
- **When The worker repeatedly completes or attempts a merge without first confirming that origin/main is a true ancestor of the resolved HEAD and that the PR's design/spec is still congruent with what main already ships.:** When you see a PR ready to merge, do Y: before pushing, run `git merge-base --is-ancestor origin/main HEAD` and diff the PR's core spec/logic against what origin/main already contains — if either check fails (ancestry broken OR semantic/design conflict detected), abort and escalate to the Director with a clear diagnosis instead of proceeding. — The majority of low-grade jobs share one root cause: the worker skipped or ignored the post-rebase ancestry verification and spec-divergence check, causing it to land broken rebases or superseded/contradictory designs onto main. Enforcing both checks as a hard gate before every merge directly addresses the 'clean rebase' and 'queue left mergeable' rubric requirements and prevents the worker from making an already-broken state worse by completing the merge.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `pr-resolve` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
