# Harden the Pax agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-07-02:** the **Pax** agent (`pr-resolve`) sits at **6.7/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When when a rebase/auto-merge of a claude/* PR shows origin/main already added an exported symbol your branch also defines (e.g. two `export async function enqueueSecurityDiffIfDue` / duplicate `EnqueueSecur*` type), meaning main merged a superseding fix-of-fix while your PR sat in the queue:** before attempting the merge, grep origin/main for each exported symbol your branch adds; if main already defines it, treat the PR as SUPERSEDED — recommend closing/withdrawing it (cite the superseding PR #s) rather than force-merging or leaving it dangling. Never land a merge that produces two identical exports. — On pr-1004 (e6f6af51) origin/main had already merged #991+#993 which replaced this branch's getSpec-based enqueueSecurityDiffIfDue (now at security-agent.ts:324 on main) with the spec_status_history approach; the attempted merge silently produced a duplicate exported type+function (tsc break) and the PR was left un-mergeable. Detecting the superseding merge up front and closing the obsolete PR keeps the queue mergeable and avoids the duplicate-symbol unwind — Pax's rolling slipped to 6.5 on exactly this class of late-detected superseding.
- **When when your rebase/merge onto main leaves `git merge-base --is-ancestor origin/main HEAD` returning NO (the 'rebuilt-on-main: merge incomplete' guard trips) AND the branch's feature already exists on main:** before spending a full resolve budget, check whether the branch is superseded — grep main for the branch's key new symbols/files (e.g. #978's enqueueSecurityDiffIfDue + security-diff-backstop-cron.ts already on main via #991/#993); if it shipped elsewhere, surface the PR for CLOSURE, don't re-resolve. And never conclude `completed` while main is not an ancestor of the resolved HEAD — that's a failed, un-mergeable resolve, not a completion. — On ca1aabd0 Pax burned ~$1.05 / 17.9k output tokens rebuilding claude/build-vault-post-merge-diff-backstop, then aborted with the incomplete-merge guard while its function had ALREADY landed on main — the resolve was unwinnable from the start and cheap to detect, which is what dragged the pr-resolve rolling average down.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `pr-resolve` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
