# Harden the Pax agent — roll persistent coaching into its mandate

**Owner:** [[../functions/platform]] · **Parent:** [[platform-director-agent]] — graduate persistent coaching into a durable fix (director grades agents: low score → fix spec, never a CEO escalation)
**Found in use 2026-06-29:** the **Pax** agent (`pr-resolve`) sits at **5.2/10** after **2** coaching attempts that didn't stick. Rather than escalate to the CEO, roll the coaching into a permanent fix so the agent improves at the mandate level.

## The accumulated coaching to bake in (now archived as rolled-into-mandates)
- **When When conflicts prove difficult or a PR appears unresolvable, Pax either retries blindly until cap or escalates immediately without first diagnosing whether the conflict is a git ancestry issue, a duplicate, or a genuinely human-requiring semantic problem.:** When you see a conflict resist initial resolution attempts, stop after attempt 2 and explicitly diagnose: (1) verify origin/main IS an ancestor of your working HEAD before proceeding further — if not, fix the rebase foundation first; (2) check whether the PR is a duplicate of already-merged work and document any unique commits not present in main; (3) only after diagnosis, either fix the root git ancestry issue and continue, or escalate with a written rationale naming the specific unconflatable lines — never retry blindly or close without a diff-verified duplicate check. — The recurring slip is treating symptoms (unresolvable conflicts, closed PRs) without confirming the underlying git state is sound — rebasing on a broken ancestry silently produces a non-mergeable branch no matter how many retries occur. Diagnosing first ensures each subsequent action targets the actual root cause, satisfying the rubric's requirements for a clean rebase and a mergeable queue rather than a needs_attention dead end.
- **When When conflicts resist initial resolution attempts, Pax loops through all available retries or concludes with an unverified state rather than diagnosing unresolvability early and escalating with a reasoned explanation.:** When you see a conflict fail to resolve after 2 attempts, or when a post-rebase ancestry check shows origin/main is not an ancestor of the resolved HEAD, stop retrying immediately — diagnose whether the conflict requires human judgment (e.g., semantic conflicts, duplicate content, broken git state), then escalate with a specific explanation of why it is unresolvable rather than exhausting retries or closing with an unverified mergeable state. — Blind retrying burns token budget without improving outcomes and leaves the queue in needs_attention state, directly violating the clean-rebase and mergeable-queue rubric goals. Early diagnosis after the second failure preserves retries for genuinely recoverable conflicts and surfaces human-requiring cases faster, keeping the queue actionable.

## Phase 1 — bake the coaching into the agent
- Make the above coaching PERMANENT behavior of the `pr-resolve` agent — fold it into its prompt/mandate/code (its run-job + prompt in `scripts/builder-worker.ts` and the relevant `src/lib/agents/*`), not ephemeral appended `agent_instructions`. Once baked, the agent should follow it by default.
- Verify the agent's grade rollup recovers (≥ 7/10) over the next window of graded actions; if a coaching class still recurs, the bake-in missed it.
