## Part A2 â€” never-fired detection + Inngest registration reliability
The Control Tower's own monitor cron sat **"awaiting first run" for days** because **Inngest never registered it** (a deploy didn't re-sync the app; a manual resync found 2 unregistered functions). The watchdog was itself the silent failure â€” and the monitor couldn't catch it, because a never-fired cron shows **amber "awaiting first run," never red**, so it never paged. Close that:
- **Distinguish "never-fired-past-grace" (red) from "awaiting first tick" (amber).** A registered cron with **0 heartbeats** whose **deploy/registration age exceeds its cadence + a grace** is **not** awaiting its first tick â€” it's **registered-but-not-firing** â†’ **red + page** ("cron X registered but has never run â€” Inngest not invoking it?").
