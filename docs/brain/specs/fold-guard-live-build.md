# Fold guard ‚Äî don't archive a spec with a live build; reconcile jobs on archive ‚úÖ

**Owner:** [[../functions/platform]] ¬∑ **Parent:** hardens the fold path + [[auto-ship-pipeline]] (Gate B auto-fold) + [[worker-orphan-reaper]]. ¬∑ **Found in use 2026-06-22:** `control-tower-escalation-idle-grace` was **folded + archived (#250) while a `needs_input` build for it was still live** ‚Äî the build paused asking questions, the spec markdown moved to `archive.d/`, and the box then showed "1 build paused" with a link that **404'd** (the spec page is gone). A spec got folded out from under a running build.

A fold archives the spec markdown (moves it to `archive.d/`, removes the active card). If a non-terminal build/spec-test job for that spec is still alive, it becomes an **orphan**: it lingers in `needs_input`/`building`/etc., shows as a paused/active item, but its spec page 404s and answering it is meaningless (the spec is gone). This gets *more* likely once auto-fold runs automatically.

## Fix (two complementary guards)
- **Don't fold a spec with a live job (preventive).** The fold path (`enqueue_fold` / the verify‚Üífold action + the auto-fold gate in [[auto-ship-pipeline]]) must **refuse / defer** folding a spec that has a non-terminal job (`queued`/`claimed`/`building`/`needs_input`/`needs_approval`/`queued_resume`). Fold only once the build is terminal (merged/completed/failed). Prevents the race at the source.
- **Reconcile jobs when a spec IS archived (cleanup backstop).** When a spec moves to `archive.d/` (fold merges), any still-non-terminal job for that slug is **cancelled** (status‚Üícompleted with a clear "spec archived" reason, questions cleared) ‚Äî so no orphaned paused/active item with a dead link survives. Fits naturally in `reapOrphans` ([[worker-orphan-reaper]]) or the fold-merge reconcile.
- **Box paused/failed callout tolerates a missing spec** ‚Äî if a job's spec page would 404 (archived/deleted), the box links to a safe target or shows it as resolved, never a dead link (defense in depth).

## Verification
- On a spec that has a live `building`/`needs_input` build or spec-test job (`agent_jobs.spec_slug = slug`, `kind in (build, spec-test)`, status in queued/claimed/building/needs_input/needs_approval/queued_resume), tap **Mark verified & archive** (dashboard `BuildButton` ‚Üí `POST /api/roadmap/build {verify:true}`, or the Slack `roadmap_verify:` action) ‚Üí expect a **409** `Can't archive ‚Äî a {kind} build for this spec is still live ({status}). It'll fold once that build finishes.` and **no `pending_folds` row / no `kind='fold'` job** created. Re-tap once the build is terminal (merged/completed/failed) ‚Üí the fold enqueues normally.
- On the auto-fold gate: a shipped spec that is all-green (latest `spec_test_runs` agent-verdict `approved`, 0 waiting/failed human checks, 0 regressions) **but** still has a live build/spec-test job ‚Üí `getAutoFoldEligibleSlugs(workspaceId)` **omits** it and `autoFoldVerifiedSpecs` does **not** enqueue a fold for it. Once that job goes terminal, the next gate pass (spec-test completion / human-queue resolution / daily cron) enqueues the fold. Probe: `select spec_slug,kind,status from agent_jobs where spec_slug='{slug}' and status in (...)`.
- Archive-cancel backstop (two spots): with a non-terminal build/spec-test `agent_jobs` row whose `spec_slug` is in `docs/brain/archive.d/` ‚Üí it is flipped to `status='completed'`, `error='spec archived ‚Äî build auto-cancelled ‚Ä¶'`, `questions=[]`, `pending_actions=[]` either (a) on box worker startup via `reapArchivedSpecJobs` (runs right after `reapOrphans`; log `cancelled N orphaned job(s) for archived spec(s): ‚Ä¶`), or (b) on the next board load after a fold PR merges via `reconcileMergedJobs` (the `kind='fold'` merge calls `cancelJobsForArchivedSpecs`). Negative: a non-terminal job whose spec is NOT archived is untouched.
- On `/dashboard/roadmap/box` (data from `GET /api/roadmap/box`): a paused/failed build whose spec was archived/deleted carries `spec_missing:true`; its callout link points to **`/dashboard/roadmap`** (the board, always 200) with a `(spec archived)` tag ‚Äî never the 404 `/dashboard/roadmap/{slug}`. Negative: a job whose spec still exists links to `/dashboard/roadmap/{slug}` as before.

## Phase 1 ‚Äî fold-refuses-live-build + archive-cancels-orphan-jobs ‚úÖ
Live-job check added to the fold path: `getLiveJobForSlug` ([[../libraries/agent-jobs]]) gates the manual verify‚Üífold in `queueRoadmapBuild` ([[../libraries/roadmap-actions]]) and the auto-fold gate `getAutoFoldEligibleSlugs` ([[auto-ship-pipeline]]). `cancelJobsForArchivedSpecs` ([[../libraries/agent-jobs]]) cancels non-terminal build/spec-test jobs for an archived slug, wired into the worker reaper as `reapArchivedSpecJobs` ([[worker-orphan-reaper]]). The box callout (`/api/roadmap/box` + `box/page.tsx`, [[../dashboard/roadmap]]) tags `spec_missing` and routes those links to the board, never a dead spec page. The worker also marks a spec-test job terminal before running the auto-fold gate so a job never blocks its own fold.
