# dashboard/agents

The owner-only **Agents hub** ([[../specs/agents-hub-role-inboxes]], M1 of [[../goals/devops-director]]) ‚Äî the single surface that renders the live org chart (**CEO ¬∑ Directors ¬∑ Workers**) read from the brain, and gives every role the same **three-tab inbox** (Messages ¬∑ Approval Requests ¬∑ Daily Summaries). The foundation the routing engine (M2), the gamified board (M3), and the live Platform director (M4) all write *into*. Formalizes today's reality ‚Äî no director is automated, so **every approval routes to one CEO inbox** ‚Äî into one place, replacing the scattered approval surfaces (Control Tower feeds, spec cards, the box `approvalHref`).

**Route:** `/dashboard/agents` (client poller, owner-only)
**Sidebar:** **Developer** section (owner-only) ‚Üí **Agents** (right under [[control-tower|Control Tower]]).

## Surfaces

- **Left role nav** ‚Äî **CEO** (the `ceo` persona seat, with its active-goal count from `goals/*.md`), then one row per `functions/*.md` **director** ([[../functions/platform|Platform]] ¬∑ [[../functions/growth|Growth]] ¬∑ [[../functions/cmo|CMO]] ¬∑ [[../functions/cs|CS]] ¬∑ [[../functions/retention|Retention]]) showing its **persona chip + SVG mascot** + a **live/autonomous badge** (M1: every director is `offline` ‚Üí "routes to CEO"; the flag itself lands in M2). Under each director, its **Workers** ‚Äî the box [[../tables/agent_jobs]] lanes it owns (the `agent-kind` loops in the Control Tower [[../libraries/control-tower|registry]], which already carry an owner function: build/plan/fold/spec-chat/spec-test/dev-ask/pr-resolve/repair ‚Üí Platform; ticket-improve/triage-escalations ‚Üí CS; migration-fix ‚Üí Retention; storefront-optimizer ‚Üí Growth).
- **Right pane** ‚Äî the selected role's header (mascot + name + role + personality + mandates/goals). For a **director**, an owner-only **Autonomy toggle** (approval-routing-engine M2 / Phase 1) ‚Äî two checkboxes, **Live** (the director-agent is running) + **Autonomous** (trusted to auto-decide), backed by [[../tables/function_autonomy]] via `POST /api/developer/agents/autonomy`. Autonomous is disabled until Live is on (an offline director can't auto-approve); the row reads "Approvals route here + log to history" when on, else "Approvals route to the CEO". The director's **live/autonomous badge** is now derived from these flags (seeded all-off ‚áí every director shows "routes to CEO" until the owner flips it). Below the header, its **three-tab inbox shell**:
  - **Messages** ‚Äî the Slack-style **#directors board** (M3 Phase 1, shipped): a workspace-wide team channel backed by [[../tables/director_messages]] (NOT the notification store), rendered by `<BoardChannel>` ([[../libraries/director-board]]) ‚Äî persona avatar + name/role per post, conversational body with @-mentions highlighted, threaded replies, kind badges. The live Platform director (M4) is the first real author; a system seed proves the surface. Two-way reply is M3 Phase 2.
  - **Approval Requests** (`agent_approval_request`) ‚Äî the routed approval queue. **Live (M2 Phase 2):** every open [[../tables/agent_jobs]] `needs_approval` is surfaced here by the box reconciler ([[../libraries/approval-inbox]]), routed to the resolved approver function ([[../libraries/approval-router]] ‚Äî first live+autonomous ancestor, else the CEO). Each request carries the agent's **investigation + proposed fix inline** (the `pending_actions[].summary`/`preview`/`cmd`) so the decision is one read; a single plain approve/decline action renders **Approve / Decline** buttons inline (‚Üí `POST /api/roadmap/approve`, the unchanged execution path ‚Üí worker flips `queued_resume`), while a multi-action / multi-choice job (coverage register-vs-exempt, hero preview) links out to its canonical surface. The request is auto-dismissed the moment its job leaves `needs_approval`.
  - **Daily Summaries** (`agent_daily_summary`) ‚Äî the EOD recaps (populated by M3/M4).
  - Each tab has **filters** (text + unread-only) + per-tab counts and an **empty/loading state**. The inbox queries the reserved `agent_*` types out of [[../tables/dashboard_notifications]] and buckets them by tab. **Approval routing:** a role sees only the approvals whose `metadata.routed_to_function` matches it (legacy/unrouted ‚áí the CEO); a director that is **not** an auto-approver captures nothing ‚Üí `routesToCeo:true` (empty + a "routes to the CEO inbox" notice). The CEO additionally carries the (not-yet-routed) Messages & Daily-Summaries.

## Data source

- `GET /api/developer/agents` (`src/app/api/developer/agents/route.ts`) ‚Üí `getOrgChart()` ([[../libraries/agent-personas|org-chart.ts]]) ‚Äî the CEO ‚Üí Directors ‚Üí Workers tree from [[../libraries/brain-roadmap]] `getFunctions()` + `getGoals()` + the Control Tower registry's `agent-kind` loops, with each director's `status`/`live`/`autonomous` derived from [[../tables/function_autonomy]] ([[../libraries/approval-router]] `loadAutonomyMap` + `isAutoApprover`). **Brain is the source of truth ‚Äî never a hand-maintained second copy of the org chart.**
- `POST /api/developer/agents/autonomy` (`src/app/api/developer/agents/autonomy/route.ts`) ‚Üí owner-gated upsert of a director's `live`/`autonomous` flag into [[../tables/function_autonomy]] (the progressive-offload switch behind [[../libraries/approval-router]] `resolveApprover`). Validates the slug against `functions/*.md`; `autonomous` is forced off when `live` is off.
- `GET /api/developer/agents/inbox?role={ceo|slug}` (`src/app/api/developer/agents/inbox/route.ts`) ‚Üí the role's Approval Requests + Daily Summaries payload. Reads [[../tables/dashboard_notifications]] (`type IN agent_*`, `dismissed=false`); **routes by the live flags (M2):** `routesToCeo` is true (empty) for a director that isn't an auto-approver, and each `agent_approval_request` is filtered to the role its `metadata.routed_to_function` names. Read-only ‚Äî the routing is done at emit time by the reconciler, not here.
- The routed Approval Requests are emitted by the **box reconciler** ([[../libraries/approval-inbox]] `reconcileApprovalInbox`, run each poll tick by `scripts/builder-worker.ts`) ‚Äî the single "one inbox, no orphans" sweep over every `needs_approval` job. The **decision** rides the unchanged `POST /api/roadmap/approve` execution path ([[../libraries/roadmap-actions|roadmap-actions.ts]] `approveRoadmapAction` ‚Üí `queued_resume`).
- `GET /api/developer/agents/board` (`src/app/api/developer/agents/board/route.ts`) ‚Üí the Messages tab's #directors channel ‚Äî `getDirectorBoard` + `threadMessages` ([[../libraries/director-board]]) over [[../tables/director_messages]]. Owner-gated, read-only (the two-way reply that writes back is M3 Phase 2). The Messages tab renders this for **every** role (one shared team channel), so the per-role `routesToCeo` notice never covers it.

## Permissions

Owner-only ‚Äî both the page (client `role` guard, mirrors [[control-tower]]) and every API (`workspace_members.role='owner'`, 403 otherwise).
