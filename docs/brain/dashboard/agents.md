# dashboard/agents

The owner-only **Agents hub** ([[../specs/agents-hub-role-inboxes]], M1 of [[../goals/devops-director]]) вЂ” the single surface that renders the live org chart (**CEO В· Directors В· Workers**) read from the brain, and gives every role the same **three-tab inbox** (Messages В· Approval Requests В· Daily Summaries). The foundation the routing engine (M2), the gamified board (M3), and the live Platform director (M4) all write *into*. Consolidates every approval into one routed surface (replacing the scattered Control Tower feeds, spec cards, the box `approvalHref`): platform-owned approvals route to the **live Platform/DevOps Director (Ada)** (`live + autonomous` since 2026-06-23, [[../tables/function_autonomy]]); the not-yet-automated directors' approvals still land in the **CEO inbox**.

**Route:** `/dashboard/agents` (client poller, owner-only)
**Sidebar:** **Developer** section (owner-only) в†’ **Agents** (right under [[control-tower|Control Tower]]).
**Sibling surface:** [[agents__scorecard|Platform scorecard]] вЂ” `/dashboard/agents/scorecard` ([[../specs/platform-scorecard-surface]] Phase 2, milestone (d) of [[../goals/platform-department-scorecard]]) вЂ” the daily/weekly/monthly KPI instrument panel, linked from the hub header.

## Surfaces

- **Active directive card** (M1 Phase 5 вЂ” [[../specs/director-executable-plans-and-priority-hub-pip]] вњ…) вЂ” rendered at the **top** of `/dashboard/agents` and on `/dashboard/agents/[director]` (e.g. Platform's profile) when a `director_function='platform'` row exists in [[../tables/director_directives]] with `status='active'`. Displays the plan summary, its ordered steps, and a "builds GATED until {spec_slug} ships" badge if `gate_builds_until` is set. Backed by `GET /api/director/directive?function=platform` (owner-gated, 403 for non-owner) + a **Clear** button (`POST /api/director/directive { id, action:"clear" }`, owner-gated) that flips the row to `cleared` and logs a `directive_cleared` row in [[../tables/director_activity]]. Silent-first-paint when no active directive exists вЂ” no empty-state box or flicker. Mirrors the standing-pass + daily-board headlines that already surface the directive, just with a dedicated, always-visible hub surface. Implemented in `src/components/agents/directive-card.tsx` via the [[../libraries/director-directives]] `clearDirective()` export. The director card is mounted on the hub main view + on every role profile ([[../libraries/director-directives]] `clearDirective` is owner-gated; the table and library already existed; this spec added only the UI rendering + the clear affordance).
- **Org chart / Inbox view toggle** (M1 Phase 4) вЂ” top-right of the hub, defaulting to the **Org chart** (the company team page). The org-chart view renders `<OrgTree>` (`src/components/agents/org-tree.tsx`) over the **same** `getOrgChart()` payload (no second copy of the org chart): a real tree вЂ” **CEO** at top в†’ a Directors rail (each director card carries its **SVG mascot + live/autonomous badge**) в†’ each director's **Workers** (its box [[../tables/agent_jobs]] lanes) as smaller persona cards beneath it. Every node is a **clickable** persona card в†’ that role's **profile detail page** `/dashboard/agents/[role]` (Phase 5 вЂ” the nodes are `<Link>`s; no node click is a dead end). The **Inbox** view is the left role nav + three-tab inbox below, and a deep-link `?view=inbox&role=вЂ¦` opens straight to a role's inbox (how a profile's **Open inbox в†’** lands).
- **Profile detail pages** (M1 Phase 5) вЂ” `/dashboard/agents/[role]` renders one page for **every seat** (`ceo` В· a director slug В· a worker kind), resolved off the same `getOrgChart()` payload: avatar + name + persona + a **tiered responsibilities list**. The **CEO**'s list = the `goals/*.md` company objectives (a **director-proposed** goal вЂ” [[../specs/director-proposed-goals]] Phase 2 вЂ” reads "вЏі Proposed by {Director} В· awaiting your greenlight"; a greenlit one reads "Greenlit В· N% complete", so the CEO sees what a director is proposing vs what's activated вЂ” `getOrgChart` carries `status`/`proposedBy`/`proposedByLabel` on each `ceo.goals` entry); a **director**'s = its `functions/*.md` mandates (metric + spec count) plus a clickable **Team** strip of its workers; a **worker**'s = the **precise `RESPONSIBILITIES` list** from [[../libraries/agent-personas|personas.ts]] (its exact mandate) + a "routes to the CEO until {director} goes live" note. An unknown role renders a graceful "No such role." card. Reachable from **any** surface вЂ” the org-tree nodes, the rosters below, and the inbox `RoleHeader`'s **View profile в†’** link вЂ” same profile page everywhere. Each profile's **Open inbox в†’** / **reports to вЂ¦** links thread back to the inbox + the parent's profile.
- **Directors + Workers rosters** (M1 Phase 5 вЂ” the top-level Agents nav items, flipped live) вЂ” `/dashboard/agents/directors` lists the CEO + every `functions/*.md` director as clickable rows (mascot + status badge + mandate/worker counts) в†’ their profiles; `/dashboard/agents/workers` lists every box `agent_jobs` lane grouped under its owning director, each chip в†’ its profile. Both brain-driven over `GET /api/developer/agents` вЂ” add a director / lane and it appears with no code change.
- **Left role nav** вЂ” **CEO** (the `ceo` persona seat, with its active-goal count from `goals/*.md`), then one row per `functions/*.md` **director** ([[../functions/platform|Platform]] В· [[../functions/growth|Growth]] В· [[../functions/cmo|CMO]] В· [[../functions/cs|CS]] В· [[../functions/retention|Retention]]) showing its **persona chip + SVG mascot** + a **live/autonomous badge** (M1: every director is `offline` в†’ "routes to CEO"; the flag itself lands in M2). Under each director, its **Workers** вЂ” the **reconciled roster** it owns ([[../archive]]): the `agent-kind` queue lanes (build/plan/fold/spec-chat/spec-test/dev-ask/pr-resolve/repair/regression в†’ Platform; ticket-improve/triage-escalations в†’ CS; migration-fix в†’ Retention; storefront-optimizer в†’ Growth) **plus** the persona-backed platform crons (Tao/monitor, Devi/db_health) **plus** any live unregistered lane (Cole/coverage-register), each with a live `active`/`idle-healthy`/`inactive` status badge.
- **Right pane** вЂ” the selected role's header (mascot + name + role + personality + mandates/goals). For a **director**, a gamified **XP card** (M3 Phase 3, shipped) atop its board channel вЂ” **specs shipped В· bugs fixed В· goals escorted В· streak** (`<XpCard>` over `GET /api/developer/agents/xp`, [[../libraries/director-xp]]); the same card renders on the director's profile page. Counts are **derived + display-only** (a gamified proxy, never an objective вЂ” [[../operational-rules]] В§ North star). For a **director**, an owner-only **Autonomy toggle** (approval-routing-engine M2 / Phase 1) вЂ” two checkboxes, **Live** (the director-agent is running) + **Autonomous** (trusted to auto-decide), backed by [[../tables/function_autonomy]] via `POST /api/developer/agents/autonomy`. Autonomous is disabled until Live is on (an offline director can't auto-approve); the row reads "Approvals route here + log to history" when on, else "Approvals route to the CEO". The director's **live/autonomous badge** is now derived from these flags (seeded all-off в‡’ every director shows "routes to CEO" until the owner flips it). Below the header, its **three-tab inbox shell**:
  - **Messages** вЂ” the Slack-style **#directors board** (M3 Phase 1, shipped): a workspace-wide team channel backed by [[../tables/director_messages]] (NOT the notification store), rendered by `<BoardChannel>` ([[../libraries/director-board]]) вЂ” persona avatar + name/role per post, conversational body with @-mentions highlighted, threaded replies, kind badges. The live Platform director (M4) is the first real author; a system seed proves the surface. Two-way reply is M3 Phase 2.
  - **Approval Requests** (`agent_approval_request`) вЂ” the routed approval queue, and (Phase 4) the **single source** where approvals are surfaced *and* decided. **Live (M2 Phase 2):** every open [[../tables/agent_jobs]] `needs_approval` is surfaced here by the box reconciler ([[../libraries/approval-inbox]]), routed to the resolved approver function ([[../libraries/approval-router]] вЂ” first live+autonomous ancestor, else the CEO). Each request carries the agent's **investigation + proposed fix inline** (the `pending_actions[].summary`/`preview`/`cmd`) so the decision is one read. **Phase 4 вЂ” multi-action/multi-branch inline:** `<ApprovalRow>` renders **every** still-pending plain action as its own sub-card with **Approve / Decline** (в†’ `POST /api/roadmap/approve` per action, the unchanged execution path в†’ worker flips `queued_resume` once all are decided) вЂ” so a multi-action `build` (migration + prod-script) and a **multi-branch `plan`** (each proposed spec branch, with its owner + parent) are resolved entirely here. The still-pending list is read **live** off the job (`inlineApproveActions`), not the emit-time snapshot, so a half-decided plan shows only the branches left. Only a genuinely **multi-CHOICE** job (coverage register-vs-exempt, hero reject-with-notes) carries no inline actions and **deep-links out** to its canonical surface ([[control-tower]] coverage section / the storefront optimizer). The request is auto-dismissed the moment its job leaves `needs_approval`. **The retired scattered surfaces** (Control Tower repair/db-health feeds, the spec-card [[roadmap|BuildButton/PlanButton]] cards, the box [[roadmap|`approvalHref`]]) no longer raise their own approval card вЂ” they `routedInboxHref()` **deep-link here** (one inbox, no orphans).
  - **Daily Summaries** (`agent_daily_summary`) вЂ” the EOD recaps (populated by M3/M4 вЂ” [[../libraries/director-recap]]). Each row **deep-links to the human-readable day narrative** ([[../specs/director-loop-grading]] Phase 5): `/dashboard/agents/recap/[date]` (`?function={slug}` for one director, omitted for the CEO roll-up) reads that day's [[../tables/director_activity]] rows via `GET /api/developer/agents/recap` and renders *what it fixed + why, which goal it moved, what it escalated* вЂ” a query over the log, never hand-maintained.
  - Each tab has **filters** (text + unread-only) + per-tab counts and an **empty/loading state**. The inbox queries the reserved `agent_*` types out of [[../tables/dashboard_notifications]] and buckets them by tab. **Approval routing:** a role sees only the approvals whose `metadata.routed_to_function` matches it (legacy/unrouted в‡’ the CEO); a director that is **not** an auto-approver captures nothing в†’ `routesToCeo:true` (empty + a "routes to the CEO inbox" notice). The CEO additionally carries the (not-yet-routed) Messages & Daily-Summaries.

## Data source

- `GET /api/developer/agents` (`src/app/api/developer/agents/route.ts`) в†’ `getOrgChart()` ([[../libraries/agent-personas|org-chart.ts]]) вЂ” the CEO в†’ Directors в†’ Workers tree from [[../libraries/brain-roadmap]] `getFunctions()` + `getGoals()` + the **reconciled worker roster** ([[../archive]]), with each director's `status`/`live`/`autonomous` derived from [[../tables/function_autonomy]] ([[../libraries/approval-router]] `loadAutonomyMap` + `isAutoApprover`). **Brain is the source of truth вЂ” never a hand-maintained second copy of the org chart.**
  - **Reconciled roster** (`buildRoster`) surfaces **100% of running agents** by unioning three sources, de-duped by persona key: (1) the `agent-kind` MONITORED_LOOPS (queue lanes), (2) the persona-backed `cron` loops вЂ” a [[../libraries/control-tower|registry]] `personaKind` (so `control-tower-monitor`в†’Tao + the two `db-health-*` cronsв†’one Devi show as workers, **invisible before**), and (3) any live `agent_jobs.kind` with rows in the last 7 days but no registry row (e.g. `coverage-register`в†’Cole) вЂ” `flagged:true`, surfaced under Platform, never silently dropped.
  - **Live status** (`computeWorkerStatus`) вЂ” each worker carries `active` (recent job/beat) В· `idle-healthy` (fired before / a beating cron, idle now) В· `inactive` (zero `agent_jobs` rows **ever** вЂ” e.g. Remi/regression). The inactive flag mirrors [[../libraries/control-tower]] `evalCron`'s never-fired grace (a `registeredAt`-within-grace lane and any cron-backed loop are never death-flagged), so an idle-but-healthy / freshly-shipped lane is never falsely called dead. Rendered via `WorkerStatusBadge` (`persona-chip.tsx`) on the org tree + workers roster, with an `unregistered` chip on flagged lanes.
- `POST /api/developer/agents/autonomy` (`src/app/api/developer/agents/autonomy/route.ts`) в†’ owner-gated upsert of a director's `live`/`autonomous` flag into [[../tables/function_autonomy]] (the progressive-offload switch behind [[../libraries/approval-router]] `resolveApprover`). Validates the slug against `functions/*.md`; `autonomous` is forced off when `live` is off.
- `GET /api/developer/agents/inbox?role={ceo|slug}` (`src/app/api/developer/agents/inbox/route.ts`) в†’ the role's Approval Requests + Daily Summaries payload. Reads [[../tables/dashboard_notifications]] (`type IN agent_*`, `dismissed=false`); **routes by the live flags (M2):** `routesToCeo` is true (empty) for a director that isn't an auto-approver, and each `agent_approval_request` is filtered to the role its `metadata.routed_to_function` names. Read-only вЂ” the routing is done at emit time by the reconciler, not here.
- The routed Approval Requests are emitted by the **box reconciler** ([[../libraries/approval-inbox]] `reconcileApprovalInbox`, run each poll tick by `scripts/builder-worker.ts`) вЂ” the single "one inbox, no orphans" sweep over every `needs_approval` job. The **decision** rides the unchanged `POST /api/roadmap/approve` execution path ([[../libraries/roadmap-actions|roadmap-actions.ts]] `approveRoadmapAction` в†’ `queued_resume`).
- `GET /api/developer/agents/recap?date=&function=` (`src/app/api/developer/agents/recap/route.ts`) в†’ the **human-readable EOD day narrative** ([[../libraries/director-recap]] `buildDirectorDayNarrative`) the Daily Summaries rows deep-link to. Owner-gated, read-only; a pure query over [[../tables/director_activity]], recomputed each view.
- `GET /api/developer/agents/xp` (`src/app/api/developer/agents/xp/route.ts`) в†’ `{ xp: { [functionSlug]: DirectorXp } }` ([[../libraries/director-xp]] `getDirectorXp`) вЂ” the four derived XP counts per director (merged builds owned В· approved repair/regression fixes В· shipped milestones in owned goals В· consecutive active days). Owner-gated, read-only, display-only.
- `GET /api/developer/agents/board` (`src/app/api/developer/agents/board/route.ts`) в†’ the Messages tab's #directors channel вЂ” `getDirectorBoard` + `threadMessages` ([[../libraries/director-board]]) over [[../tables/director_messages]]. Owner-gated, read-only (the two-way reply that writes back is M3 Phase 2). The Messages tab renders this for **every** role (one shared team channel), so the per-role `routesToCeo` notice never covers it.
