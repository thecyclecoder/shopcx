# dashboard/agents

The owner-only **Agents hub** ([[../specs/agents-hub-role-inboxes]], M1 of [[../goals/devops-director]]) тАФ the single surface that renders the live org chart (**CEO ┬╖ Directors ┬╖ Workers**) read from the brain, and gives every role the same **three-tab inbox** (Messages ┬╖ Approval Requests ┬╖ Daily Summaries). The foundation the routing engine (M2), the gamified board (M3), and the live Platform director (M4) all write *into*. Formalizes today's reality тАФ no director is automated, so **every approval routes to one CEO inbox** тАФ into one place, replacing the scattered approval surfaces (Control Tower feeds, spec cards, the box `approvalHref`).

**Route:** `/dashboard/agents` (client poller, owner-only)
**Sidebar:** **Developer** section (owner-only) тЖТ **Agents** (right under [[control-tower|Control Tower]]).

## Surfaces

- **Org chart / Inbox view toggle** (M1 Phase 4) тАФ top-right of the hub, defaulting to the **Org chart** (the company team page). The org-chart view renders `<OrgTree>` (`src/components/agents/org-tree.tsx`) over the **same** `getOrgChart()` payload (no second copy of the org chart): a real tree тАФ **CEO** at top тЖТ a Directors rail (each director card carries its **SVG mascot + live/autonomous badge**) тЖТ each director's **Workers** (its box [[../tables/agent_jobs]] lanes) as smaller persona cards beneath it. Every node is a **clickable** persona card тЖТ that role's **profile detail page** `/dashboard/agents/[role]` (Phase 5 тАФ the nodes are `<Link>`s; no node click is a dead end). The **Inbox** view is the left role nav + three-tab inbox below, and a deep-link `?view=inbox&role=тАж` opens straight to a role's inbox (how a profile's **Open inbox тЖТ** lands).
- **Profile detail pages** (M1 Phase 5) тАФ `/dashboard/agents/[role]` renders one page for **every seat** (`ceo` ┬╖ a director slug ┬╖ a worker kind), resolved off the same `getOrgChart()` payload: avatar + name + persona + a **tiered responsibilities list**. The **CEO**'s list = the `goals/*.md` company objectives (with % complete); a **director**'s = its `functions/*.md` mandates (metric + spec count) plus a clickable **Team** strip of its workers; a **worker**'s = the **precise `RESPONSIBILITIES` list** from [[../libraries/agent-personas|personas.ts]] (its exact mandate) + a "routes to the CEO until {director} goes live" note. An unknown role renders a graceful "No such role." card. Reachable from **any** surface тАФ the org-tree nodes, the rosters below, and the inbox `RoleHeader`'s **View profile тЖТ** link тАФ same profile page everywhere. Each profile's **Open inbox тЖТ** / **reports to тАж** links thread back to the inbox + the parent's profile.
- **Directors + Workers rosters** (M1 Phase 5 тАФ the top-level Agents nav items, flipped live) тАФ `/dashboard/agents/directors` lists the CEO + every `functions/*.md` director as clickable rows (mascot + status badge + mandate/worker counts) тЖТ their profiles; `/dashboard/agents/workers` lists every box `agent_jobs` lane grouped under its owning director, each chip тЖТ its profile. Both brain-driven over `GET /api/developer/agents` тАФ add a director / lane and it appears with no code change.
- **Left role nav** тАФ **CEO** (the `ceo` persona seat, with its active-goal count from `goals/*.md`), then one row per `functions/*.md` **director** ([[../functions/platform|Platform]] ┬╖ [[../functions/growth|Growth]] ┬╖ [[../functions/cmo|CMO]] ┬╖ [[../functions/cs|CS]] ┬╖ [[../functions/retention|Retention]]) showing its **persona chip + SVG mascot** + a **live/autonomous badge** (M1: every director is `offline` тЖТ "routes to CEO"; the flag itself lands in M2). Under each director, its **Workers** тАФ the box [[../tables/agent_jobs]] lanes it owns (the `agent-kind` loops in the Control Tower [[../libraries/control-tower|registry]], which already carry an owner function: build/plan/fold/spec-chat/spec-test/dev-ask/pr-resolve/repair тЖТ Platform; ticket-improve/triage-escalations тЖТ CS; migration-fix тЖТ Retention; storefront-optimizer тЖТ Growth).
- **Right pane** тАФ the selected role's header (mascot + name + role + personality + mandates/goals). For a **director**, a gamified **XP card** (M3 Phase 3, shipped) atop its board channel тАФ **specs shipped ┬╖ bugs fixed ┬╖ goals escorted ┬╖ streak** (`<XpCard>` over `GET /api/developer/agents/xp`, [[../libraries/director-xp]]); the same card renders on the director's profile page. Counts are **derived + display-only** (a gamified proxy, never an objective тАФ [[../operational-rules]] ┬з North star). For a **director**, an owner-only **Autonomy toggle** (approval-routing-engine M2 / Phase 1) тАФ two checkboxes, **Live** (the director-agent is running) + **Autonomous** (trusted to auto-decide), backed by [[../tables/function_autonomy]] via `POST /api/developer/agents/autonomy`. Autonomous is disabled until Live is on (an offline director can't auto-approve); the row reads "Approvals route here + log to history" when on, else "Approvals route to the CEO". The director's **live/autonomous badge** is now derived from these flags (seeded all-off тЗТ every director shows "routes to CEO" until the owner flips it). Below the header, its **three-tab inbox shell**:
  - **Messages** тАФ the Slack-style **#directors board** (M3 Phase 1, shipped): a workspace-wide team channel backed by [[../tables/director_messages]] (NOT the notification store), rendered by `<BoardChannel>` ([[../libraries/director-board]]) тАФ persona avatar + name/role per post, conversational body with @-mentions highlighted, threaded replies, kind badges. The live Platform director (M4) is the first real author; a system seed proves the surface. Two-way reply is M3 Phase 2.
  - **Approval Requests** (`agent_approval_request`) тАФ the routed approval queue, and (Phase 4) the **single source** where approvals are surfaced *and* decided. **Live (M2 Phase 2):** every open [[../tables/agent_jobs]] `needs_approval` is surfaced here by the box reconciler ([[../libraries/approval-inbox]]), routed to the resolved approver function ([[../libraries/approval-router]] тАФ first live+autonomous ancestor, else the CEO). Each request carries the agent's **investigation + proposed fix inline** (the `pending_actions[].summary`/`preview`/`cmd`) so the decision is one read. **Phase 4 тАФ multi-action/multi-branch inline:** `<ApprovalRow>` renders **every** still-pending plain action as its own sub-card with **Approve / Decline** (тЖТ `POST /api/roadmap/approve` per action, the unchanged execution path тЖТ worker flips `queued_resume` once all are decided) тАФ so a multi-action `build` (migration + prod-script) and a **multi-branch `plan`** (each proposed spec branch, with its owner + parent) are resolved entirely here. The still-pending list is read **live** off the job (`inlineApproveActions`), not the emit-time snapshot, so a half-decided plan shows only the branches left. Only a genuinely **multi-CHOICE** job (coverage register-vs-exempt, hero reject-with-notes) carries no inline actions and **deep-links out** to its canonical surface ([[control-tower]] coverage section / the storefront optimizer). The request is auto-dismissed the moment its job leaves `needs_approval`. **The retired scattered surfaces** (Control Tower repair/db-health feeds, the spec-card [[roadmap|BuildButton/PlanButton]] cards, the box [[roadmap|`approvalHref`]]) no longer raise their own approval card тАФ they `routedInboxHref()` **deep-link here** (one inbox, no orphans).
  - **Daily Summaries** (`agent_daily_summary`) тАФ the EOD recaps (populated by M3/M4).
  - Each tab has **filters** (text + unread-only) + per-tab counts and an **empty/loading state**. The inbox queries the reserved `agent_*` types out of [[../tables/dashboard_notifications]] and buckets them by tab. **Approval routing:** a role sees only the approvals whose `metadata.routed_to_function` matches it (legacy/unrouted тЗТ the CEO); a director that is **not** an auto-approver captures nothing тЖТ `routesToCeo:true` (empty + a "routes to the CEO inbox" notice). The CEO additionally carries the (not-yet-routed) Messages & Daily-Summaries.

## Data source

- `GET /api/developer/agents` (`src/app/api/developer/agents/route.ts`) тЖТ `getOrgChart()` ([[../libraries/agent-personas|org-chart.ts]]) тАФ the CEO тЖТ Directors тЖТ Workers tree from [[../libraries/brain-roadmap]] `getFunctions()` + `getGoals()` + the Control Tower registry's `agent-kind` loops, with each director's `status`/`live`/`autonomous` derived from [[../tables/function_autonomy]] ([[../libraries/approval-router]] `loadAutonomyMap` + `isAutoApprover`). **Brain is the source of truth тАФ never a hand-maintained second copy of the org chart.**
- `POST /api/developer/agents/autonomy` (`src/app/api/developer/agents/autonomy/route.ts`) тЖТ owner-gated upsert of a director's `live`/`autonomous` flag into [[../tables/function_autonomy]] (the progressive-offload switch behind [[../libraries/approval-router]] `resolveApprover`). Validates the slug against `functions/*.md`; `autonomous` is forced off when `live` is off.
- `GET /api/developer/agents/inbox?role={ceo|slug}` (`src/app/api/developer/agents/inbox/route.ts`) тЖТ the role's Approval Requests + Daily Summaries payload. Reads [[../tables/dashboard_notifications]] (`type IN agent_*`, `dismissed=false`); **routes by the live flags (M2):** `routesToCeo` is true (empty) for a director that isn't an auto-approver, and each `agent_approval_request` is filtered to the role its `metadata.routed_to_function` names. Read-only тАФ the routing is done at emit time by the reconciler, not here.
- The routed Approval Requests are emitted by the **box reconciler** ([[../libraries/approval-inbox]] `reconcileApprovalInbox`, run each poll tick by `scripts/builder-worker.ts`) тАФ the single "one inbox, no orphans" sweep over every `needs_approval` job. The **decision** rides the unchanged `POST /api/roadmap/approve` execution path ([[../libraries/roadmap-actions|roadmap-actions.ts]] `approveRoadmapAction` тЖТ `queued_resume`).
- `GET /api/developer/agents/xp` (`src/app/api/developer/agents/xp/route.ts`) тЖТ `{ xp: { [functionSlug]: DirectorXp } }` ([[../libraries/director-xp]] `getDirectorXp`) тАФ the four derived XP counts per director (merged builds owned ┬╖ approved repair/regression fixes ┬╖ shipped milestones in owned goals ┬╖ consecutive active days). Owner-gated, read-only, display-only.
- `GET /api/developer/agents/board` (`src/app/api/developer/agents/board/route.ts`) тЖТ the Messages tab's #directors channel тАФ `getDirectorBoard` + `threadMessages` ([[../libraries/director-board]]) over [[../tables/director_messages]]. Owner-gated, read-only (the two-way reply that writes back is M3 Phase 2). The Messages tab renders this for **every** role (one shared team channel), so the per-role `routesToCeo` notice never covers it.

## Permissions

Owner-only тАФ both the page (client `role` guard, mirrors [[control-tower]]) and every API (`workspace_members.role='owner'`, 403 otherwise).

## Reskinnable personas

The cast (ЁЯЫая╕П Ada/Platform ┬╖ ЁЯЪА Max/Growth ┬╖ ЁЯОи Iris/CMO ┬╖ ЁЯТм June/CS ┬╖ ЁЯз▓ Theo/Retention ┬╖ ЁЯСС CEO) is **one config file** тАФ [[../libraries/agent-personas]] (`src/lib/agents/personas.ts`). Names/mascots/colors are config keyed by function slug; a new director inherits a persona by adding one entry (and falls back to a neutral persona with no code change). The inline SVG mascots live in `src/components/agents/mascots.tsx`.
