# `src/lib/ads/creative-agent.ts`

The deterministic loop behind **Dahlia**, the Ad Creative Agent — a box lane, peer to [[media-buyer-agent|Bianca]] under Max ([[../functions/growth]]). She keeps Bianca's ready-to-test bin stocked with fresh, fully-backed static ads so the media-buyer test loop never starves for angles.

## The pipeline (per creative)
```
getProductIntelligence  →  selectAngles + buildCreativeBrief  →  generateCreative  →  QC gate  →  insert into bin
   [[product-intelligence]]     [[creative-brief]]              [[creative-generate]]  [[creative-qa]]
```
Every step is grounded so the creative can auto-publish with **no human gate**: the brief's claims trace to product-intelligence, the price is an allowed treatment (never bare MSRP), and the render passes a vision QC. On a QC fail it regenerates (up to `MAX_QA_ATTEMPTS`).

The QC gate has two paths (dahlia-creative-qc-via-box-session): the DEFAULT is `qaCreativeViaBoxSession` — a top-level `claude -p` on Max via the [[creative-qc|creative-qc]] skill, so the lane never needs an `ANTHROPIC_API_KEY`; the `DAHLIA_QC_MODE=direct` env kill-switch falls back to the legacy `qaCreative` Opus API call unchanged. Both paths **fail closed** — any session error / cap / timeout / undecodable image / unparseable verdict resolves to `pass:false` so nothing unchecked reaches the bin.

## `runAdCreativeLoop(admin, { workspaceId, productId?, count?, binFloor?, qcDispatcher?, copyAuthorDispatcher?, copyQcDispatcher?, intent?, pinnedCompetitorSkeletonId? }) → AdCreativeRunResult`
- With `productId` + `count`: tops that product up by `count` (capped at `MAX_PER_JOB`) — the shape the cadence cron sends. **Hero-product advertising gate ([[advertised-products]]):** the single `productId` is checked with `isAdvertisedProduct` first — an attachment-SKU target (stray job / manual trigger) yields ZERO creatives (empty `targets`).
- With no `productId`: finds every product with ad intelligence (a [[../tables/product_ad_angles]] row), **intersects with `listAdvertisedProductIds`** ([[advertised-products]]) so attachment SKUs (Tumbler, Sleep Gummies, Handheld Drink Mixer, Bamboo Coffee Mug) never earn Dahlia work, then measures each one's bin depth via [[ready-to-test]] `listReadyToTest` and tops up any below `DEFAULT_BIN_FLOOR` (4). Empty gate ⇒ zero targets, no fallback.
- `pinnedCompetitorSkeletonId?: string` — **Research › Ads "Generate ad like this" pin ("make one like THIS ad").** A `creative_skeletons.id` threaded to `stockProduct`; when it resolves (via [[creative-sourcing]] `getCompetitorAngleBySkeletonId`) it becomes the SOLE competitor angle — `stockProduct` skips `getProvenCompetitorAngles`, and BYPASSES `selectAnglesForTemperature`'s cold/warm exclusion + the retired filter (an explicit human pick overrides the auto-selection guards). The downstream composition-transfer + riff pipeline is unchanged, so the authored ad reuses that ad's layout adapted to our product. Only applied on the single-product path (`opts.productId` set) — a cadence top-up never pins. A missing/hook-less id falls back to normal shelf ranking + `console.warn('dahlia_pinned_skeleton_unusable')`. Set by [[builder-worker]] `runAdCreativeCopyAuthorJob` from the job's `competitor_skeleton_id` instruction ([[ad-creative-trigger]]).
- `qcDispatcher?: QcSessionDispatcher` — when set, each per-creative QC pass runs as a `claude -p` box session on Max via the caller's dispatcher ([[creative-qa]] `qaCreativeViaBoxSession`); when unset, `stockProduct` falls back to the direct-API `qaCreative`. [[builder-worker]] `runAdCreativeJob` reads `DAHLIA_QC_MODE` and injects the dispatcher iff mode `= "box"` (default) — mode `= "direct"` passes `undefined` so the legacy path runs unchanged.
- `copyQcDispatcher?: CopyQcSessionDispatcher` — **max-final-qa-7of10-eligibility-gate-with-bounce-to-dahlia Phase 1–3 (WIRED).** When set, `stockProduct` runs [[creative-qa]] `runQaCreativeCopyViaBoxSession` on every author-mode creative — AFTER Dahlia's copy-author session returns ok, BEFORE `insertReadyCreative`. The dispatcher spawns Max via [[builder-worker]] `runBoxLane` (kind `ad-creative-copy-qc`, `sandbox: "qc"` — no `ANTHROPIC_API_KEY`, minimal env, same `AD_CREATIVE_QC_ALLOWED_IMAGE` PreToolUse gate the image-QC / copy-author lanes use). Max grades the entire ad on parity access with Dahlia: same shared consumer-psychology rubric ([[../../../src/lib/ads/copy-rubric.ts]] `renderRubricForPrompt`), same `BRIEF` (the fully-backed product-intelligence surface Dahlia authored from), the same `AUDIENCE_TEMPERATURE`/`TARGET_SCHWARTZ_LEVEL`/`MARKET_SOPHISTICATION_EVIDENCE`, plus `DAHLIA_SELF_SCORE` for context (never mirror). The parsed `CopyQaVerdict` is persisted to [[../tables/ad_creative_copy_qc_verdicts]] via `insertCopyQaVerdict` after `insertReadyCreative` returns the campaign id (`retry_index: 0`). **Phase 2 (2026-07-18) — 7/10 eligibility gate WIRED:** the CEO's rule from the spec ("a creative is bin-eligible only if Max's hard gates pass AND his whole-ad score is at least 7/10; below 7 it is NOT eligible") is enforced by the pure exported predicate `isCopyQcEligible(verdict)` against the exported constant `MAX_QC_ELIGIBILITY_FLOOR = 7` — eligible IFF the verdict exists AND `hard_gate_pass` AND `persuasion_score >= 7`. Scroll-stop sub-scores are DELIBERATELY ignored (advisory-only Goodhart guard — only the top-line score gates). A non-eligible verdict (hard-gate fail, sub-7 score, parse error, dispatch error) does NOT reach `insertReadyCreative` — instead `stockProduct` records a growth `director_activity` row `action_kind='max_qc_below_floor_hold'` carrying the score / hard-gates / bounce reason, and the StockedCreative row surfaces the distinct `max_qc_below_floor: <reason>` string for the batch summary. **Selection = explore/exploit slot allocation** ([[creative-learning]]): the bin is stocked as a **2 exploit / 2 explore** mix so Bianca always has both to launch. **Exploit** = a fresh *combination* (new treatment) of a proven WINNING concept — double down on what converts without re-running the fatiguing ad. **Explore** = a fresh unproven concept — find the next winner before the current fatigues. Self-adjusts: with no winners yet it's all explore. A concept is only dropped once **retired** (3 distinct combinations lost with none won) — never after one loss. Transformation stories are scanned directly from [[../tables/product_reviews]].

**Explore/exploit provenance badge tells the CROWN-GATED truth (ad-creative-box-session-only-retire-deterministic-path-and-honest-explore-exploit Phase 4 — WIRED, 2026-07-19):** `buildAngleProvenance(angle, intent)` now takes the planner's per-slot intent and sets `provenance.mode = intent` — the crown-gated truth `stockProduct`'s planner already computes (`wantExploit = Math.min(Math.floor(count/2), exploitPool.length)` where `exploitPool = eligible.filter(isWon)`, so **0 crowns ⇒ 0 exploit slots ⇒ every ad in the batch badges `explore`**). Pre-Phase-4 the badge was `isCompetitor ? 'explore' : 'exploit'`, so every own-brand angle was labeled EXPLOIT even when there were zero crowned winners anywhere — the badge silently lied against the same crown-gated selection code sitting one function above. Now an own-brand angle in an explore slot badges `explore`, and the rare re-crowned competitor imitation in an exploit slot badges `exploit`. Competitor-only fields (`competitor_advertiser` / `competitor_ad_image_url` / `competitor_hook`) stay gated on `source === 'competitor'` — those are genuinely source-specific and NOT intent-derived. The intent is threaded through `insertReadyCreative`'s new `intent?: 'explore' | 'exploit'` opt from both call sites (the main postable insert AND the Max-exhaustion always-bin ineligible insert), so both dispositions carry the honest badge; `insertReadyCreative` defaults to `'explore'` when a caller (test / bench replay) does not thread one, matching the planner's fallback `if (!plan.length) … intent:'explore'`. The persisted `ad_campaigns.metadata.provenance.mode` therefore ALWAYS agrees with the crown-gated intent, and the future `src/lib/ads/ads-read-sdk.ts` `deriveExploreExploit` read chokepoint reads the same truth. Pinned by 5 tests in [[../../../src/lib/ads/creative-agent.test.ts]] (competitor-explore slot; competitor-hook fallback; own-brand explore slot badges explore — the anti-regression of the pre-Phase-4 EXPLOIT lie; own-brand exploit slot badges exploit; competitor exploit slot badges exploit while still carrying source-specific fields).

**Imitate-first explores (CEO 2026-07-12):** the explore pool is sorted **competitor-angle FIRST** (`source==='competitor'`, from [[creative-sourcing]] `getProvenCompetitorAngles(productId)` — the product's own scouted [[../inngest/creative-scout]] shelf) before our own unproven concepts. This is Dylan's imitate→innovate flow: "she goes to the scouted ads for that competitor list and finds great examples to explore." Without it, the explore slots defaulted to our own weight-loss transformation stories and never imitated the product's deliberately-chosen competitors. Exploit (proven own-brand winners) is unchanged — so a product with wins gets 2 exploit (own) + 2 explore (competitor imitations).

## Bin insertion (mirrors `/api/ads/upload-static`)

**Phase 2 — Dahlia now authors the finished 3-placement multi-copy pack ([[creative-pack]])** (`stockProduct` renders 3 placements from one brief via [[creative-generate]], calls `buildMetaCopyPack` to produce 4 headlines + 4 primary texts, and uses `planCreativePackInserts` to structure the 3-placement DB inserts — feed 4:5 canonical + stories/reels 9:16 + right-column 1:1 siblings linked via `format_variant_of_id`). `insertReadyCreative` stamps `format_variant_of_id` on siblings after the canonical ad_videos insert returns, expressing the same-psychology invariant.

**Ad name composed as `Dahlia - {productTitle} - {sanitizedHeadline}` (dahlia-names-each-ad-from-its-headline-minus-weight-loss-not-a-generic-template Phase 1 — WIRED):** `ad_campaigns.name` is composed as `` `Dahlia - ${productTitle} - ${deriveAdName(copyPack.headlines[0], competitorTokens, conceptTagFallback)}` `` instead of the old generic `Dahlia · {productTitle} · {angle.source}` template that made every row indistinguishable in the bin / Ads Manager AND leaked the `competitor` source label on every explore row. The `Dahlia` owner prefix + product name are kept (CEO's rule — owner + product must stay identifiable) and the trailing slot becomes the creative's own canonical headline, so names become unique per ad automatically. Sanitizer rules for the trailing slot (same rule set the [[../libraries/copy-validator|copy-validator]]'s no-competitor-leak rail uses — whitespace-split, ≥3 chars, drop the shared product-noun allowlist): (a) any `weight loss` / `weightloss` substring stripped case-insensitively (the CEO's rule, even when the headline itself carries it); (b) every competitor brand token + the literal word `competitor` stripped as a whole word (manual boundary check so a token like `MUD/WTR` still matches); (c) whitespace + orphan leading/trailing punctuation collapsed; (d) if sanitizing empties the string, fall back to the caller-supplied `fallback` (author-mode passes the Andromeda `concept_tag` label like `transformation` / `objection` / `mechanism`; deterministic-mode passes the safe default `"ad"`) so the composite name never ends with a blank trailing slot. Competitor tokens are derived at the call site from `angle.raw.advertiser` on `angle.source==='competitor'` angles (whitespace-split); own-brand angles pass `[]`. Pinned by 15 unit tests in [[../../../src/lib/ads/creative-agent.derive-ad-name.test.ts]] (weight-loss stripping / `weightloss` no-space / clean passthrough / composite-shape assertions for each rule / competitor-brand strip / literal `competitor` strip / sanitized-empty-fallback to a concept tag / deterministic-mode `"ad"` fallback / multi-token advertiser / slash-carrying token / whitespace + orphan-separator collapse / `Dahlia - {product} -` prefix pin / never-middot regression).

**Phase 1 (original):** `insertReadyCreative` writes: a [[../tables/product_ad_angles]] row (`generated_by:'ad-creative-agent'`, with Meta copy from the brief) → an [[../tables/ad_campaigns]] row `status:'ready'` → a static [[../tables/ad_videos]] child (`media_kind:'static'`, `format:'feed_4x5'`), uploads the render to the private `ad-tool` bucket at `finals/{ws}/{video_id}.jpg` ([[ad-storage]]), stamps `static_jpg_url` + `status:'ready'`, and sets the campaign's `landing_url` to the **battle-tested Shopify PDP** `{shopify_primary_domain}/products/{handle}` (e.g. `superfoodscompany.com/products/superfood-tabs`; policy CEO 2026-07-10 — the in-house storefront / advertorial-variant landers are a LATER experiment tested only once a creative wins). Never the unreliable `shopify_domain` field. `listReadyToTest` then surfaces it to Bianca automatically.

**Angle-before-ready invariant (dahlia-creative-requires-angle-before-ready Phase 1):** a bin creative can only land at `status='ready'` when it carries an `angle_id`. The `product_ad_angles` insert is fallible (a race, RLS deny, or schema drift), so if `angleRow?.id` is null we route the campaign to `status='draft'` instead of a phantom 'ready' — expressed once as `readyStatusForAngle(angleId)` at the top of the file, greppable, and used at the ad-campaign insert.
