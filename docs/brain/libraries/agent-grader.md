# `src/lib/agents/agent-grader.ts` — the worker-action grader

The DevOps Director's **worker-action grade** — one row per graded **concluded `agent_jobs` row**, 1–10 + reasoning ([[../specs/worker-grading-and-director-management]] Phase 1; the devops-director goal "the org learns + self-manages"). One level **down the org chart** from [[director-grader]]: there the CEO grades the Director's calls; here the Director grades each WORKER's actions, and a slip in a worker's rollup triggers a coaching pass. Persists to [[../tables/agent_action_grades]]; calibrated by approved [[../tables/agent_grader_prompts]] rules; coaches via [[worker-instructions]] `coachAgent`.

The defining invariant (inherited from [[director-grader]]): **grade the work, not outcome luck** — a sound, well-scoped action that hit a rare reversible bump still grades well if the reasoning was right; a careless action that happened to land grades low. The grader is a SUPERVISED TOOL ([[../operational-rules]] § North star): it scores a bounded proxy (action quality); the Director owns the objective and the CEO overrides it.

## The gradeable unit
One **concluded `agent_jobs` row** (terminal status `completed｜merged｜failed｜needs_attention`) — a build merged, an error fixed/dismissed, an index proposed, a spec verified. **`merged` IS terminal** — a build goes `building → completed → merged` (PR landed on `main`; `SUCCESSFUL_BUILD_STATUSES` in [[agent-jobs]]), and the `build` rubric literally scores "PR merged clean", so a `merged` build is the canonical thing to grade. (Omitting `merged` from `TERMINAL_JOB_STATUSES` silently no-op'd every merged build for days — the same STARVED-grading root cause as [[director-grader]]; a post-merge finalization step that leaves the job at `merged` must not starve worker grading.) The job IS the worker's atomic action, so the director grader's polymorphic key collapses to a single FK (`agent_job_id`) + one rubric per worker `kind`.

## Per-worker rubrics
`AGENT_RUBRICS` (exported) maps each `agent_jobs.kind` → `{name, criteria}` (the spec's locked config): `build`→Bo (spec phases · `tsc` · clean merge), `repair`→Rafa (root-cause · fix held), `regression`→Remi, `db_health`→Devi, `spec-test`→Vera, `migration-fix`→Mira, `pr-resolve`→Pax, `fold`→Fenn, `coverage-register`→Cole, `monitor`→Tao, `plan`→Pia, `product-seed`→Sol, `spec-chat`→Sage, `dev-ask`→Dex. `GRADEABLE_KINDS` = its keys — only rubric-backed kinds are graded (the Director's own + non-worker kinds are excluded).

## Exports

| Symbol | Signature | Notes |
|---|---|---|
| `gradeAgentAction` | `({ agentJobId, admin? }) → Promise<AgentGradeResult>` | Grade ONE concluded job against its worker's rubric via the deployed Sonnet API path (log_tail + metadata only). Concluded-only (`not_concluded` until terminal — `completed｜merged｜failed｜needs_attention`), rubric-gated (`not_a_gradeable_worker`), idempotent on `agent_job_id`, never clobbers a human grade. **Fallback path** as of [[../specs/grading-cascade-to-box-sessions]] Phase 1 — the primary grade path is now the box-hosted [[#The box-hosted grading lane (Phase 1)]]. |
| `applyBoxGrade` | `({ agentJobId, grade, reasoning, admin? }) → Promise<AgentGradeResult>` | Persist a grade produced by the box's Max session (which read the REAL merged diff). Same UNIQUE(agent_job_id) upsert + human-override invariant as `gradeAgentAction`, so the rollup + coaching cascade fire identically off it. Stamps `model='box-max-session'`, `cost_cents=0` (Max sub — no per-token API bill). Called by `runAgentGradeJob` in [[../../scripts/builder-worker.ts]]. |
| `pickAgentGradeBatch` | `({ workspaceId, admin?, limit?, cap? }) → Promise<UngradedJob[]>` | Pick the ids the box-hosted grader will read this beat — same selection logic as the deployed sweep (failures first, then round-robin-by-worker sample of successes, proven workers throttled). Used by [[../inngest/platform-director-cron]] to construct the `agent-grade` box job's `instructions.agent_job_ids`. |
| `gradeConcludedAgentActions` | `({ workspaceId, admin?, limit?, cap? }) → Promise<{ considered, graded, gradedKinds }>` | The deployed-runtime batched grading pass. **Deprecated as the primary path** ([[../specs/grading-cascade-to-box-sessions]] Phase 1 — evidence-based grading needs the diff). Retained as a fallback the cron / an operator can invoke when the box lane is unavailable; retire once the box lane is proven green for a full rollup window. |
| `agentGradingBatchReady` | `(admin, workspaceId, now?) → Promise<{ ready, ungraded }>` | The batched-cadence gate: ready when ≥`BATCH_MIN` (5) ungraded concluded jobs OR the oldest ungraded is >`BATCH_FALLBACK_MS` (~3h). Keeps the LLM spend to one session per batch. |
| `computeAgentRollup` | `(admin, workspaceId, agentKind) → Promise<AgentRollup>` | The standing score: last-`ROLLUP_WINDOW` (10) average + `priorAverage` (jobs 11–20) + `drop` (prior − current). |
| `detectGradeDropCoaching` | `({ workspaceId, agentKind, admin? }) → Promise<CoachingTriggerResult>` | The coaching trigger: slip = avg `< COACH_LOW_ROLLUP` (≥3 grades) OR `drop > DROP_THRESHOLD`; on a slip, **enqueues an `agent-coach` box job** (grading-cascade-to-box-sessions Phase 2 — the LLM synthesis moved to a Max session that reads the REAL diffs). Loop-guarded via inline `rollCoachingIntoFixSpec`. Dedup: skips re-enqueue while an `agent-coach` job for this `(workspace, agent_kind)` is already queued/building. `coached=true` now means the box coach lane has been dispatched. |
| `applyBoxCoaching` | `({ workspaceId, agentKind, learning, sourceGradeId?, admin? }) → Promise<{ ok, coached?, attempt?, instructionId?, reason? }>` | Persist a coaching learning produced by the box's Max coach session. Re-checks rollup + loop-guard (a slip may have recovered while the coach job queued; open coachings may have crossed the max and need `rollCoachingIntoFixSpec` instead), then calls the unchanged `coachAgent` (Director-gated DB write) + announces on `#directors`. Called by `runAgentCoachJob` in [[../../scripts/builder-worker.ts]]. |
| `CoachingLearning` | `{ errorClass, triggeringPattern, guidance, reasoning }` | The durable rule shape the box coach session produces — kebab-case class + 1-sentence pattern + "when you see X, do Y" guidance + evidence-grounded reasoning citing `path.ts:LINE` from the merged diffs. |
| `buildAgentGraderSystemPrompt` | `(admin, workspaceId, agentKind) → Promise<string>` | The worker's rubric + approved [[../tables/agent_grader_prompts]] rules (worker-targeted + cross-cutting NULL). |

**Constants:** `ROLLUP_WINDOW=10` · `COACH_LOW_ROLLUP=7` · `DROP_THRESHOLD=1.5` · `BATCH_MIN=5` · `BATCH_FALLBACK_MS=~3h` · `GRADE_BATCH_CAP=12`.

**Bounded, prioritized cadence (CEO directive — don't jam the box).** A grading session grades **at most `GRADE_BATCH_CAP`** jobs. Selection (`selectGradingBatch`): **every failure** (`failed`/`needs_attention` — a worker mistake worth coaching on, newest first) up to the cap, then the remaining slots filled by a **round-robin-by-worker** random sample of the successes — so noisy routine workers (fold, pr-resolve) get spot-checked without crowding out quieter ones, and the rest stay ungraded for a later beat. A failure also makes `agentGradingBatchReady` true immediately (graded promptly, not on the ~3h fallback).

## How it grades
- **Inputs:** the concluded job's row — `kind` / `spec_slug` / `status` / `error` / `log_tail` / `pr_url` / the approved `pending_action`. (Phase 1 grades from the job-row context the runtime has; the Phase-2 Max box session that reads the real PR diff is the cadence layer.)
- **Model:** Sonnet ([[ai-models]]) for the fallback `gradeConcludedAgentActions` API path. Cost via [[ai-usage]] (`purpose='agent_action_grading'`). Both the box-hosted primary grader (Phase 1) and the box-hosted coach (Phase 2) run on Max — no per-token API bill; `purpose='agent_coaching_synthesis'` rows are retired.
- **Output (strict JSON):** `{ grade, reasoning }`, 1–10. The grade weights judgment-within-the-rubric over outcome luck.

## The coaching cascade
`detectGradeDropCoaching` computes the rollup; on a slip it pulls the worker's recent low-graded actions and **enqueues an `agent-coach` box job** ([[../specs/grading-cascade-to-box-sessions]] Phase 2 — the LLM synthesis moved off the deployed API to a Max session that reads the REAL merged diffs). The box lane (`runAgentCoachJob` in [[../../scripts/builder-worker.ts]]) distills ONE durable learning (`CoachingLearning = {errorClass, triggeringPattern, guidance, reasoning}`) grounded in `git show <merge_sha>` output, then calls `applyBoxCoaching` — which re-checks rollup + loop-guard and delegates the actual write to the unchanged [[worker-instructions]] `coachAgent` (`coachedBy=PLATFORM` — the Director is the gate, a worker can't coach itself). That amends [[../tables/agent_instructions]] and logs the director→worker message in [[../tables/agent_coaching_log]]. Loop-guard: ≥`PLATFORM_DIRECTOR_LOOP_GUARD_MAX` coaching attempts that never `stuck` → `rollCoachingIntoFixSpec` (deterministic; unchanged from Phase 1) — never a CEO escalation.

## Where it's wired
- **Phase 1 ✅:** the store + grader library + the rollup/coaching trigger.
- **Phase 2 ✅:** the batched cadence — [[../inngest/platform-director-cron]]'s `grade-and-coach-workers` step runs `agentGradingBatchReady` (gate: ≥5 ungraded OR oldest >~3h) → **enqueues an `agent-grade` box job** (grading moved box-side, see below) → `detectGradeDropCoaching` per rubric-backed worker (idempotent + loop-guarded; catches slips landed by the box job within one beat), on the same `*/5` beat as the director grade sweep. Best-effort + idempotent. The step's `{considered,graded}` feeds the same cron's grading-liveness beat (`loop_id='director-decision-grading'`) so a starved worker-grading sweep is observable, not silent — `graded` here counts enqueued box jobs (a real backlog with a broken box lane still trips starvation after ≥2 consecutive sweeps).
- **Phase 3 ✅:** each worker profile (`/dashboard/agents/[role]`) gets a **rollup-grade card** (last-10 avg + trend) + a **graded-actions feed** (recent concluded jobs each with its grade) via `AgentGradePanel` ← `GET /api/developer/agents/agent-grades?kind=…` (owner-gated: `computeAgentRollup` + the recent concluded `agent_jobs` joined to their grades).

## The box-hosted coaching lane (Phase 2)

[[../specs/grading-cascade-to-box-sessions]] Phase 2 (same CEO directive 2026-06-30): the coaching **synthesis** LLM call also runs box-side on Max, so it reads the low-graded jobs' REAL merged diffs — not the paraphrased `agent_action_grades.reasoning` the deployed `synthesizeCoaching` used (which was itself paraphrased from the `log_tail`, coaching a self-report-of-a-self-report three layers removed from the actual code).

**Wiring:**
- New `agent_jobs.kind='agent-coach'` box lane in [[../../scripts/builder-worker.ts]] (`runAgentCoachJob` + `runAgentCoachClaude`, concurrency-1, `sandbox:'max'`, `ANTHROPIC_API_KEY` stripped). Skill: `.claude/skills/agent-coach/SKILL.md`.
- `detectGradeDropCoaching` on slip → ENQUEUES ONE `agent-coach` row with `instructions.agent_kind + low_grade_ids + low_agent_job_ids + rollup_average + rollup_drop + open_coaching_count` (dedup: skip re-enqueue while an `agent-coach` job for this `(workspace, agent_kind)` is already `queued｜queued_resume｜claimed｜building｜needs_input`). The recovery-announce + loop-guard (`rollCoachingIntoFixSpec`) paths remain deterministic and inline.
- The box lane fetches the low grades + their `agent_jobs` rows and asks the session to `git fetch origin main` + `git show <merge_sha>` (or `git diff origin/main...origin/<branch>`) each diff, spot the recurring code mistake, and emit a single-object verdict `{status, errorClass, triggeringPattern, guidance, reasoning}`.
- The learning lands via `applyBoxCoaching` — which re-verifies the rollup (`recovered_before_coach` no-op) + the loop-guard (`rolled_into_fix_spec` handoff), then calls the unchanged `coachAgent` + announces `🛠️ Ada coached <Worker>: <guidance>` on `#directors`. `agent_coaching_log` + `agent_instructions` versioning + `attempt` counting are byte-for-byte the same as the Phase-1 deployed path — only the LLM source moved.
- `synthesizeCoaching` (the deployed API call in `agent-grader.ts`) is fully **removed**. `ai_token_usage` rows with `purpose='agent_coaching_synthesis'` stop accruing.

## The box-hosted grading lane (Phase 1)

[[../specs/grading-cascade-to-box-sessions]] Phase 1 (CEO directive 2026-06-30): every grader that emits a grade runs box-side on Max. The deployed Sonnet sweep only saw the job-row metadata + `log_tail` — so it graded the worker's SELF-REPORT and capped itself at 7-9 explicitly "because I can't see the diff" (visible in `agent_action_grades.reasoning`). Grading compute moved to a Max `claude -p` session that reads the REAL diff (`git show <merge_sha>` / `git diff origin/main...origin/<branch>`), Reads the touched files, runs `npx tsc --noEmit` if in doubt, and checks `gh` CI status.

**Wiring:**
- New `agent_jobs.kind='agent-grade'` box lane in [[../../scripts/builder-worker.ts]] (`runAgentGradeJob` + `runAgentGradeClaude`, concurrency-1, `sandbox:'max'`, `ANTHROPIC_API_KEY` stripped). Skill: `.claude/skills/agent-grade/SKILL.md`.
- [[../inngest/platform-director-cron]] `grade-and-coach-workers` step: picks the batch via `pickAgentGradeBatch` (same selection as the deployed sweep — failures first, round-robin sample, proven workers throttled), enqueues one `agent-grade` row per batch-ready workspace with `instructions.agent_job_ids=[…]`. Dedup: skip re-enqueueing while an `agent-grade` job for this workspace is already `queued｜queued_resume｜building｜claimed` — one batched box session at a time.
- The box lane fetches each id, builds a prompt with the per-kind rubrics + approved [[../tables/agent_grader_prompts]] calibration rules + a compact pointer block per job (kind / spec / status / PR / branch / gated action / log_tail), and asks the session to emit `{decisions:[{agent_job_id, grade, reasoning}]}`. Grades land via `applyBoxGrade` — `model='box-max-session'`, `cost_cents=0`, same UNIQUE(agent_job_id) upsert + human-override invariant.
- After the batch writes, `runAgentGradeJob` fires `detectGradeDropCoaching` per newly graded worker kind — the rollup + `coachAgent` + eventual `rollCoachingIntoFixSpec` chain is unchanged; only the grade source moved.
- `gradeConcludedAgentActions` (the deployed API sweep) stays in the codebase as a **fallback** — an operator can invoke it manually if the box lane is offline. Retire once the box lane is proven green for a full rollup window.

## Gotchas
- **The ungraded-pool window must track the RECENT tail, not the OLDEST head** (fix-starved-grading round 2 — the recurring "she doesn't grade her agents at all"). `ungradedConcludedJobs` fetches gradeable+terminal jobs and subtracts the already-graded set. It MUST `order('created_at', { ascending: false }).limit(N)` (newest N), then re-sort ascending for the batch-ready gate. Ordering **ascending** clipped the window to the OLDEST N jobs — which are 100% already-graded once grading has ever run — so the `considered`/`ungraded` count was permanently ~0 while hundreds of fresh concluded jobs sat ungraded (the heartbeat read `considered:0` = "nothing to grade", not starvation, so it never even paged). Symptom in prod: 2972 gradeable-kind+terminal jobs all-time, the oldest 1000 all graded → `ungraded:1` despite a ~1000-job real backlog. After the fix: `ungraded:1000`, and every `*/5` beat drains 12 (`GRADE_BATCH_CAP`).
- **The already-graded set must be PAGINATED** — PostgREST caps a single response at 1000 rows regardless of `.limit()`, so the graded-job set (one row per ever-graded job, thousands over time) is fetched with a `range(from, from+999)` loop. A truncated graded set would falsely re-surface old graded jobs as ungraded and re-spend the LLM.
- **A failed grade attempt is logged LOUD — but in-flight skips stay silent.** In `gradeConcludedAgentActions`, a `gradeAgentAction` result with `!r.ok` is `console.error`'d ONLY when `r.reason` is NOT in `AGENT_INFLIGHT_SKIP_REASONS` (`not_concluded` · `job_not_found`). Those two are benign TOCTOU races with director triage (the director re-queues a parked job between the sweep snapshot and the grader's re-fetch — `needs_attention → queued`) and would otherwise light up the Control Tower / Vercel error feed as if the grader were broken. True grader errors (`grader_http_429` · `parse_failed` · `no_api_key` · `not_a_gradeable_worker`) still page — mirrors [[director-grader]]'s `INFLIGHT_SKIP_REASONS` / `isInflightSkip` pattern.
- **Supervised tool** ([[../operational-rules]] § North star) — scores a bounded proxy (action quality); the Director owns the objective, the CEO overrides. The grade only recommends a coaching pass, never a leash change.
- **Concluded-only + rubric-gated** — an in-flight job returns `not_concluded`; a non-worker / Director kind returns `not_a_gradeable_worker`.
- **Idempotent + human-safe** — keyed on the `agent_job_id` unique; a re-run UPDATEs in place, and `graded_by='human'` is never re-written.
- **Best-effort, never blocking** — the sweep is wrapped so a grader failure never breaks the caller; a missing `ANTHROPIC_API_KEY` is a clean no-op.
