# libraries/author-spec

The DB-only chokepoint every spec-AUTHOR surface (planner, director-coach, triage, regression, spec-chat, db-health, coverage-register, repair, security, migration-fix, storefront-optimizer, developer-message-center, director split/bounce-back) routes through to write the spec BODY to [[../tables/specs]] + [[../tables/spec_phases]]. There is NO `docs/brain/specs/{slug}.md` commit ŌĆö the per-spec markdown was retired ([[../specs/spec-readers-from-db-retire-parser]] ┬Ę spec-pm-markdown-purge); the readers read the DB rows.

**File:** `src/lib/author-spec.ts`

## Why this exists

[[../specs/spec-body-table-and-backfill]] M1 stood up [[../tables/specs]] + [[../tables/spec_phases]] and the [[specs-table]] writer. This module is the single authoring chokepoint that EVERY author surface routes through to write the spec row + its phases ŌĆö so the Verification-enforcement gate (no untestable spec) and the `upsertSpec` write discipline live in ONE place. Post-cutover the DB row is the sole source of truth; nothing commits a `.md`.

## Exports

- **`authorSpecRowFromMarkdown(workspaceId, slug, markdown, intendedStatus, opts?)`** ŌåÆ `Promise<boolean>` ŌĆö author from a MARKDOWN body: parses it via [[brain-roadmap]] `parseSpec` + the local `extractPhaseBodies`, extracts the typed regression headers + repair signature, and calls [[specs-table]] `upsertSpec` to UPSERT [[../tables/specs]] + REPLACE [[../tables/spec_phases]] by `(spec_id, position)`. The Verification gate (`assertEveryPhaseHasVerification`) runs FIRST and THROWS `MissingVerificationError` on an untestable phase; a genuine DB error is best-effort (`false`). `opts.milestoneId` binds `specs.milestone_id` at author time. **pm-structured-intent-and-refs Phase 1** ŌĆö `extractIntentHeaders` pulls `**Why:** ŌĆ”` / `**What:** ŌĆ”` from the markdown when present; missing headers land as NULL with a soft warn (surfaces migrate incrementally to emit them).
- **`authorSpecRowStructured(workspaceId, slug, spec, intendedStatus, opts?)`** ŌåÆ `Promise<boolean>` ŌĆö author from ALREADY-TYPED fields + phases (NO markdown parse). Same Verification gate + same `upsertSpec` write path. This is the DB-driven entry point the **goal planner** uses: it holds the proposed spec as structured data (`StructuredSpecInput` ŌĆö title/summary/owner/parent/blocked_by + **REQUIRED `why` + `what`** + `StructuredPhaseInput[]` each carrying a REQUIRED non-empty `verification` + `why` + `what` + optional `checks:[{position,description,kind:'auto'|'human'}]`) and never needs a `.md` scratch buffer on disk. `opts.milestoneId` binds the goalŌåÆmilestoneŌåÆspec link. Removes the disk-`.md`-round-trip that authored 0 specs (planner-authors-specs-to-db). **pm-structured-intent-and-refs Phase 1ŌĆō3 gates run before the DB write:** `assertEveryNodeHasIntent` (plain-language why/what non-empty on spec + every phase, with a lint rejecting code fences / `file:line` refs / `**Header:**` lines) and `assertEveryPhaseHasChecks` (Ōēź1 structured `spec_phase_check` row per phase ŌĆö parsed from `verification` when the caller doesn't pass an explicit `checks` array). Both throw the same fail-loud shape as `assertEveryPhaseHasVerification`. AFTER a successful `upsertSpec`, the writer persists structured brain refs via [[spec-brain-refs-table]] `replaceSpecBrainRefs` (from `opts.brainRefs` or the derived suggester) and per-phase check rows via [[spec-phase-checks-table]] `upsertPhaseChecks`.
- **`auto_build` default = ON (`!== false`).** Both entry points write `auto_build: <input>.autoBuild !== false`, so an autonomously-authored spec **auto-builds by default** ŌĆö only an EXPLICIT `autoBuild: false` parks it. This HONORS the [[brain-roadmap]] `parseSpec` contract for the markdown path (*"`**Auto-build:**` absent = on; only off/no/false/manual/disabled flips it false"*): `card.autoBuild` is `undefined` when no line is present, which means "on". **Prior bug (`=== true`):** a spec with no `**Auto-build:**` line (the common case) or a structured caller that omitted `autoBuild` landed `auto_build=false`, which silently PARKED **every** autonomously-authored spec ŌĆö Pia's goal decomposition, spec-chat, repair/security/director-authored ŌĆö so a greenlit goal's decomposed specs never built (`noop-goal-test` had to have it flipped by hand). The only deliberate opt-outs (`request-fix` route + `pre-merge-fix`) set `autoBuild: false` **explicitly** and are unaffected by the default flip ŌĆö a fix spec still needs commission.
- **`extractPhaseBodies(raw)`** ŌĆö per-phase body extractor (sibling to `parseSpec` which only captures title + status). Position is 1-indexed and lines up with `parseSpec().phases[i]`. Shared with [[../recipes/backfill-specs-from-markdown]] to keep one impl.
- **`extractRepairSignature(raw)`** ŌĆö the SIGNATURE TEXT (not just presence) for a Repair-Agent-authored spec ŌĆö `**Repair-signature:** ``<sig>```. Pulls into `specs.repair_signature`.
- **`extractRegressionHeaders(raw)`** ŌåÆ `{ ofSlug, signature }` ŌĆö the regression-agent header lines `**Regression-of:** [[<slug>]]` + `**Regression-signature:** ``<sig>```. Pull into `specs.regression_of_slug` / `specs.regression_signature` (typed columns added in the [[../specs/spec-authoring-writes-db-and-worker-materialize]] Phase 1 additive migration).
- **`toDbStatus(s)`** ŌĆö map a [[brain-roadmap]] `SpecStatus` to a [[specs-table]] `SpecStatus` DB enum. `'rejected'` (a phase-only state) is rewritten to `'planned'`.
- **(internal, both entry points) `**Brain refs:**` suggester** ŌĆö [[../specs/spec-brain-refs]] Phase 2. After the Verification gate + before the `upsertSpec` write, [[brain-ref-suggest]] scans the body for the src/ files, tables, and existing brain wikilinks it names, resolves each to a `docs/brain/{libraries|inngest|tables|ŌĆ”}/{name}.md` verified on disk, and injects the top Ōēż4 as a `**Brain refs:**` line just below the last metadata header. Best-effort: nothing mappable ŌćÆ no line (Phase 1's grep-the-brain fallback covers it); a body already carrying a `**Brain refs:**` line (the author picked) is passed through unchanged. The line lives IN the summary text ([[build-spec-materializer]] Rendered shape) ŌĆö no new column, no SDK change. The [[../.claude/skills/spec-chat|spec-chat]] finalize instructions also ask the box to propose refs; this suggester is the deterministic safety net. **Skippable ([[../specs/fix-spec-brain-refs]]):** both entry points also short-circuit the suggester when the body carries either durable skip signal ŌĆö an empty `**Brain refs:**` header (no wikilinks after the colon) or an `<!-- brain-refs: skip -->` HTML comment marker anywhere on the scan surface. Both persist "author explicitly picked NONE" through re-authoring so a refine can skip refs without them being re-injected. See [[brain-ref-suggest]] `hasBrainRefsSkip`.
- **(internal) `reopenIfReauthoredAndChanged(workspaceId, slug, existing, next)`** ŌĆö the **re-author-re-opens-dismissed** root patch. Both entry points snapshot the PRE-upsert row (`getSpec`) and, AFTER the upsert, call this: if the spec EXISTED and its CONTENT CHANGED (title / summary / any phase title┬Ębody┬Ęverification differs, whitespace-normalized) it RE-OPENS the spec ŌĆö `[[spec-card-state]] markSpecCardBackToReview` resets the review signals (`vale_pass` / `vale_review_passed_at` / `ada_disposition` / `intended_status`) and sets `specs.status='in_review'`, and `[[director-activity]] clearDirectorSpecDismissals` clears the standing init/groom dismissal ledger. No-op for a brand-new spec, an identical/metadata-only re-author, or a folded spec. Best-effort (never fails the authoring write that already landed).

## Caller patterns

**Markdown-holding surfaces** (auto-fix lanes, spec-chat, director-coach) hand a markdown buffer to `markNewSpecInReview`:

```ts
await markNewSpecInReview(workspaceId, slug, "planned", actor, reason, markdown);
//                                                                       ^^^^^^^^ ŌĆö the scratch markdown body.
//   markNewSpecInReview wraps spec-card-state + authorSpecRowFromMarkdown; the .md is a buffer, never committed.
```

**The goal planner** authors DB-only from STRUCTURED data ŌĆö no markdown, no disk:

```ts
await markSpecCardForReview(workspaceId, slug, "planned", { actor: "planner", reason });
await authorSpecRowStructured(workspaceId, slug, { title, summary, owner, parent, blocked_by, phases }, "planned",
  { intendedStatusSetBy: "planner", milestoneId });
```

`runPlanJob`'s RESUME pass asks Pia for the spec bodies as STRUCTURED JSON (phases + a non-empty `verification` per phase), then authors each via `authorSpecRowStructured` BEFORE queuing any build ŌĆö so a build is never queued for a spec that wasn't authored. There is no `docs/brain/specs/*.md` write and no `git status` glob anywhere on this path.

**spec-chat authors DB-only (no `.md` commit).** The `runSpecChatJob` finalize/verify lanes (Sage's authoring chat) DON'T commit `docs/brain/specs/{slug}.md` to `main` ŌĆö under spec-pm-markdown-purge / retire-md-reads the spec lives ONLY in `public.specs` + `public.spec_phases`. Sage WRITES the spec markdown into the worktree as a **scratch buffer**; the worker reads it and authors the DB row via this chokepoint (`markNewSpecInReview` for a fresh spec, `authorSpecRowFromMarkdown` for a refine), then **validates by reading the row back** (`getSpec` ŌåÆ a spec row + Ōēź1 phase must exist) ŌĆö the old "box did not write a spec file" markdown-existence gate is gone. The finalize slug is derived from the buffer filename / the model's JSON hint and **guarded to a kebab slug** (a new-feature chat is keyed by a UUID; a UUID is never allowed to become the spec slug). For a refine/verify the worker first MATERIALIZES the existing row back to `docs/brain/specs/{slug}.md` (via [[build-spec-materializer]] `renderSpecRow`) so Sage can Read it as grounding ŌĆö read-then-author, never read-from-disk-canonical.

## Idempotency

Re-running `authorSpecRowFromMarkdown` with the same body is a no-op against the rolled-up `specs.status` and the phase set: `upsertSpec` UPSERTs by `(workspace_id, slug)` and REPLACES phases by `(spec_id, position)` preserving each phase's stable `id + pr + merge_sha + created_at`. A same-slug convergence (e.g. a recurring repair-agent signature) keeps the original card row's provenance intact.

## Gotchas

- **`owner` is normalized to a bare function slug.** Both entry points call `normalizeOwnerSlug` to strip any `[[../functions/ŌĆ”]]` wikilink wrapper before writing to `specs.owner` ŌĆö so an author can pass `"[[../functions/platform]]"` and the DB stores `"platform"`. This defensive normalizer prevents a regression where pre-merge-red authoring wrote mangled wikilink-wrapped owners ([[../specs/fix-pre-merge-red-owner-shape]]) and was added to every authoring surface.
- **`intendedStatus` is the author's SUGGESTION.** Vale's disposition lane reads the field; it's not binding. Default to `'planned'` (a bug fix you don't want built is a contradiction) unless the surface knows the author meant `'deferred'`.
- **Verification is a HARD gate, the DB write is best-effort.** A phase with no non-empty `## Verification` THROWS `MissingVerificationError` (caught by the planner loop ŌåÆ fails the job loudly). A genuine DB/upsert failure logs + returns `false`. Both entry points run the SAME gate.
- **DB-only ŌĆö no `.md`.** Nothing here commits `docs/brain/specs/{slug}.md`. The readers ([[brain-roadmap]] `getSpec`/`listSpecs`, the board, the build pipeline) read the DB rows.
- **Re-authoring a DISMISSED or already-reviewed spec RE-OPENS it (when content changed).** Ada's `init_dismissed` / a groom `groomed_dismissed` is a STANDING `director_activity` dedup row (`metadata.init_key`/`groom_key`) the init/groom lanes scan to SKIP a spec forever ŌĆö and a prior Vale pass leaves `vale_review_passed_at` stamped. If a spec is corrected-after-rejection but those stale signals stay, the corrected content sits DEAD under the old verdict (the `migration-pricing-preserved-base-above-msrp` gap). `reopenIfReauthoredAndChanged` (above) closes it: a content-changing re-author resets the review signals + flips to `in_review` (Vale re-reviews the NEW content, Ada re-disposes) AND clears the dismissal ledger. Same class as the orphan-park fixes. A NO-OP re-author (identical content) does NOT churn Vale.
