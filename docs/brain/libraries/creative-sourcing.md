# `src/lib/ads/creative-sourcing.ts`

The shared SDK for **where great ad ideas come from** + **how ads actually perform** (CEO 2026-07-11). Dahlia (sources angles), Bianca (reads signal), and Max (supervises) all call this one surface instead of re-deriving it from raw Meta/DB queries. It codifies the ad-hoc analysis that validated our signal model.

## The three idea pools + the analyzer
- **`getProvenCompetitorAngles(admin, ws, {minDaysRunning, productId, niche, limit})`** — the [[../tables/creative_skeletons]] competitor library, RANKED by `days_running` (longevity = a competitor is profitably scaling it = a validated angle). **Pass `productId`** (CEO 2026-07-12) → reads ONLY skeletons the [[../inngest/creative-scout]] tagged for THAT product's deliberately-chosen competitors (`creative_skeletons.product_id`) — the imitate→innovate path, a product imitates only its own shelf. **STATIC ONLY (`media_type='static'`, CEO 2026-07-17):** the query also filters to static ads — Dahlia makes STATIC creatives, so she imitates only static competitor ads (a video's composition / keyframe framing does not transfer to a static). The `status='analyzed'` filter alone leaked PROCESSED videos onto the shelf: a video is routed to `status='video_pending'`, then the video pipeline drains it to `status='analyzed'` with `media_type='video'` (25 such rows were on the shelf; all 206 `static/analyzed` rows ARE vision-tagged, so filtering to static also makes the shelf ~100% concept-tagged). Pinned by the video-exclusion case in `creative-sourcing.winner-tier.test.ts`. `niche` is the LEGACY substring filter (e.g. "coffee", "weight"), superseded by `productId` and ignored when `productId` is set. [[creative-agent|Dahlia]] `stockProduct` now passes `productId` (the coffee/weight niche hack is gone). **The strongest pool** — real market-validated hooks.
- **`getOurWinningAngles(admin, ws, metaAccountId, {maxCpaCents, minSpendCents})`** — our OWN best-performing ads ranked by cost-per-ATC then CPP — "what works for US", the **exploit seed** (the concepts to make variations of).
- **`analyzeAccountAds(token, metaAccountId, {datePreset})`** — the per-ad performance analyzer: spend, purchases, CPP, ATC, **cost-per-ATC**, **CPM**, CTR, reactions/saves/shares. Meta ground truth.
- *(Web DR-angle research — a future pool.)*

## The validated signal model (why these fields)
Proven on **99 historical ads** (45 winners, 24 losers): **cost-per-ATC** ($25 winners vs $79 losers) and **CPM** ($93 vs $148) discriminate winners. **CTR + reactions are TRAPS** — losers click 2× and react 7× *more* (clickbait). So the analyzer surfaces CTR/engagement for visibility but winners are chosen on cost-per-ATC + CPM. This is the same model [[meta-cpa-signal]] trims/crowns on.

## Per-ad "do not use" exclusion ([[../specs/flag-a-competitor-ad-do-not-use-manual-ceo-then-max-graded]] Phase 1)
`queryProvenAngles` adds `.eq("do_not_use", false)` to the `creative_skeletons` select so a competitor ad the CEO (Phase 2) or Max (Phase 3) has flagged as a weak imitation base is NEVER surfaced as an angle — even if it's a proven long-runner with a strong `winner_tier`. The signal exists because winner-tier / days-running can't tell a strong imitation base (Onnit "Lock in when it matters most" — hard hook + benefit stack + dynamic composition) from a lame one (Magic Mind display-box packshot — no hook, no benefit, static shelf shot) when BOTH have run forever. The flag is per-AD, not per-advertiser (a brand can hold both great and lame ads), and is preserved across the weekly scout re-observation by design (see [[creative-skeleton]] `ingestAd` upsert + `reobserveAd` — neither touches the `do_not_use` columns). Backed by [[../tables/creative_skeletons]] `do_not_use` / `do_not_use_reason` / `do_not_use_by` / `do_not_use_at` (migration `20261119120000`) + a partial index on `(workspace_id, product_id) WHERE do_not_use`.

## Shelf-derived sophistication signal ([[../specs/dahlia-five-frameworks-copy-skill]] M2)
[[sophistication|`src/lib/ads/sophistication.ts`]] exports **`computeSophisticationLevel(shelf: CompetitorAngle[]) → 1|2|3|4|5`** — a pure, no-I/O helper that reads the same `getProvenCompetitorAngles` shelf and returns the **modal Schwartz awareness level** the market is writing at (higher-wins tiebreak; empty shelf → `3`, safe solution-aware default). The token detector's five rule buckets — L2 problem tokens (`tired`/`crash`/`fog`/`stress`), L3 solution-category (`clean energy`/`focus support`), L4 mechanism (`adaptogen`/`L-theanine`/`ashwagandha`), L5 versus (`vs coffee`/`compared to`/`instead of`) — are pinned by `src/lib/ads/sophistication.test.ts` (`npm run test:sophistication`).

## Winner-tier + intent-scoped research ([[../specs/dahlia-researches-from-winners-flow-ad-library]] Phase 1)
`getProvenCompetitorAngles` now selects the **winners-flow longitudinal signals** — `winner_tier`, `winner_score`, `concept_tags` — and re-ranks the returned angles by **winner-tier rank (proven > building > new > retired) → winner_score → days_running**. Retired competitor ads sink to the tail even at high `days_running` (a killed ad is not a research base). Each returned `CompetitorAngle` carries the new fields (`winnerTier`, `winnerScore`, `conceptTags`) so downstream ([[creative-brief]] / [[creative-agent|Dahlia]] session / Max's Phase 2 grader) reads OUR unified breakdown (angle · archetype · why_it_works · cialdini_lever · awareness_stage · format) as the imitation rubric.

The **declared-intent envelope** — `CreativeIntent = { audience_temperature: 'cold'|'warm'|'hot', purpose: 'test-to-find-winner' }` — is passed as `opts.intent`. When set, angles whose `concept_tags.awareness_stage` matches the temperature bucket (**cold → unaware / problem_aware · warm → solution_aware / product_aware · hot → most_aware**) rank at the FRONT; off-temperature angles fill the tail so a thin temperature-matched shelf never starves the batch (**preference, not filter**). Dahlia declares the intent FIRST in her [[creative-agent]] `stockProduct` call (default `{cold, test-to-find-winner}` via `resolveResearchIntent`); Max's Phase 2 grader benchmarks the same declared intent. Pure helpers `rankByWinnerSignalAndIntent`, `winnerTierRank`, `awarenessStageMatchesTemperature` are exported + pinned by `src/lib/ads/creative-sourcing.winner-tier.test.ts`.

**Focal-point temperature fit (`competitorTemperatureFit`, CEO 2026-07-17).** The partition is no longer awareness_stage-only — it now reads the ad's **focal point** too, because the presence of an **offer** is itself a temperature tell: a competitor ad running a discount is almost certainly retargeting a warm/hot (already-aware) audience, so it's the wrong imitation base for a cold prospecting creative. The CEO rule: **cold ⇒ curiosity / problem-solution ads; warm/hot ⇒ offer / mechanism / customer-review ads.** `competitorTemperatureFit(angle, temperature) → 'match' | 'neutral' | 'mismatch'` with this precedence: (1) an **offer** (`creative_skeletons.offer`) is a HARD warm/hot tell — a `mismatch` for cold, a `match` for warm/hot, whatever the stage; (2) the **awareness_stage** is the direct audience signal (a cold stage wins over a softer social-proof lever — `competitorFocalIsWarmHot` / `competitorFocalIsCold` read the `cialdini_lever` [`social_proof`=review, `scarcity`=offer/urgency — but NOT `authority`, which is cold-appropriate, see below] + the archetype/angle text only as the tiebreak when the stage is untagged). `rankByWinnerSignalAndIntent` partitions **match → neutral → mismatch** (winner-tier order preserved within each), so an offer-bearing proven winner sinks BELOW an offer-less problem ad for a cold task. This closes the 2026-07-17 `cold_offer_leak` at its SOURCE (Dahlia stops imitating an offer/retargeting ad for cold), of which the [[creative-agent]] competitor-DNA-offer strip is the downstream defense-in-depth. Pinned by the same test file.

**Cold angle selection HARD-EXCLUDES warm/hot competitor angles + own-brand fallback (cold-prospecting-never-imitates-a-warm-hot-offer-or-retargeting-competitor-ad Phase 2 — WIRED).** `competitorTemperatureFit`'s partition merely DEPRIORITIZES warm/hot competitor ads for a cold task (they sort to the tail but stay eligible), so an offer-heavy or cross-category competitor shelf could still yield a retargeting ad as the "least-bad" pick for a cold prospecting test — the 2026-07-17 Amazing Creamer regression (a Holiday-bundle GLP-1 slimming-probiotic ad — retargeting + wrong category — was selected for a cold creamer test). Phase 2 closes this by HARD-EXCLUDING (not just ranking last) any competitor angle where `competitorFocalIsWarmHot` is true when the target is COLD. The chokepoint is [[creative-brief]] `selectAnglesForTemperature(competitorAngles, ownBrandAngles, temperature) → ScoredAngle[]`: for `temperature === 'cold'` it filters the competitor pool through `competitorFocalIsWarmHot({ offer, conceptTags })` (drops any angle with an offer set, or a `cialdini_lever` of `social_proof`/`scarcity`, or an archetype/angle text naming offer/review/guarantee/comparison), then concatenates the SURVIVING competitor angles with the caller's own-brand angles — so when the cold competitor pool empties, the returned pool is JUST the own-brand angles at their natural rank (own-brand angles are cold-appropriate by construction — `selectAngles`'s scoring favors transformation / objection / curiosity, the CEO's `results_first` / `benefit-led` / `problem→solution` families). Warm/hot temperatures return `[...competitorAngles, ...ownBrandAngles]` unchanged — the exclusion is TEMPERATURE-SCOPED because warm/hot tests WANT the offer/mechanism/review angles as their imitation base. [[creative-agent]] `stockProduct` now merges via this helper (was `const ranked = [...competitorAngles, ...ownAngles]`) with `researchIntent.audience_temperature` as the third arg, so the cold-prospecting default lands the hard-exclude by construction. Pinned by six cases in [[../../../src/lib/ads/creative-brief.test.ts]] (`npx tsx --test src/lib/ads/creative-brief.test.ts`): cold + only warm/hot competitors → own-brand fallback; cold + a cold competitor curiosity/problem_aware angle IS selected; warm + hot selections unchanged; cold + own-brand-only pool passes through; and the Amazing Creamer regression fixture (GLP-1 holiday bundle) is hard-excluded.

**`authority` + mechanism are COLD-appropriate — un-excluded 2026-07-19.** The Phase-2 hard-exclude above originally lumped `cialdini_lever === 'authority'` and mechanism-focal text in with the genuine warm/hot tells. That was an over-correction: a cold stranger is opened by **credibility** ("clinically shown", "nutritionist-formulated", 3rd-party-tested) and **education** ("how it actually works"), not just curiosity — those are prospecting levers, not retargeting. It emptied the cold shelf for any product whose entire proven competitor set is authority-framed: **Ashwavana Guru Focus had all 8 deeply-proven winners at `cialdini=authority` (Cymbiotika, Magic Mind, Onnit, Goli) → 0 cold-eligible → an own-brand ingredient angle fell into every cold run** (verified via [[ads-read-sdk]] `traceAdOrigin` + `getProvenCompetitorAngles`). So `competitorFocalIsWarmHot` no longer treats `authority` (lever) or `mechanism` (text) as warm/hot; the **OFFER** check + `social_proof`/`scarcity`/review/comparison stay, so the 2026-07-17 `cold_offer_leak` protection is intact (an offer-bearing authority ad is still excluded). Net: 6 of the 8 Guru winners became cold-eligible (only the offer-bearing + review-focal ones stay excluded). Pinned by the `competitorFocalIsWarmHot` cases in [[../../../src/lib/ads/creative-sourcing.winner-tier.test.ts]].

## Market-sophistication escalation policy ([[../specs/dahlia-market-sophistication-escalation]])
`src/lib/ads/market-sophistication.ts` exports **`computeMarketSophistication(admin, workspaceId, productId) → Promise<{shelfModal: 1|2|3|4|5; targetLevel: 1|2|3|4|5; evidence: string[]}>`** — the **+1 escalation POLICY** over the shelf modal above. It reads THIS product's deliberately-chosen shelf via `getProvenCompetitorAngles({productId, minDaysRunning: 30})` (the per-product filter is what makes "the shelf" mean "this product's shelf" — `creative_skeletons.product_id` is the deliberate imitate link per [[../tables/creative_skeletons]] line 50), delegates to `computeSophisticationLevel(shelf)` for `shelfModal`, and applies **`targetLevel = Math.min(5, shelfModal + 1)`**. The escalation exists because writing AT the modal is the Schwartz failure mode the goal calls out: everyone at level `target-1` loses because the market already heard it and yawns; the +1 turns "know the market" into "write above it." Empty shelf → `{shelfModal:3, targetLevel:4, evidence:['no proven competitor shelf — defaulting to mid-market']}` (safe mid-market default; write one step above). [[creative-agent|Dahlia]]'s `stockProduct` calls this in place of `computeSophisticationLevel(sourced)` and threads **`target_schwartz_level = result.targetLevel`** + **`market_sophistication_evidence = result.evidence`** into the per-creative author-mode session input (the SKILL contract carries the escalation rule verbatim: *the shelf modal is target-1; everyone at target-1 loses because the market already heard it and yawns. If you cannot write at target without fabricating, drop to target-1 and cite the fallback in your verdict rationale*). Max's independent copy QC (`.claude/skills/max-copy-qc/SKILL.md`) receives the same evidence via TRUSTED CONTEXT so his advisory persuasion score can flag when Dahlia's actual level (as detected by his own read of the copy) is below `target_schwartz_level`. Pinned by `src/lib/ads/market-sophistication.test.ts` (`npm run test:market-sophistication`): empty shelf → `L4` target, all-L4 shelf → `L5` target, all-L5 shelf → `L5` (clamped), L3 majority + L4 tail → `L4` target with one evidence line per angle, and the productId argv structural pin.
