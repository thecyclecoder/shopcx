# `src/lib/ads/creative-sourcing.ts`

The shared SDK for **where great ad ideas come from** + **how ads actually perform** (CEO 2026-07-11). Dahlia (sources angles), Bianca (reads signal), and Max (supervises) all call this one surface instead of re-deriving it from raw Meta/DB queries. It codifies the ad-hoc analysis that validated our signal model.

## The three idea pools + the analyzer
- **`getProvenCompetitorAngles(admin, ws, {minDaysRunning, productId, niche, limit})`** — the [[../tables/creative_skeletons]] competitor library, RANKED by `days_running` (longevity = a competitor is profitably scaling it = a validated angle). **Pass `productId`** (CEO 2026-07-12) → reads ONLY skeletons the [[../inngest/creative-scout]] tagged for THAT product's deliberately-chosen competitors (`creative_skeletons.product_id`) — the imitate→innovate path, a product imitates only its own shelf. `niche` is the LEGACY substring filter (e.g. "coffee", "weight"), superseded by `productId` and ignored when `productId` is set. [[creative-agent|Dahlia]] `stockProduct` now passes `productId` (the coffee/weight niche hack is gone). **The strongest pool** — real market-validated hooks.
- **`getOurWinningAngles(admin, ws, metaAccountId, {maxCpaCents, minSpendCents})`** — our OWN best-performing ads ranked by cost-per-ATC then CPP — "what works for US", the **exploit seed** (the concepts to make variations of).
- **`analyzeAccountAds(token, metaAccountId, {datePreset})`** — the per-ad performance analyzer: spend, purchases, CPP, ATC, **cost-per-ATC**, **CPM**, CTR, reactions/saves/shares. Meta ground truth.
- *(Web DR-angle research — a future pool.)*

## The validated signal model (why these fields)
Proven on **99 historical ads** (45 winners, 24 losers): **cost-per-ATC** ($25 winners vs $79 losers) and **CPM** ($93 vs $148) discriminate winners. **CTR + reactions are TRAPS** — losers click 2× and react 7× *more* (clickbait). So the analyzer surfaces CTR/engagement for visibility but winners are chosen on cost-per-ATC + CPM. This is the same model [[meta-cpa-signal]] trims/crowns on.

## Shelf-derived sophistication signal ([[../specs/dahlia-five-frameworks-copy-skill]] M2)
[[sophistication|`src/lib/ads/sophistication.ts`]] exports **`computeSophisticationLevel(shelf: CompetitorAngle[]) → 1|2|3|4|5`** — a pure, no-I/O helper that reads the same `getProvenCompetitorAngles` shelf and returns the **modal Schwartz awareness level** the market is writing at (higher-wins tiebreak; empty shelf → `3`, safe solution-aware default). The token detector's five rule buckets — L2 problem tokens (`tired`/`crash`/`fog`/`stress`), L3 solution-category (`clean energy`/`focus support`), L4 mechanism (`adaptogen`/`L-theanine`/`ashwagandha`), L5 versus (`vs coffee`/`compared to`/`instead of`) — are pinned by `src/lib/ads/sophistication.test.ts` (`npm run test:sophistication`).

## Winner-tier + intent-scoped research ([[../specs/dahlia-researches-from-winners-flow-ad-library]] Phase 1)
`getProvenCompetitorAngles` now selects the **winners-flow longitudinal signals** — `winner_tier`, `winner_score`, `concept_tags` — and re-ranks the returned angles by **winner-tier rank (proven > building > new > retired) → winner_score → days_running**. Retired competitor ads sink to the tail even at high `days_running` (a killed ad is not a research base). Each returned `CompetitorAngle` carries the new fields (`winnerTier`, `winnerScore`, `conceptTags`) so downstream ([[creative-brief]] / [[creative-agent|Dahlia]] session / Max's Phase 2 grader) reads OUR unified breakdown (angle · archetype · why_it_works · cialdini_lever · awareness_stage · format) as the imitation rubric.

The **declared-intent envelope** — `CreativeIntent = { audience_temperature: 'cold'|'warm'|'hot', purpose: 'test-to-find-winner' }` — is passed as `opts.intent`. When set, angles whose `concept_tags.awareness_stage` matches the temperature bucket (**cold → unaware / problem_aware · warm → solution_aware / product_aware · hot → most_aware**) rank at the FRONT; off-temperature angles fill the tail so a thin temperature-matched shelf never starves the batch (**preference, not filter**). Dahlia declares the intent FIRST in her [[creative-agent]] `stockProduct` call (default `{cold, test-to-find-winner}` via `resolveResearchIntent`); Max's Phase 2 grader benchmarks the same declared intent. Pure helpers `rankByWinnerSignalAndIntent`, `winnerTierRank`, `awarenessStageMatchesTemperature` are exported + pinned by `src/lib/ads/creative-sourcing.winner-tier.test.ts`.

**Focal-point temperature fit (`competitorTemperatureFit`, CEO 2026-07-17).** The partition is no longer awareness_stage-only — it now reads the ad's **focal point** too, because the presence of an **offer** is itself a temperature tell: a competitor ad running a discount is almost certainly retargeting a warm/hot (already-aware) audience, so it's the wrong imitation base for a cold prospecting creative. The CEO rule: **cold ⇒ curiosity / problem-solution ads; warm/hot ⇒ offer / mechanism / customer-review ads.** `competitorTemperatureFit(angle, temperature) → 'match' | 'neutral' | 'mismatch'` with this precedence: (1) an **offer** (`creative_skeletons.offer`) is a HARD warm/hot tell — a `mismatch` for cold, a `match` for warm/hot, whatever the stage; (2) the **awareness_stage** is the direct audience signal (a cold stage wins over a softer social-proof lever — `competitorFocalIsWarmHot` / `competitorFocalIsCold` read the `cialdini_lever` [`social_proof`=review, `scarcity`=offer, `authority`=mechanism] + the archetype/angle text only as the tiebreak when the stage is untagged). `rankByWinnerSignalAndIntent` partitions **match → neutral → mismatch** (winner-tier order preserved within each), so an offer-bearing proven winner sinks BELOW an offer-less problem ad for a cold task. This closes the 2026-07-17 `cold_offer_leak` at its SOURCE (Dahlia stops imitating an offer/retargeting ad for cold), of which the [[creative-agent]] competitor-DNA-offer strip is the downstream defense-in-depth. Pinned by the same test file.

## Market-sophistication escalation policy ([[../specs/dahlia-market-sophistication-escalation]])
`src/lib/ads/market-sophistication.ts` exports **`computeMarketSophistication(admin, workspaceId, productId) → Promise<{shelfModal: 1|2|3|4|5; targetLevel: 1|2|3|4|5; evidence: string[]}>`** — the **+1 escalation POLICY** over the shelf modal above. It reads THIS product's deliberately-chosen shelf via `getProvenCompetitorAngles({productId, minDaysRunning: 30})` (the per-product filter is what makes "the shelf" mean "this product's shelf" — `creative_skeletons.product_id` is the deliberate imitate link per [[../tables/creative_skeletons]] line 50), delegates to `computeSophisticationLevel(shelf)` for `shelfModal`, and applies **`targetLevel = Math.min(5, shelfModal + 1)`**. The escalation exists because writing AT the modal is the Schwartz failure mode the goal calls out: everyone at level `target-1` loses because the market already heard it and yawns; the +1 turns "know the market" into "write above it." Empty shelf → `{shelfModal:3, targetLevel:4, evidence:['no proven competitor shelf — defaulting to mid-market']}` (safe mid-market default; write one step above). [[creative-agent|Dahlia]]'s `stockProduct` calls this in place of `computeSophisticationLevel(sourced)` and threads **`target_schwartz_level = result.targetLevel`** + **`market_sophistication_evidence = result.evidence`** into the per-creative author-mode session input (the SKILL contract carries the escalation rule verbatim: *the shelf modal is target-1; everyone at target-1 loses because the market already heard it and yawns. If you cannot write at target without fabricating, drop to target-1 and cite the fallback in your verdict rationale*). Max's independent copy QC (`.claude/skills/max-copy-qc/SKILL.md`) receives the same evidence via TRUSTED CONTEXT so his advisory persuasion score can flag when Dahlia's actual level (as detected by his own read of the copy) is below `target_schwartz_level`. Pinned by `src/lib/ads/market-sophistication.test.ts` (`npm run test:market-sophistication`): empty shelf → `L4` target, all-L4 shelf → `L5` target, all-L5 shelf → `L5` (clamped), L3 majority + L4 tail → `L4` target with one evidence line per angle, and the productId argv structural pin.
