# migrate-to-internal.ts

`src/lib/migrate-to-internal.ts` — the **strangler migration**: flip a customer's Appstle subscriptions to internal billing **in place** (no new rows, so the stable `subscriptions.id` and every reference to it stays valid). Called wherever we capture a payment method (checkout, portal add-card, recovery) and by the self-healing guard on portal load.

Full flow: [[../lifecycles/subscription-billing]] § Migration path.

## Exports

- **`migrateCustomerAppstleSubsToInternal(workspaceId, customerId, opts?) → MigrateResult`** — migrates **all** Appstle subs across the customer's link group. **HARD RULE: a migration must be billable** — resolves the link-group member with a default Braintree PM, reassigns the sub to it, and **skips** every sub when no linked account has a PM. Per sub: read the **live** Appstle contract → translate lines → internal catalog UUID refs (grandfathered lines get `price_override_cents`, via [[appstle-pricing]] `inferAppstleLineBase`) → cancel the Appstle contract → flip the row `is_internal=true` / native `internal-*` id / billable customer → **pin the default card** (`payment_method_id`) → record + verify a [[migration-audit]] audit. `opts.isRecovery` threads from the payment-recovery flow. Returns `{ migrated, skipped, failed }`.
  - **Card pin (mirrors recovery).** `findBillableCustomer` returns both the billable member AND its default `customer_payment_methods` row id; the flip sets `payment_method_id` so the renewal charges that card **explicitly** (the portal sub-detail shows it; a later default change can't silently move the charge). The default-card fallback ([[../lifecycles/subscription-billing]] § payment method) stays the safety net for any unpinned sub. Idempotent — a re-run re-pins the *current* default. A customer with **no** card → still skipped by the billable rule, never pinned.
- **`migrateContractToInternalComp(workspaceId, contractId, { compNote? }) → { ok, subId?, internalContractId?, error? }`** — **comp** migration for **one** contract. Same translate-lines + cancel-contract, but **no billable-PM requirement** (a comp sub never charges) and the `customer_id` is **preserved** (no reassignment). Sets `comp=true` + `comp_note` + **every item `price_override_cents=0`** (base $0, overriding any grandfathered base). Preserves items/cadence/next date/status. **Idempotent** (a contract already `is_internal && comp` returns ok). **No `migration_audit`** is recorded — the audit's `card_pinned` check expects a billable card a comp sub deliberately lacks. Used by `scripts/migrate-zach-comp-subscription.ts` (first comp sub: Zach Zavala, employee).
- **`ensureGroupMigratedIfBillable(workspaceId, customerId) → count`** — cheap self-healing guard (one count query); runs `migrateCustomerAppstleSubsToInternal` only when a link-group Appstle straggler exists **and** there's a working default Braintree PM. Call it wherever subs are fetched.

## Internal helpers

- `appstleLinesToInternalItems` — translate live Appstle lines → internal catalog UUID refs (no baked price; grandfathered → `price_override_cents`). Resolves each line by **`shopify_variant_id` first, then `sku`** (workspace-scoped) — a line can carry a Shopify id we never synced while its sku still resolves the variant. A line that matches **neither** is genuinely unmappable (out of stock / discontinued / no internal product) → **DROPPED** (returned in `droppedLines`), never left as a dangling Shopify-id line. Returns `{ items, shippingProtectionCents, droppedLines }`. **Shipping protection is special**: Appstle bills it as a line item titled "Shipping Protection", but internally it's a **flag** the engine bills separately — so this helper EXCLUDES that line from `items[]` and returns its charge separately. The normal migration sets `shipping_protection_added=true` + `shipping_protection_amount_cents` on the flip when `shippingProtectionCents > 0`; the comp path ignores it (a comp sub ships free).
- **Unmappable items (drop + escalate).** Each `DroppedLine` is `{ title, shopifyVariantId, sku, priceCents, quantity, paid }`. The migration records all drops as a `dropped_unmappable_items` note in [[../tables/migration_audits]]`.notes` (so the drop is auditable). A `paid` drop (`priceCents > 0`) ALSO fires a **critical** [[notify-ops-alert]] — a human must add the right variant or refund, since the next renewal would otherwise ship short. A $0 free-gift/promo drop is safe and stays note-only. This is what stops Therese's case (an out-of-stock $0 ACV-Gummies line with a dangling Shopify `variant_id`) from failing `items_on_uuids` **forever** — the dangling line is now dropped at migration time, not left for the retry loop to fail on.
- `isShippingProtectionLine` — title-match helper (`title` includes "shipping protection", case-insensitive — same convention [[../libraries/pricing]] and [[migration-audit]]'s `items_on_uuids` check use).
- `linkedCustomerIds` / `findBillableCustomer` — link-group expansion + billable-member resolution (default Braintree PM, prefer the passed-in customer).

## Gotchas

- **Cancel-then-flip** — cancel the Appstle contract FIRST so a later flip failure stops the sub (re-runnable) rather than double-billing it. Already-cancelled subs skip the cancel.
- **Comp vs billable** — `migrateContractToInternalComp` deliberately bypasses the billable-PM rule and skips the audit. Never use it for a sub that should actually charge — it sets base $0. Conversely a normal migration never sets `comp`.
- Cancelled subs migrate using the local row when the live Appstle contract is unreadable; an active/paused sub that can't be read is left alone (re-runnable).
- **Base ≤ MSRP invariant.** `price_override_cents` exists ONLY to lock a grandfathered base **STRICTLY BELOW** catalog MSRP. `appstleLinesToInternalItems` gates the override on `isGrandfathered && trueBaseCents > 0 && trueBaseCents < msrp` (`msrp = product_variants.price_cents`) — an at-or-above-MSRP base is **never** stored, because it would feed the −25% S&S + quantity-break math from too high a start and inflate the charge. The old code stored some reverse-engineered above-MSRP bases (Lisa Baker `fdc1d5e3` → ~$119.92 instead of $110.34); `scripts/backfill-drop-over-msrp-overrides.ts` swept + dropped those, and the matching `> MSRP` clamp lives in [[migration-fix]] `price_reconcile`. See [[../specs/base-price-never-above-msrp]].
- **Shipping protection: line → flag.** Appstle carries protection as a "Shipping Protection" line; internally it's `shipping_protection_added` + `shipping_protection_amount_cents`, billed separately on top of the product subtotal by [[../libraries/pricing]] / [[../inngest/internal-subscription-renewals]]. The migration pulls that line OUT of `items[]` into the flag, and **excludes it from `pre_migration_charge_cents`** so the audit baseline is the product subtotal only (what the engine's `product_subtotal_cents` reports). The customer's total is unchanged (subtotal + flag protection). Before this fix the protection line landed in `items[]` as a bogus catalog-less line, the flag was never set, and `pre_migration_charge_cents` over-counted by the protection amount → `pricing_preserved` failed on every protection-carrying migration (see [[migration-fix]] `shipping_protection_convert`, the repair for already-stuck subs).
