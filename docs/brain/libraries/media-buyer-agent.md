# libraries/media-buyer-agent

Media Buyer agent ‚Äî the weekly Test‚ÜíMeasure‚ÜíPromote‚ÜíKill cadence that the box worker's `media-buyer` lane runs (media-buyer-test-winner-loop Phase 2). Owned by [[../functions/growth]]; supervised by the Growth Director's leash.

**File:** `src/lib/media-buyer/agent.ts` ¬∑ **Runner:** [[builder-worker]] `runMediaBuyerJob` (scripts/builder-worker.ts).

The Media Buyer is the FIRST autonomous static-ad optimizer in this repo ‚Äî the mandate's "source ‚Üí test ‚Üí scale winners / cut losers ‚Üí feed learnings back" loop, made continuous and agent-owned. It IS a **supervisable-autonomy tool** ([[../operational-rules]] ¬ß North star): it proposes proxy-optimizing actions (scale-up on winners, pause on losers, replenish the test cohort) but every irreversible write goes through EXISTING sanctioned chokepoints:

- **Meta budget/status changes** ‚Äî the agent writes to [[../tables/iteration_actions]] at `status='decided'`; [[meta__execution]] `executeAutonomousActions` (Storefront Iteration Engine Phase 6a) picks them up and calls the Meta Graph. The agent NEVER calls `updateObjectStatus` / `updateObjectBudget` directly.
- **Live publishes into the test cohort** ‚Äî the agent inserts [[../tables/ad_publish_jobs]] rows with `origin='media-buyer-test'` + `publish_active=true` and fires `ad-tool/publish-to-meta`. [[media-buyer-publish-gate]] (Phase 1) then decides whether the ad actually ships ACTIVE ‚Äî a wrong ad set or over-ceiling projection DOWNGRADES + escalates.
- **Every action** stamps one [[../tables/director_activity]] row (`director_function='growth'`) citing the source `meta_ad_id` + realized ROAS + policy version, so the audit trail names the concrete creative, not the wrapper adset.

## The five verbs

Every pass emits at most five kinds of typed action:

- `promote` ‚Äî a detected winner (via [[winning-creative-detect]] `detectWinners`) whose ROAS ‚â• policy `scale_up_roas_trigger` triggers a `scale_up` on the winner's parent Meta adset (via `iteration_actions`). The step is bounded by the policy's `scale_up_step_pct` / `scale_up_cap_pct`. Each action carries the source `meta_ad_id` + its ROAS + the parent `adset_id`.
- `kill` ‚Äî a decision-tree loser from [[media-buyer__meta-cpa-signal]] `detectMetaCpaLosers` triggers a `pause` (via `iteration_actions`). Phase 2 of [[../specs/media-buyer-kill-on-decision-tree-retire-roas-floor]] pins the kill predicate ‚Äî `isDecisionTreeKill` in `meta-cpa-signal.ts` ‚Äî to two sources: **(a)** `tierForTest === 'dud'` (1:1 parity with [[winning-creative-detect|../ads/testing-results-sdk]] `tierForTest`, so an agent kill == a `/ad-testing-results` "dud" badge) ‚Äî deadline dud (`spend ‚â• max_test_spend_cents` AND (`purchases === 0` OR `cac > hold_band_max_cpa_cents`)) OR early dud (`spend ‚â• early_trim_min_spend_cents` AND `purchases === 0`) ‚Äî and **(b)** the EARLY leading-signal trim (cost-per-ATC / CPM / clicks-no-ATC) past `early_trim_min_spend_cents`, gated by the HOLD-band converter guard so a profitable converter (`purchases > 0` AND `cac ‚â§ hold_band_max_cpa_cents`) is NEVER trimmed. The legacy ROAS-floor / `pause_min_spend_cents` kill trigger is RETIRED (Phase 1); the legacy (S) slow-kill (converter above hold_band, past `crown_min_spend_cents`, pre-deadline) and (F1) 0-purchase-past-`crown_min_spend_cents` backstop are folded into `tierForTest`'s deadline / early-dud rules (Phase 2) ‚Äî a test with sales, under the deadline, within/near the hold band is NEVER killed (the spec's skeptic v3 protection). The action cites the source `meta_ad_id` so the audit trail names the creative, not just the wrapper.
- `replenish` ‚Äî when the test cohort ([[../tables/media_buyer_test_cohorts]]) has fewer than `cohortTargetCount` live ads, the agent picks from [[ready-to-test]] (top-of-bin, in ready order ‚Äî TEMPERATURE-scoped to `'cold'` per [[../specs/bianca-route-ready-creatives-by-dahlia-temperature-tag]] Phase 1, so a Warm/Hot creative Dahlia tagged cannot leak into the cold rail's deficit fill; every media-buyer cohort we ship today is a per-test cold cohort) and publishes each via the Phase 1 rail (`origin='media-buyer-test'`, `publish_active=true`). The cohort ceiling is enforced by [[media-buyer-publish-gate]]. **([[../specs/dahlia-andromeda-concept-diversity-tags]] Phase 2 ‚Äî concept-diversity gate)** the picker is scoped by [[../tables/ad_campaigns]] `concept_tag`: the runner precomputes `liveConceptTags = readLiveCohortConceptTags(admin, {workspaceId, productId})` (distinct non-null `concept_tag` values from ACTIVE `origin='media-buyer-test'` publish jobs joined to `ad_campaigns`, product-scoped), and `computeMediaBuyerPlan`'s replenish loop SKIPS any ready candidate whose `concept_tag ‚àà liveConceptTags` in favor of the next ready row. Each accepted pick reserves its tag against the same-pass bin so a single deficit fill stays internally diverse. **NULL is its own 'untagged' bucket** ‚Äî never conflicts with any Andromeda token so a deterministic-mode cohort (`DAHLIA_COPY_MODE=deterministic`, every campaign carries `concept_tag=null`) is byte-identical to the pre-Phase-2 pick order. If the loop exhausts the ready bin without filling the deficit AND the shortfall was caused by the diversity gate rejecting ‚â•1 candidate, the plan carries `replenishDiagnostic={kind:'no_diverse_candidate', liveConceptTags, readyTagsAvailable}` and the runner emits ONE `media_buyer_replenish_no_diverse_candidate` [[../tables/director_activity]] row (metadata: `{cohort_id, product_id, live_concept_tags, ready_tags_available, cohort_target_count, current_test_cohort_size, replenish_filled}`) so #director-growth-max surfaces the concept-shortage as a Dahlia diversity nudge. A straight-up empty bin still hits the pre-existing `replenish short: ‚Ä¶ ready-to-test bin exhausted` summary line (no diagnostic ‚Äî the shortage is inventory, not diversity).
- `fatigue_replenish` ‚Äî **(Phase 3)** ‚Äî a WINNING ad whose parent adset's `iteration_scorecards_daily.fatigue_score` crosses `FATIGUE_REPLENISH_THRESHOLD = 0.5` (the same cutoff decision-engine uses to suppress a scale-up on fatigue) triggers a call to [[winning-creative-detect]] `amplifyWinner` ‚Äî spawns N fresh variants of the winning angle at `status='ready'`, respecting the per-day `MAX_AMPLIFICATIONS_PER_DAY` cap. The variants land in [[ready-to-test]] and the standard `replenish` verb picks them up on the next pass to publish into the test cohort. The action cites the source `meta_ad_id` + its ROAS + fatigue score + the resulting `new_ad_campaign_ids` so the lineage is traceable.
- (implicit) `shadow` ‚Äî **([[../specs/media-buyer-shadow-mode]] Phase 2)** ‚Äî when the active policy is on `mode='shadow'`, the runner computes the plan (unchanged pure path) but writes ZERO [[../tables/iteration_actions]] + ZERO [[../tables/ad_publish_jobs]] and NEVER calls [[winning-creative-detect]] `amplifyWinner`. Instead, it emits ONE `<verb>_shadow` [[../tables/director_activity]] row per plan action ‚Äî `media_buyer_promoted_winner_shadow` ¬∑ `media_buyer_paused_loser_shadow` ¬∑ `media_buyer_replenished_test_cohort_shadow` ¬∑ `media_buyer_fatigue_replenish_triggered_shadow` ‚Äî each carrying `metadata.mode='shadow'` + the full `plan_action` JSON + the source citation (`source_meta_ad_id`, `roas`, `policy_version_id`). The `media_buyer_pass_completed` heartbeat's metadata ALSO carries `mode='shadow'` so a downstream reviewer can filter shadow passes vs armed passes uniformly. The CEO's non-negotiable read-only-before-armed guardrail (parent goal [[../goals/autonomous-media-buyer-supervision]] M2) ‚Äî a freshly authored policy defaults to `shadow` at author-time, and the flip to `armed` is a separate, audited surface (spec `media-buyer-armed-flip-surface`).
- `deferred` (surfaced on `plan.deferred[]`, not a fifth verb ‚Äî one entry per promote the pure plan-computer dropped because a scale-edit rail fired) ‚Äî **([[../specs/bianca-scale-edit-rails-cooldown-and-account-delta-ceiling]] Phase 1)** ‚Äî Bianca now honors the two rails the storefront decision engine already enforces on scale actions (mirrored from [[../meta/decision-engine]] `computeAutonomousActions`): (1) **`per_object_cooldown_hours`** ‚Äî a promote against an object that has moved inside the cooldown window is dropped with `rail='per_object_cooldown'` carrying `sinceLastActionMs` + `cooldownMs`; (2) **`per_account_daily_budget_delta_ceiling_cents`** ‚Äî a promote whose `|after ‚àí before|` would push the pass's cumulative delta past the per-account daily ceiling is dropped with `rail='per_account_daily_budget_delta_ceiling'` carrying `wouldBeDelta` + `cumulativeSoFar` + `ceiling`. `MediaBuyerPlanInputs` gains `recentActions` (typed slice of `iteration_actions` rows ‚Äî `object_id, action_type, created_at, before_budget_cents, after_budget_cents` ‚Äî shape mirrors the decision-engine's `RecentAction` plus the budget columns for future same-day historical delta seeding) plus `nowMs`; both are OPTIONAL so pre-Phase-1 callers (unit tests, legacy fixtures) keep the pre-rail behavior. Phase 2 will thread them through the runner and turn each `deferred` entry into ONE `media_buyer_scale_rail_deferred` [[../tables/director_activity]] row so the promote ledger explains suppression instead of a "silently empty" pass ‚Äî a supervisable-autonomy guardrail on the goal's M1 clean-cold-read fix list. Unit-pinned in `agent.test.ts` (cooldown-in-window ‚Üí deferred + rail='per_object_cooldown'; two winners whose combined delta breaches the ceiling ‚Üí first emitted, second deferred + rail='per_account_daily_budget_delta_ceiling').
- (implicit) `dormant ‚Äî no active policy` ‚Äî no active [[../tables/iteration_policies]] row ‚Üí NO actions; the pass records `media_buyer_no_active_policy` and returns. Never silent.
- (implicit) `dormant ‚Äî sensor-trust denied` ‚Äî **([[../specs/media-buyer-sensor-trust-probe]] Phase 3)** ‚Äî BEFORE `computeMediaBuyerPlan`, the runner loads the newest [[../tables/media_buyer_sensor_trust]] snapshot for `(workspaceId, metaAdAccountId)` and enforces (a) present, (b) age ‚â§ `SENSOR_TRUST_MAX_AGE_MS = 48h`, (c) `band !== 'red'`. Any check failing writes ONE `media_buyer_sensor_trust_denied` [[../tables/director_activity]] row (metadata: `{reasons, snapshot_date, band, coverage_ratio}` ‚Äî the probe's own numbers, cited verbatim) + returns the same dormant summary shape as the no-active-policy path ‚Äî ZERO `iteration_actions` writes, ZERO `ad_publish_jobs`, no Meta motion. This is the short-circuit the parent goal's "shadow-mode winner/loser calls match a human review within tolerance" criterion hinges on: only trust ROAS numbers once the attribution sensor is provably clean for that cohort. A `stale_snapshot` reason is added when the freshness cap trips (stale trust ‚â° untrusted); `missing_snapshot` when the row is absent entirely.

## Exports

### `MediaBuyerPlan` ‚Äî interface

The typed plan a pass emits. `policyActive`, `policyVersionId`, `cohortConfigured`, `cohortTargetCount`, `currentTestCohortSize`, `promote[]`, `kill[]`, `replenish[]`, `fatigueReplenish[]`, `replenishDiagnostic`, `summary`. **`replenishDiagnostic`** ([[../specs/dahlia-andromeda-concept-diversity-tags]] Phase 2): `MediaBuyerReplenishDiagnostic | null` ‚Äî non-null when the replenish loop partialed because the concept-diversity gate rejected every remaining candidate; the runner reads it to emit `media_buyer_replenish_no_diverse_candidate`. NULL when the deficit was filled OR the ready bin was empty (that hits the existing "ready-to-test bin exhausted" summary line, not a diversity failure).

### `MediaBuyerDeferredAction` ‚Äî interface

**([[../specs/bianca-scale-edit-rails-cooldown-and-account-delta-ceiling]] Phase 1)** ‚Äî `{ rail: 'per_object_cooldown' | 'per_account_daily_budget_delta_ceiling', targetObjectId, sourceMetaAdId, policyVersionId, rationale, sinceLastActionMs?, cooldownMs?, wouldBeDelta?, cumulativeSoFar?, ceiling? }`. One entry per promote the pure plan-computer dropped because a scale-edit rail fired ‚Äî cooldown-in-window promotes carry `sinceLastActionMs` + `cooldownMs`; delta-ceiling-breaching promotes carry `wouldBeDelta` + `cumulativeSoFar` + `ceiling`. The runner (Phase 2) turns each into a `media_buyer_scale_rail_deferred` [[../tables/director_activity]] row so the promote ledger explains suppression.

### `MediaBuyerReplenishDiagnostic` ‚Äî interface

**([[../specs/dahlia-andromeda-concept-diversity-tags]] Phase 2)** ‚Äî `{ kind: 'no_diverse_candidate', liveConceptTags: string[], readyTagsAvailable: string[] }`. `liveConceptTags` mirrors the input set (sorted, distinct, non-null); `readyTagsAvailable` is the distinct non-null tags encountered in the ready-to-test bin during the pass. The runner's `media_buyer_replenish_no_diverse_candidate` audit row carries both arrays verbatim so the concept-shortage is CITED, not paraphrased.

### `MediaBuyerPromoteAction` ‚Äî interface

`{ kind: 'promote', sourceMetaAdId, roas, spendCents, targetLevel, targetObjectId, beforeBudgetCents, afterBudgetCents, rationale, policyVersionId, sourceAdCampaignId }`. The `sourceMetaAdId` + `roas` pair is the spec's verification citation.

### `MediaBuyerKillAction` ‚Äî interface

`{ kind: 'kill', sourceMetaAdId, roas, spendCents, targetLevel, targetObjectId, rationale, policyVersionId }`.

### `MediaBuyerReplenishAction` ‚Äî interface

`{ kind: 'replenish', adCampaignId, testMetaAdsetId, dailyTestCeilingCents, rationale }`.

### `MediaBuyerFatigueReplenishAction` ‚Äî interface

**(Phase 3)** ‚Äî `{ kind: 'fatigue_replenish', sourceMetaAdId, roas, fatigueScore, variantCount, rationale, policyVersionId, sourceAdCampaignId }`. The runner calls `amplifyWinner` for each and stamps a `media_buyer_fatigue_replenish_triggered` [[../tables/director_activity]] row carrying `{source_meta_ad_id, roas, fatigue_score, variants_spawned, new_ad_campaign_ids}` ‚Äî the fatigue signal is CITED, not narrated.

### `FATIGUE_REPLENISH_THRESHOLD` / `DEFAULT_FATIGUE_REPLENISH_VARIANTS` ‚Äî const

`0.5` / `2`. The fatigue cutoff (mirrors decision-engine's `fatigue_score >= 0.5` scale-up suppression) and the default variant count per fatiguing winner. `amplifyWinner` clamps the variant count at `MAX_VARIANTS_PER_WINNER = 4` and enforces the per-day `MAX_AMPLIFICATIONS_PER_DAY = 8` cap.

### `MediaBuyerLoser` ‚Äî interface

Input row for the plan-computer. `{ sourceMetaAdId, targetLevel, targetObjectId, roas, spendCents, triggeringScorecardId }`. The runner sources these from [[media-buyer__meta-cpa-signal]] `detectMetaCpaLosers` ‚Äî the crown/kill decision-tree ‚Äî under the trust-Meta path; a non-trust-Meta policy produces no losers (the legacy ROAS-floor scorecard query is retired per [[../specs/media-buyer-kill-on-decision-tree-retire-roas-floor]] Phase 1).

### `computeMediaBuyerPlan(input: MediaBuyerPlanInputs): MediaBuyerPlan` ‚Äî function

Pure. Takes fully-hydrated winners + losers + ready-to-test + cohort + policy + budgets, returns the typed plan. Encodes the promote thresholds (`scale_up_roas_trigger`, `scale_up_step_pct`, `scale_up_cap_pct`) and the `never_pause_object_ids` guard on the kill path ‚Äî the runner does no policy math itself. Post-Phase-1 the pure function TRUSTS its `input.losers` list (already vetted by the decision-tree source) and no longer re-gates on `roas_floor` / `pause_min_spend_cents`.

### `readLiveCohortConceptTags(admin, args): Promise<Set<string>>` ‚Äî function

**([[../specs/dahlia-andromeda-concept-diversity-tags]] Phase 2)** ‚Äî reads the DISTINCT non-null `ad_campaigns.concept_tag` values currently LIVE in the (workspace, productId) cohort ‚Äî every ACTIVE `origin='media-buyer-test'` publish job in `ad_publish_jobs`, joined to `ad_campaigns` narrowed to `productId` (null productId keeps the workspace-wide read used by the null-product default cohort). Returns an empty set for a workspace with no live tests, a deterministic-mode cohort where every campaign carries `concept_tag=null`, or a product with no live rows. NULL concept_tags are NEVER included ‚Äî NULL is its own 'untagged' bucket in `computeMediaBuyerPlan`'s replenish diversity gate. Signature: `{ workspaceId, productId }` ‚Üí `Set<string>`.

### `escalateUnderProvisionedCohort(admin, args)` ‚Äî function

Raises a deduped [[../tables/dashboard_notifications]] CEO card (type `agent_approval_request`, routed to CEO, `escalated_by_director='growth'`) when Bianca's replenish defers on missing config for an ACTIVE per-test cohort ‚Äî the visible half of the escalation Phase 3 of [[../specs/media-buyer-cohort-adset-template-guard-backfill-and-escalate]] adds on top of the existing quiet `media_buyer_replenish_missing_config` [[../tables/director_activity]] audit row. **Dedupe:** `dedupe_key = under_provisioned_cohort:{workspaceId}:{cohortId}:{reason}:{yyyy-mm-dd}` ‚Äî at most one card per (cohort, reason) per UTC day, so the 2h media-buyer pass cadence never spams the inbox but a persistent under-provisioning surfaces once daily until fixed. Read-then-insert is a compare-and-set on that key. The runner gates the call on `cohort?.isActive === true` (a dormant/retired cohort silently defers per the pre-existing contract; only an ACTIVE rail hit escalates). Signature: `{ workspaceId, productId, cohortId, reason, nowMs? }` ‚Üí `{ emitted: boolean }`.

### `runMediaBuyerLoop(admin, opts): Promise<{ plan, writes }>` ‚Äî function

The orchestrator. Reads all inputs, computes the plan, persists the writes:

1. `iteration_actions` upsert for promote (scale_up) + kill (pause) + reactivation (unpause). Same shape [[../meta/execution]] reads. The upsert returns row ids so the inline execute path can compare-and-set on `status='decided'`.
2. **Execute each decided action against Meta INLINE** ([[../specs/media-buyer-decided-kills-must-execute-on-meta-not-just-be-recorded]] Phase 1) via {@link executeDecidedActionAgainstMeta} ‚Äî kill ‚Üí `updateObjectStatus(objectId, 'PAUSED')`, unpause ‚Üí `updateObjectStatus(objectId, 'ACTIVE')`, scale_up ‚Üí `updateObjectBudget(objectId, { dailyBudgetCents: after })`. On success the row ‚Üí `status='executed'` with the Graph response in `external_result`; on failure the row ‚Üí `status='failed'` with the error message. **A failed row is NEVER stamped executed** ‚Äî the ledger cannot claim an action that never happened (no-false-promises).
3. `director_activity` row per plan action + `media_buyer_pass_completed` heartbeat. The per-action `media_buyer_paused_loser` / `media_buyer_promoted_winner` / `media_buyer_reactivated_recovered` emits (and their sibling `stampCreativeOutcome` learning-flywheel writes) are gated on `executed.has(key)` ‚Äî a decided-but-unfired row emits ZERO claim lines, so the audit trail never lies about a pause/promote that Meta refused. The heartbeat is always emitted.
4. `ad_publish_jobs` insert per replenish (via the local `enqueueReplenishPublish` helper) + `ad-tool/publish-to-meta` event. **([[../specs/bianca-never-posts-a-creative-without-a-max-grade-of-7-or-higher]] Phase 1 ‚Äî hard Max copy-QC rail, with [[../specs/bianca-posts-only-at-9of10-plus-ceo-manual-score-override-oversight-gate]] Phase 1 raising the floor 7‚Üí9 AND Phase 2 adding a CEO manual override.)** BEFORE any other DB read, `enqueueReplenishPublish` calls [[media-buyer-publish-gate]] `evaluateMaxCopyQcAtPublish` on the ad campaign ‚Äî which reads Max's verdict + the CEO postability override (via [[postability-override]] `readPostabilityOverride`, on [[../tables/ad_campaigns]] `override_postable/score/reason/by/at`) in parallel. When `override_postable=true` the gate SHORTCUTS to `ok:true` regardless of Max's score (CEO overruled). Otherwise a missing/NULL [[../tables/ad_creative_copy_qc_verdicts]] row, a `hard_gate_pass=false` verdict, or a hard-gate-pass verdict whose `persuasion_score < MAX_QC_ELIGIBILITY_FLOOR` (9) REFUSES the post fail-closed: no `ad_publish_jobs` row is inserted, no `ad-tool/publish-to-meta` event is fired, no dollars flow. The runner emits ONE `media_buyer_publish_refused_missing_max_copy_qc` [[../tables/director_activity]] row citing `{ad_campaign_id, refusal_reason: 'missing_max_copy_qc_verdict'|'hard_gate_fail'|'below_score_floor', score_floor: 9}` and DOES NOT fire `escalateUnderProvisionedCohort` (a below-floor creative isn't under-provisioning ‚Äî different rail, different owner). This is DEFENCE-IN-DEPTH over the `ad_campaigns` bin eligibility flag: a mis-set flag can NEVER publish a below-floor ad because the money step re-verifies the verdict + override independently. Max's real grade on the verdict row is NEVER mutated by the override ‚Äî the whole design preserves the Max-vs-CEO gap as the tuning signal for future live Claude sessions. Skipped with `media_buyer_replenish_missing_config` (below) is a SEPARATE branch for cohort-config gaps that still fires `escalateUnderProvisionedCohort` on an ACTIVE cohort. **Ad copy is sourced from the M3 variant pack (then the angle)** ‚Äî `enqueueReplenishPublish` calls `readCopyVariants(admin, adCampaignId)` (the SDK chokepoint in [[ad-copy-variants]] for [[../tables/ad_creative_copy_variants]], warm‚Üícold‚Üíhot ordered) and hands the pack to `resolveReplenishAdCopy(angle, {variants})`; when non-empty, the resolver splats 1:1 into the job's `headlines`/`primary_texts`/`descriptions` arrays (so a 3-variant pack lands 3-entry arrays and [[../inngest/ad-tool]] `adToolPublishToMeta` builds Meta's `asset_feed_spec` `titles[]`/`bodies[]`/`descriptions[]` with three entries ‚Äî Meta's Advantage+ picks per user). When the pack is empty (deterministic mode / legacy campaign that never had a pack authored), the resolver falls back to loading `product_ad_angles` via `ad_campaigns.angle_id` (the same source the human publish route uses in [[meta__cpa-signal]]) and populates `headlines`/`primary_texts` as single-element arrays exactly as pre-M3 ‚Äî byte-identical to today. **Fail-closed:** the exported pure helper `resolveReplenishAdCopy(angle, {variants})` returns `ok:false` when neither source yields usable copy, and the caller SKIPS the publish with a reason instead of enqueueing an invalid job. This closes the 2026-07-12 defect where the helper hard-coded `headlines:[]`/`primary_texts:[]`, so [[../inngest/ad-tool]] built a Meta creative with empty `asset_feed_spec` `titles[]`/`bodies[]` and Graph rejected EVERY auto-replenish publish with `meta_400 "The link field is required."` (Meta's misleading error for absent ad copy). Unit-pinned in `agent.test.ts` (`resolveReplenishAdCopy` cases ‚Äî including variants=[warm,cold,hot]‚Üí3-entry-arrays, variants=[warm]‚Üí1-entry, null variants‚Üíangle fallback, and the asset_feed_spec 1:1 structural pin).
5. **(Phase 3)** ‚Äî [[winning-creative-detect]] `amplifyWinner` call per fatigue-replenish action (which fires `ad-tool/generate-full` / `ad-tool/static-requested` for each variant and writes its own `amplified_winner` audit row). The runner also stamps a `media_buyer_fatigue_replenish_triggered` audit row so the FATIGUE-DRIVEN reason (not a manual amplify) is preserved.
6. **([[../specs/dahlia-andromeda-concept-diversity-tags]] Phase 2)** ‚Äî before `computeMediaBuyerPlan`, the runner reads `liveConceptTags = readLiveCohortConceptTags(admin, {workspaceId, productId})` and threads it in so the replenish loop enforces the diversity gate. When the plan comes back with `replenishDiagnostic.kind === 'no_diverse_candidate'` (partial replenish caused by every remaining candidate colliding with a live tag) AND the cohort is ACTIVE, the runner emits ONE `media_buyer_replenish_no_diverse_candidate` [[../tables/director_activity]] row citing `{cohort_id, product_id, live_concept_tags, ready_tags_available, cohort_target_count, current_test_cohort_size, replenish_filled}` ‚Äî the concept-shortage is CITED to Growth, not paraphrased. A partial from a straight-up empty ready bin hits only the pre-existing summary line (no diagnostic row).

Returns `{ plan, writes: { iterationActionsInserted, directorActivityRows, publishJobsInserted, amplifiedAdCampaignIds } }`.

**Injectable Meta seam.** `RunMediaBuyerOptions` accepts `metaExecutor?: MediaBuyerMetaExecutor` and `loadMetaToken?` ‚Äî defaults to the real [[../meta-ads]] exports; tests override with a fake so the inline execute path can be unit-tested without touching the real Graph.

### `executeDecidedActionAgainstMeta(args)` ‚Äî function

The pure inline executor for ONE decided `iteration_actions` row ([[../specs/media-buyer-decided-kills-must-execute-on-meta-not-just-be-recorded]] Phase 1). Calls the correct Meta primitive for the action type (pause ‚Üí `updateObjectStatus PAUSED`; unpause ‚Üí `updateObjectStatus ACTIVE`; scale_up ‚Üí `updateObjectBudget { dailyBudgetCents }`), then stamps the row: `success` ‚Üí `status='executed'` + `external_result` (Graph response) + `executed_at`; `failure` ‚Üí `status='failed'` + `external_result.error`. Row update is a compare-and-set on `status='decided'` so a re-run can never double-flip. Returns `{ success, external_result, error? }`. The runner's audit-line emit reads `success` ‚Äî a false-claim `media_buyer_paused_loser` for an unpaused ad is structurally impossible. Unit-pinned in `agent.test.ts` (success + failure cases for every action type).

### `escalateMediaBuyerExecuteFailure(admin, args)` ‚Äî function

Raises a deduped [[../tables/dashboard_notifications]] CEO card when the inline Meta execute call throws (or when the workspace has no Meta user token). `actionKind` is one of `media_buyer_kill_execute_failed` / `media_buyer_promote_execute_failed` / `media_buyer_reactivate_execute_failed` / `media_buyer_no_meta_token`. Dedupe key: `${actionKind}:${workspaceId}:${targetObjectId}:${yyyy-mm-dd}` ‚Äî at most one card per (object, kind) per UTC day. The card cites the decision rationale + Meta error verbatim so the CEO sees WHY the ledger row is `failed`, not `executed`. Same pattern as `escalateUnderProvisionedCohort` ‚Äî a compare-and-set on the dedupe key prevents 2h cadence spam.

## Sensor-trust contract ‚Äî dormant without a clean probe

**([[../specs/media-buyer-sensor-trust-probe]] Phase 3.)** The Media Buyer refuses to grade shadow-mode calls against untrusted spend/revenue. Every pass first loads the newest [[../tables/media_buyer_sensor_trust]] row for `(workspaceId, metaAdAccountId)` ‚Äî ordered `snapshot_date desc, limit 1` ‚Äî and gates the pass on:

1. **Present** ‚Äî a null row denies with `reasons=['missing_snapshot']`.
2. **Fresh** ‚Äî `now - created_at ‚â§ SENSOR_TRUST_MAX_AGE_MS` (48h), measured from row insertion (not `snapshot_date`, a date bucket) so a day-late probe run doesn't silently keep the pass alive on cold data. Past-cap denies with `stale_snapshot` appended to the probe's own reasons.
3. **Band ‚âÝ 'red'** ‚Äî a red band is the probe's explicit "sensor untrusted" verdict; the probe's own reasons flow through verbatim on the denial. Yellow is a warning the probe carries via its own reasons (unresolved-share nearing cap, thin spend allocation) ‚Äî the runner still proceeds; only red short-circuits.

A denial writes ONE `media_buyer_sensor_trust_denied` [[../tables/director_activity]] row + returns the same dormant summary shape the no-active-policy path uses (0 promote/kill/replenish, no writes, `plan.summary` names the denial reason). The row's metadata is `{ meta_ad_account_id, snapshot_date, band, coverage_ratio, reasons, autonomous:true }` ‚Äî the probe's numbers CITED, not paraphrased. Restore the pass by re-running the [[media-buyer__sensor-trust-probe]] lane (the box worker's `sensor-trust-probe` kind) ‚Äî a fresh green/yellow snapshot lifts the gate on the next cadence pass.

The pure `evaluateSensorTrustSnapshot` (DB-free) is the seam the unit tests pin the gate math against; the orchestrator's `readLatestSensorTrust` handles the read, and the runner writes the audit row + returns the dormant plan. This mirrors the pure/orchestrator split in [[media-buyer__sensor-trust-probe]] itself so the two libraries stay symmetric.

**Related scripts ‚Äî cold-test purchaser-overlap measurement** ([[../specs/bianca-measure-cold-test-purchaser-overlap]] Phase 1). `scripts/_measure-cold-test-purchaser-overlap.ts` is a one-shot dry-run-by-default measurement that produces the CITED overlap number the [[../goals/bianca-temperature-aware-campaign-structure]] M2 recent-purchaser-exclusion build is gated on. It enumerates every ACTIVE per-test cohort, joins [[../tables/meta_attribution_daily]] (attributed spend) with [[../tables/storefront_events]] (clicker identity, with orders.attributed_utm_content as a belt-and-suspenders fallback), expands each clicker's link group via `public.resolve_customer_link_group`, flags prior-purchasers by pre-first-click order, and writes ONE `media_buyer_purchaser_overlap_measured` [[../tables/director_activity]] row per cohort carrying `{ cohort_id, window_days, distinct_clickers, prior_purchasers, overlap_ratio, spend_cents_total, spend_cents_allocated_to_prior_purchasers, verdict }`. Verdict is `'proceed'` at overlap ‚â• 15% (the goal's threshold), else `'defer'`. Idempotent same-UTC-day short-circuit. See [[../recipes/measure-cold-test-purchaser-overlap]].

## Policy contract ‚Äî dormant without it

The Media Buyer refuses to autonomously act without an active [[../tables/iteration_policies]] row. On a pass with no policy:

- No `iteration_actions` writes.
- No `ad_publish_jobs` inserts.
- One `media_buyer_no_active_policy` [[../tables/director_activity]] row + a dormant summary in `plan.summary`.

The Growth Director (or a human) activates a conservative policy via `scripts/seed-media-buyer-iteration-policy.ts` ‚Äî that opens the loop. The seed uses [[iteration-policy-authoring]] `authorIterationPolicy` + `activateIterationPolicy` (never raw upsert on `iteration_policies`).

**Conservative seed values** ‚Äî 1.5√ó ROAS floor ¬∑ 3.0√ó scale trigger ¬∑ +15% step (cap 25%) ¬∑ $100 pause min-spend ¬∑ $10 per-pass account motion ceiling ¬∑ 24h per-object cooldown.

**Per-cohort calibration** ([[../specs/media-buyer-per-cohort-iteration-policy-calibration]]) replaces the hardcoded 1.5√ó/3.0√ó seed with a **data-derived per-cohort proposal**: [[media-buyer-policy-calibrator]] `runMediaBuyerPolicyCalibration` reads each cohort's realized ROAS + spend distribution (30d) + recent account spend (7d) and authors a `pending` [[../tables/iteration_policies]] row at `version = prior_max+1` with a rationale citing every quantile. The runner is gated on a `green` [[../tables/media_buyer_sensor_trust]] snapshot ‚Äî a `yellow`/`red`/missing snapshot defers via a `media_buyer_calibration_deferred` [[../tables/director_activity]] row (never activates). Activation stays with the Growth Director (via `propose_policy_activation`) ‚Äî the calibrator NEVER flips `status='active'`. The Growth Director's `buildGrowthDirectorBrief` already surfaces the new pending version under `iterationPolicies` proposals (see [[growth-director]]).

## Test-cohort defaults contract

For the replenish path to actually insert `ad_publish_jobs` rows, the [[../tables/media_buyer_test_cohorts]] row needs its Phase-2 default publish targets set: `default_meta_account_id`, `default_meta_page_id`, `default_meta_instagram_user_id`. Migration `20260707130000_media_buyer_test_cohorts_publish_targets.sql` adds them (all NULLABLE). Without them, replenish is deferred with `media_buyer_replenish_missing_config` ‚Äî the plan still emits the replenish action, the runner just doesn't fire the publish.

**Cold-test audience default = F50-65** ([[../specs/bianca-cold-test-audience-align-to-f50-65-converter]] Phase 1). `DEFAULT_TEST_TARGETING` in [[provision-cohort]] resolves to US women 50-65 (home+recent, Advantage+ Audience on) ‚Äî the proven Superfoods converter documented in [[../reference/meta-scaling-methodology]] ¬ß "Test audience held constant". `buildAdsetTemplate` falls through to it when a caller omits `targeting`, so every freshly-provisioned per-test cohort inherits the F50-65 shape and its per-test ad sets mint against the customer the cold-50+ creative is actually meant to sell. The pre-fix default (US 18-65, no gender filter) confounded the per-creative CPA read the M4 crown/kill call downstream depends on ‚Äî every per-test cohort was measuring against a pool the ad wasn't optimized for. Pinned in [[provision-cohort.test]] (`DEFAULT_TEST_TARGETING: pinned to the F50-65 converter cohort ‚Ä¶`) so a stray edit reverting to 18-65 fails at build. Existing live per-test cohorts are re-aligned via a one-shot idempotent backfill in Phase 2 of the same spec ‚Äî provision-cohort only mutates freshly-provisioned rows.

**Purchaser exclusion is baked into `targeting.excluded_custom_audiences`** ([[../specs/bianca-cold-test-recent-purchaser-exclusion]] Phase 2). `buildAdsetTemplate` accepts an `excludedCustomAudienceIds: string[]` option and merges it into the resolved targeting as `excluded_custom_audiences: ids.map(id => ({ id }))` ‚Äî Meta's required shape for a custom-audience exclusion on an ad set. Caller-supplied `targeting` that already carries its own `excluded_custom_audiences` key WINS (caller intent respected); omitting the option (or passing `[]`) leaves targeting exactly as the caller asked (no empty `excluded_custom_audiences` key is added). The runner's per-test enqueue helper (`buildReplenishJobInsert`) re-asserts the exclusion on the outbound `create_adset_spec.targeting` via `ensureExcludedPurchaserAudience(tmpl.targeting, cohort.excludedPurchaserAudienceId)` ‚Äî a belt-and-suspenders layer so a template that lost the exclusion (hand-edit / legacy pre-Phase-2 row) can never mint an ad set without it. The publish-gate refuses `missing_purchaser_exclusion` on any per-test publish whose spec doesn't. See [[media-buyer-publish-gate]] ¬ß `MediaBuyerTestRefusalReason` for the refuse enum + the two test pins on the gate.

**Under-provisioned ACTIVE cohorts escalate, not just audit** ([[../specs/media-buyer-cohort-adset-template-guard-backfill-and-escalate]] Phase 3). When a defer lands on an ACTIVE cohort (`cohort?.isActive === true`) ‚Äî because a per-test cohort is missing `test_meta_campaign_id` / `adset_template` (the Phase-2 [[provision-cohort]] `isCohortReplenishable` predicate), or the default publish targets aren't set ‚Äî the runner ALSO calls `escalateUnderProvisionedCohort` alongside the existing quiet `director_activity` row, raising a deduped CEO card on the [[../tables/dashboard_notifications]] inbox. This turns the silent 2/4 Superfood Tabs stall into a visible rail hit per [[../operational-rules]] ¬ß North star (a rail hit ESCALATES). Dormant cohorts are unchanged ‚Äî only an ACTIVE cohort's rail hit fires the card.

## Gotchas

- **The agent executes decided kill/promote/unpause INLINE on Meta before claiming the action** ([[../specs/media-buyer-decided-kills-must-execute-on-meta-not-just-be-recorded]] Phase 1). Before this spec, the runner upserted `iteration_actions` at `decided` and emitted `media_buyer_paused_loser` immediately ‚Äî the Phase-6a executor cron was supposed to fire on the next pass. That gap let four Superfood duds stay live at ROAS 0.00 for hours (three Creatine Prime + one Ashwavana Guru Focus), each burning $325-$354, while the ledger CLAIMED a pause it never made. The fix imports [[../meta-ads]] `getMetaUserToken` / `updateObjectStatus` / `updateObjectBudget` directly and calls them inline via {@link executeDecidedActionAgainstMeta} right after the upsert; the audit-line emit is gated on the row landing at `status='executed'` so a decided-but-unfired action ships ZERO claim rows. The Phase-6a executor still exists as a belt-and-suspenders backstop for any row the runner couldn't reach (token missing, process died mid-loop). Replenish is still routed through `ad_publish_jobs` ‚Üí the publisher, unchanged. A Graph failure raises a CEO card via {@link escalateMediaBuyerExecuteFailure} ‚Äî no rail hit is silent.
- **Loser's `source_meta_ad_id` is best-effort.** The scorecard grain is adset/campaign; the runner joins `meta_ads` in the losing adsets and picks the HIGHEST-SPEND child ad. When no children exist (a wrapped-empty adset), the fallback is the adset id itself. The pause target is unchanged ‚Äî this is just the audit citation.
- **Bianca honors the scale-edit rails on the promote path** ([[../specs/bianca-scale-edit-rails-cooldown-and-account-delta-ceiling]] Phase 1). The pure plan-computer reads `MediaBuyerPlanInputs.recentActions` + `nowMs` and drops any promote that would breach `per_object_cooldown_hours` (in-window scale on the same object) or `per_account_daily_budget_delta_ceiling_cents` (the pass's cumulative `|after-before|` would exceed the account's daily ceiling). Each dropped promote lands on `plan.deferred[]` with the specific `rail`, and the runner (Phase 2) writes ONE `media_buyer_scale_rail_deferred` [[../tables/director_activity]] row per entry ‚Äî the promote ledger explains suppression instead of the pass looking silently empty. Same seams the [[../meta/decision-engine]]'s `computeAutonomousActions` already enforces on scale actions; the two loops stay in step. The kill path is still unaffected ‚Äî kills never write budget and are unrelated to the delta ceiling.
- **Replenish is capped by BOTH the cohort deficit AND the ready-to-test bin.** A 4-target cohort with 2 live ads and 0 ready-to-test ‚Üí 0 replenish + a summary flag that the bin is exhausted. That's the correct autonomous behavior ‚Äî the Growth Director should top up the bin (via [[../inngest/ad-tool]] `ad-tool/generate-full`).
- **The test cohort is PRODUCT-SCOPED** ([[../specs/media-buyer-product-scoped-test-rail]] Phase 2). `DEFAULT_TEST_COHORT_TARGET = 4` ‚Äî each per-product cohort gets its own 4-live-test target, not one shared across products in the same Meta ad account. The runner reads `cohort.productId` and passes it to BOTH `listReadyToTest` (filters `ad_campaigns.product_id`, so product B's ready creative can never be selected for product A's cohort) AND `readCurrentTestCohortSize`. A null-product default cohort (Superfood Tabs today) omits both filters and preserves the pre-Phase-2 workspace-wide reads.
- **`readCurrentTestCohortSize` counts LIVE campaign adsets, ORIGIN-AGNOSTIC (fixed 2026-07-12).** For a per-test cohort (has `testMetaCampaignId`) it delegates to [[media-buyer-publish-gate]] `countLiveTestAdsetsInCampaign` ‚Äî counting live `meta_adsets` in the cohort's testing campaign (freed statuses excluded), NOT `ad_publish_jobs`. The old `ad_publish_jobs`-scoped count was blind to adsets minted by the *legacy* loop, so the **Amazing Coffee over-launch** read the current size as 0 (4 pre-existing skeptic adsets, no publish-job rows) ‚Üí deficit 4-0 ‚Üí replenished 4 ON TOP ‚Üí 8 live, double the $600 ceiling. The deficit is `cohortTargetCount ‚àí currentTestCohortSize`; an origin-agnostic count makes it impossible to exceed `maxConcurrentTests`. The null-product/legacy path (no campaign) keeps the old `ad_publish_jobs` count so Superfood Tabs is unchanged. The publish-gate ceiling is also product-scoped: `evaluateMediaBuyerTestPublish` accepts a `productId` and routes the cohort resolution to the per-product row, so A's over-ceiling projection is refused while the same amount under B's separate ceiling is allowed in the same account. Phase 3 fans the loop out over every active `(account, product)` cohort.
- **The Bianca-stuck-at-2/4 pin ([[../specs/media-buyer-replenish-per-product-scope]] Phase 1).** The regression the per-cohort count closes: a workspace with 25 workspace-wide live `media-buyer-test` ad sets against a product P cohort with 2 live in its own testing campaign ‚Äî the pre-fix workspace-wide count computed `deficit = 4 ‚àí 25 = 0` and replenish never fired for ANY product (Superfood Tabs sat at 2/4 despite 9 ready ad_campaigns in its bin). The per-cohort read (`readCurrentTestCohortSize({productId: P, testMetaCampaignId: CAMP_P})` ‚Üí 2) and the product-scoped ready bin (`listReadyToTest({productId: P})` ‚Üí only P's rows) together restore `deficit = 4 ‚àí 2 = 2` so the plan actually drains P's bin. Pinned in [[agent.test.ts]] (`Phase 1 pin ‚Äî cohort P has 2/4 live in its testing campaign against 25 workspace-wide ‚Ä¶`).
- **`buildReplenishJobInsert` ‚Äî the per-test enqueue artifact ([[../specs/media-buyer-replenish-per-product-scope]] Phase 2).** A PURE helper the runner's `enqueueReplenishPublish` calls after the DB reads; given `(cohort, action, campaign, video, adCopy)` it returns the exact `ad_publish_jobs` insert body. Two branches, one artifact: **per-test** (`action.adsetPerTest=true`) ‚Äî writes `create_adset_spec = { campaign_id: cohort.testMetaCampaignId, daily_budget_cents: cohort.perTestDailyBudgetCents, name: adName, ‚Ä¶adsetTemplate }` and forces `meta_adset_id: null` so the publisher MINTS a FRESH $150/day ad set under P's testing campaign (one ad per ad set) ‚Äî **never** targets `cohort.testMetaAdsetId` (null in the per-product model, but the invariant survives even if a stray edit repopulated it). **Legacy** (`adsetPerTest=false`) ‚Äî writes `meta_adset_id: cohort.testMetaAdsetId`, `create_adset_spec: null` for the pre-Phase-2 single-shared-adset shape. Fails CLOSED when a per-test cohort is missing `test_meta_campaign_id` / `adset_template` (never a malformed set). The publish-gate ceiling ([[media-buyer-publish-gate]] `evaluateMediaBuyerTestPublish` for `adsetPerTest`) runs at insert AND publish so `(live + 1) √ó perTestDailyBudgetCents ‚â§ dailyTestCeilingCents` is structurally impossible to violate. Pinned in [[agent.test.ts]] (`Phase 2 ‚Äî per-test replenish inserts a job whose create_adset_spec targets ‚Ä¶`).
- **The replenish read is TEMPERATURE-scoped to `'cold'`** ([[../specs/bianca-route-ready-creatives-by-dahlia-temperature-tag]] Phase 1). Every media-buyer cohort we ship today is a per-test cold cohort ([[../tables/media_buyer_test_cohorts]]), and Bianca's replenish fetch always passes `temperature: "cold"` to [[ready-to-test]] `listReadyToTest`. A Warm/Hot creative Dahlia tagged on `ad_campaigns.audience_temperature` cannot leak into the cold rail's `readyToTest.slice(0, deficit)` fill ‚Äî the M3 fix on the goal's crown signal, which is only meaningful when the tested set is temperature-uniform. Pinned in [[agent.test.ts]] (`agent.ts ‚Äî Phase 1 (bianca temperature-tag): replenish fetch calls listReadyToTest with temperature: "cold"`); the surfaced `ReadyToTestRow.audience_temperature` column lets downstream audits cite the routed band verbatim.
- **The `(account, product)` fan-out lives in `runMediaBuyerLoopForAccount`** ([[../specs/media-buyer-product-scoped-test-rail]] Phase 3). `RunMediaBuyerOptions` carries `productId`; the box worker's `media-buyer` lane calls `runMediaBuyerLoopForAccount(admin, {workspaceId, metaAdAccountId, cohortTargetCount})` per account, which reads `readActiveCohortProductIds(admin, {workspaceId, metaAdAccountId})` ‚Äî a sorted, dedup list of the account's active `media_buyer_test_cohorts.product_id` values (null-product last, no active cohort ‚Üí `[null]`) ‚Äî and calls `runMediaBuyerLoop({...opts, productId})` per entry. The pre-Phase-2 product-blind `listReadyToTest(admin, { workspaceId })` call is RETIRED from the replenish path ‚Äî a structural test in [[agent.test.ts]] (`agent.ts ‚Äî Phase 3 replenish path uses the product-scoped listReadyToTest`) pins the absence; any regression to it fails on the grep guard.
- **Bianca only sees max-QC-eligible creatives** ([[../specs/max-qc-always-bins-ad-7of10-gates-only-bianca-postability]] Phase 2). Dahlia now ALWAYS bins finished creatives ‚Äî a Max-below-floor exhaustion lands the last-attempted caption at [[../tables/ad_campaigns]]`.max_qc_eligible=false` instead of discarding it ‚Äî so the [[ready-to-test]] reader filters `.not("max_qc_eligible","is",false)` at the DB + a JS-side `if (c.max_qc_eligible === false) continue` guard to hide those binned-but-ineligible rows from Bianca's postable list. `TRUE` and `NULL` both surface ‚Äî NULL is the deterministic-mode / kill-switch-off / legacy pre-Phase-2 signal (Max never ran), so today's byte-for-byte behavior is preserved. The M1 keystone's north-star split ("Max's 7/10 gates POSTABILITY, not existence") lives in Bianca's read: the binned-but-ineligible row is still visible on the ad detail page with Max's critiques for founder inspection, just excluded from replenish's `readyToTest.slice(0, deficit)` fill.
- **Publish-gate re-check is the safety net.** Even though the runner uses `cohort.testMetaAdsetId` as the adset, the publisher runs the Phase-1 gate again on the queued job. A cohort retired between the plan and the publish gets caught + escalated ‚Äî the ad ships PAUSED.
- **`media_buyer_pass_completed` is ALWAYS emitted.** Even a no-op pass (no winners, no losers, no replenish) writes the heartbeat so the audit trail proves the pass ran. Absence of the row = the lane never fired = investigate the cron/enqueue path.
- **Shadow branch is a return-early carve-out, NOT a code-path fork.** ([[../specs/media-buyer-shadow-mode]] Phase 2.) The runner's shadow branch (`if (policy.mode === "shadow")` right after `computeMediaBuyerPlan`) emits the shadow rows + heartbeat and RETURNS. That preserves the armed executor writes by construction ‚Äî a stray edit that removes the check would silently start moving budget on a shadow policy, so a small structural test in `agent.test.ts` pins the branch predicate + the pure `buildShadowActivityRows` seam it calls. Never inline the shadow rules into the armed path; keep the carve-out at the top so armed behavior stays untouched.
